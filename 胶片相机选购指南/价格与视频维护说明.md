# 价格与 B 站视频维护说明

这份说明面向日常维护，不要求修改前端代码。

## 一、价格只改一张表

价格维护文件：

```text
胶片相机选购指南/data/camera-prices.csv
```

可以直接使用 Excel、WPS 或 Numbers 打开。保存时保留 CSV UTF-8 格式。

### 字段说明

| 字段 | 是否必填 | 说明 |
|---|---|---|
| `slug` | 是 | 机型唯一键，对应 `/cameras/<slug>`，不要随意修改 |
| `机型` | 是 | 方便阅读，不参与程序匹配 |
| `最低价` | 可空 | 功能正常普通成色的合理低位价，只填整数 |
| `典型价` | 可空 | 当前市场中更常见的价格，用于预算分组和重点展示 |
| `最高价` | 可空 | 普通交易范围的合理高位价，不包含收藏溢价 |
| `价格备注` | 可空 | 版本、颜色、附件和套机差异，例如“仅机身，不含镜头” |
| `成色口径` | 是 | 价格对应的成色和功能状态 |
| `市场` | 是 | 例如“中国大陆二手市场” |
| `核价月份` | 是 | 使用 `YYYY-MM`，例如 `2026-07` |
| `状态` | 是 | `有效`、`待核价` 或 `暂停推荐` |

### 推荐价格口径

默认口径：

> 仅机身，功能正常，普通使用成色；不含镜头、包装、收藏溢价和待修机。

中画幅、固定镜头便携机或常按套装交易的型号，可以在“价格备注”中单独说明。

### 修改示例

```csv
slug,机型,最低价,典型价,最高价,价格备注,成色口径,市场,核价月份,状态
nikon-fm2,尼康 FM2,1800,2200,2800,仅机身；黑色和收藏成色可能更高,仅机身，功能正常，普通使用成色,中国大陆二手市场,2026-07,有效
```

### 同步方式

在项目根目录运行：

```bash
node 胶片相机选购指南/scripts/sync-camera-prices.mjs
```

同步后会更新各机型 Markdown 的：

- `price_min`
- `price_typical`
- `price_max`
- `price_region`
- `price_checked_at`
- `price_condition`
- `price_note`
- `price_status`

详情页、相机图鉴、预算选机页和结构化数据都读取这些字段。正文中的手写价格卡片会被自动删除，避免同一机型出现两套价格。

GitHub PR 工作流也会自动运行同步脚本。CSV 中有重复 slug、数字错误、最低价高于最高价、典型价超出区间或缺少核价月份时，构建会直接失败。

## 二、新增机型时初始化价格表

首次建立价格表或确实需要从 Markdown 重新导出时运行：

```bash
node 胶片相机选购指南/scripts/export-camera-prices.mjs --force
```

注意：`--force` 会用 Markdown 当前字段覆盖 CSV，不适合日常核价后使用。平时只编辑 CSV，再运行同步脚本。

## 三、B 站视频与文章映射

视频维护文件：

```text
胶片相机选购指南/data/article-videos.csv
```

### 字段说明

| 字段 | 说明 |
|---|---|
| `article_path` | 对应文章的仓库路径，必须真实存在 |
| `article_title` | 方便阅读 |
| `bvid` | B 站 BV 号，例如 `BV1xxxxxxxxx` |
| `video_title` | 视频标题 |
| `creator` | UP 主名称 |
| `url` | 可留空，脚本会根据 BV 号生成链接 |
| `cover` | 可选封面链接；不填写则显示站内占位封面 |
| `duration` | 可选时长 |
| `role` | 视频在文章中的作用，例如“机型体验”“操作教学”“验机参考” |
| `search_keyword` | 待核验时用于人工搜索 |
| `status` | `有效`、`待核验` 或 `停用` |
| `notes` | 型号边界、版本提醒和编辑说明 |

### 视频上线规则

只有同时满足以下条件，视频才会出现在文章页：

1. `status` 改为 `有效`；
2. 填写真实 BV 号或具体视频 URL；
3. 填写视频标题；
4. 填写创作者。

“待核验”行不会生成空视频模块，也不会展示搜索关键词。

同步命令：

```bash
node 胶片相机选购指南/scripts/sync-article-videos.mjs
```

### 一个文章对应多个视频

在 CSV 中使用相同 `article_path` 增加多行即可。建议每篇文章最多保留 1～3 条真正有价值的视频，不要把页面变成视频列表。

## 四、日常维护流程

1. 在新分支修改 `camera-prices.csv` 或 `article-videos.csv`；
2. 本地运行对应同步脚本；
3. 检查变更后的 Markdown；
4. 提交 Pull Request；
5. 等待价格校验、类型检查、Quartz 构建和浏览器验收通过；
6. 查看 `/preview/` 页面；
7. 确认后合并。

不要直接在 `master` 上批量改 83 个 Markdown 的价格。

## 五、图片与参数维护

图片继续使用：

- `hero_image`
- `image_credit`
- `image_source`
- `image_license`

只有作者、原始文件页和许可都能核对时，才把图片接入页面。远程 Wikimedia 图片会由 `localize-images.mjs` 自动下载到仓库本地。

参数维护时优先核验：

1. 机身重量还是套机重量；
2. 快门是否依赖电池；
3. 无电时可用哪些速度；
4. 测光方式；
5. 电池型号；
6. 同系列版本差异；
7. 常见故障与二手验机重点。

每次核验后同步更新 `last_updated`，价格变化则更新 CSV 中的 `核价月份`。
