## 摘要

2026 年 AI 时代，数分最快求职路径压缩为 **Excel + SQL + 分析报告**；BI 可面试话术带过、入职再学。日常用得最多的是 Excel，考核硬门槛是 SQL，最难但最重要是分析思维。

## 各块定位

| 模块 | 2026 定位 |
| --- | --- |
| Excel | **用得最多**；只学一个工具就选 Excel；跨端、可嵌 SQL、可直接给业务 |
| SQL | **必考**；写不顺不配称分析师；笔试不过 HR/面试官都难捞 |
| 分析思维 | **最难回答、最重要**；只学工具无法掌握 |

| Python | 2026 很少分析师用 Python 做分析；AI 已覆盖大部分代码场景 |

- 求职负责：分析思维 + Excel + SQL；看板截图放作品集，话术过关即可
- 必做项目：报表、看板、**分析报告**（缺报告如同没写论文）

## 文章正文

戴师兄在这期短课里直接给出 **2026 年 AI 时代数分最快求职路径**：压缩到 **Excel + SQL + 分析报告** 三块；原来可能要 3～6 个月，现在**数周**可以跑通。

有观众问找第一段实习简历上写什么项目——他先说结论：分析师的数据呈现，基本就是**报表和看板**，这两个项目一定要有；再加上**分析报告**——「你分析半天写不出报告，就像大学毕业写不出论文，白干了」，这三类是必写项。

### 各工具在 2026 的真实权重

**Excel：日常用得最多。** 2026 年没几个分析师还在用 Python 做日常分析——Python 代码场景 largely 已被 AI 攻破；只有数据建设很烂的公司才会让你大量用 Python，建设好的至少用 SQL。Excel 最快，可以直接发给业务同事做模板，也可以在 Excel 里嵌入 SQL 不断刷新；还跨端，手机也能打开。「如果只学一个工具，学 Excel」——初学扎扎实实学，入职后也不吃亏。

**SQL：一定会考察。** 你是不是「正儿八经的分析师」，就看 SQL 熟不熟。写得一塌糊涂、逻辑像浆糊、从数据库取数都取不出来——「你不配称之为数据分析师」。大厂笔试 SQL 不过，HR 和面试官都没法捞你，「非常尴尬」。这块一定要捏好。

**分析思维：最难回答、最重要。** 面试官问「你会怎么分析数据、思路是什么」——只学工具、没有实际业务场景，掌握不了分析思维。他类比：你会各种画笔技法，但不知道艺术创作本身是什么，永远成不了出色画家。

**BI：可以不学（求职阶段）。** 这是很多人「大跌眼镜」的一点。入职后大概率要重学——以前是 Tableau/Power BI，后来 FineBI 多，现在各家公司自研 BI，尤其 AI 时代自建数据平台，你学的哪款入职后大概率对不上。建议：找一门课把看板**截图放作品集**，面试话术说「我会做 BI 看板」就过关，**入职后再学**——能省接近 1/4 学习时间。

**Python：** 2026 年很少分析师日常用 Python 做分析，AI 已覆盖大部分代码场景。

### 学习效率为什么要快

AI 提效背景下，学得慢就没有意义——你提示词捣鼓一两个小时才调出一个报表，不如自己八九个小时学会 Excel 来得快；领导从长期提效角度也是这么想的。但如果你花几百小时还学慢，「还不如让 AI 一直去套模板」。**只有学得足够快、足够扎实，人的价值才大于 AI**；学得好、学得快，牛逼分析师借助 AI 是放大器，基础执行岗才容易被替代。

### 2026 最快路径总结

**求职负责的三块：分析思维 + Excel + SQL**；看板截图放作品集、话术过关即可，不必深学 BI。必做项目：**报表、看板、分析报告**。这就是目前他给出的最快树根学习路径。
