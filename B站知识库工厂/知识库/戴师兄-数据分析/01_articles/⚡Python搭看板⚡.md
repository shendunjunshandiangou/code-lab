> **摘要**：在陌生领域用「需求驱动 + 查官网」主动选工具；演示 PyGWalker 拖拽出图与 Shiny 搭建 Web 看板。无硬性工具要求时 BI 拖拽仍最快。

## 一、被动学习的问题

跟教程被动学 Python 可视化常见问题：动力衰退、只会教程内图表不会扩展、工具版本过时跟不下去。

## 二、工具选择原则

- **无硬性工具要求时**：BI 工具（Tableau/Power BI）拖拽 **比写代码快、学习成本低**，是效率首选。
- Python 可视化生态工具极多（Matplotlib、ECharts、PyGWalker、Shiny 等），应 **按自身需求主动查官网** 而非只跟 B 站教程。

**需求示例**：熟悉 BI 拖拽，希望 Python 里也有类似 Tableau/Power BI 的操作 → 找到 **PyGWalker**。

## 三、PyGWalker：Tableau 式拖拽

- 官网：一行代码将数据集转为交互式可视化；拖拽字段分析。
- 与 Tableau 差异：度量需拖到 **X 轴** 等轴上才显示（非直接拖到「行/列」）。
- **AI 作图**：自然语言描述图表（如面积图展示不同期数成交金额随日期变化）；筛选器部分需手动补；AI 功能需 CMD 安装 + 官网订阅（有免费档）。
- **局限**：能作单图，**难以做多图仪表盘**。

## 四、Shiny：Web 看板框架

- 官方展示即为 **仪表盘** 形态；需配合独立作图工具（如 Plotly）嵌入。
- Shiny 提供 Web 应用框架与布局，作图库负责图表本身。

**搭看板四步**（类比做饭）：
1. **构思轮廓**：标题、指标卡、图表、布局（不必过细）
2. **确认组件**：对应 Shiny 的 `titlePanel`、`valueBox`、图表组件等
3. **逛官方文档**：在组件分类里找示例语法（如 `ui.valueBox`）
4. **照示例改参数**：KPI 改 GMV、换 logo 路径等；不会改可用 AI 兜底

**代码结构**：
- `ui`：布局（`layout_columns` 等）+ 组件（日期筛选器、valueBox、图表）
- `server`：响应筛选器，计算 GMV 等指标并返回图表，实现联动

## 五、主动学习的意义

从 0 到 1 自己探索官方文档产出第一个看板，比被动跟课更能建立迁移能力；面对 Web 需求时不只会想到 Flask/Django。

## 文章正文

本节主题是用 Python 做可视化看板，但戴师兄先花大量篇幅讲**为什么在陌生领域不能被动跟课**——这直接关系到你能不能产出。

他描述一个常见场景：领导让你用 Python 做可视化看板，你上 B 站搜教程，学几个小时甚至十几个小时，投入大量时间却**既没学会也没产出**。原因之一是**太被动**：等 UP 更新、教程只教折线图柱状图却要做饼图、2020 版软件和 2025 版界面对不上——动力衰退、动手能力衰退、信息质量不可控。解决办法是**主动学习**：先明确自己的需求，再按需求去官网查工具，而不是 B 站发什么学什么。

一个重要共识：**若领导/同事对工具没有硬性要求，只要可视化做出来**，在保证效率前提下，**BI 工具（Tableau、Power BI）拖拽永远比写代码快、学习成本低**。本节因直播主题是 Python，故仍演示 Python 路径，但不展开 BI 与 Python 的优劣辩论——「非常显然」。

Python 的本质是学习其**生态工具**——据称有近 62 万款包，可视化领域工具远多于 B 站常见的 Matplotlib、ECharts 等四五种。被动学习视野窄；主动学习的第一步是**依据需求选工具**：戴师兄的需求是——对 BI 拖拽很熟悉，希望 Python 里也有 Tableau/Power BI 式操作、少写代码。查官网：Matplotlib 偏静态/动画图表库，不符合；搜到 **PyGWalker**，官网一句话「一行代码将数据集转为交互式可视化」，界面像拖拽 BI，**主动锁定**。

PyGWalker 用法：按官方文档安装，导入数据，拖拽字段作图。与 Tableau 的差异：度量往往要拖到 **X 轴** 才显示，不是直接拖到「行/列」。数据用的是戴师兄 Excel 课的数据。它有 **AI 作图**：删掉面积图后用自然语言描述「不同期数的成交金额随成交日期变化」，执行后自动生成；筛选器部分 AI 加不上需手动补。AI 功能需 CMD 执行安装指令、官网登录订阅（有免费档）。

PyGWalker 的缺陷：**能作单图，难做多图仪表盘**——多张图拼一块很头疼，所以要继续找工具。搜到 **Shiny**，看官网首图就是仪表盘形态，「大概率是我要的」。Shiny 是 **Web 应用框架**，需配合独立作图库（如 Plotly）嵌入——像提供画框，你把「画」放进去。

搭看板四步（他用做饭类比）：**第一步构思轮廓**——标题、独立展示指标的 value box、图表、布局（可模糊但要有轮廓，如「酸口肉类菜」）；**第二步确认组件**——对应 Shiny 的 `titlePanel`、`valueBox`、图表组件；**第三步逛官方文档**——在组件分类里找示例（按钮不要、日期筛选器可能不要，看到图表和 value box 进去看 `ui.valueBox` 写法）；**第四步照示例改参数**——KPI 改 GMV、猪 logo 换猫图片路径；真不会改还有 AI 兜底。不断循环「逛文档—改参数—运行」，就能从 0 到 1 写出代码。

已写好的看板结构：`ui` 里 `layout_columns` 做布局——垂直布局放标题和日期筛选器；下一层 columns 放两个 valueBox；分割线；四张图同一布局组件内排列。`server` 定义交互：读日期筛选起止值，在数据里做日期过滤，算 GMV 等汇总塞进 valueBox，图表同理返回，实现联动。强调**第一个看板意义最大**——全程主动探索，做出第一个就能做第二个；被动学习遇到 Web 需求可能只会想到 Flask/Django，因为教程多教这些，但主动查文档会知道 Shiny 等更合适的框架。Python 很多工具免费，PyGWalker/Shiny 代码可进群获取。
