# 5年大厂分析师感悟

**摘要**：22 年一线大厂月薪约 3 万、14-18 薪、总包税前 45 万。大厂一天开会到下午，晚上加班做数据建设和专项分析；小厂随时被抓开会、修报表。分析师越老越吃香，30 岁不会被必然优化。

## 一、薪资参考

- 22 年某一线宇宙大厂：月薪 3 万左右（浮动 ±5000）
- 14~18 薪，年底一般 3~4 薪
- 年薪总包税前约 45 万，税后 35~36 万

## 二、大厂一天

| 时段 | 工作 |
| --- | --- |
| 早餐时 | 刷手机确认报表/看板/邮件有无报错 |
| 10:00~16:00 | 各种日会、周会、专项会议 |
| 午休 | 可能跟业务同事吃午饭收集信息 |
| 17:00 后 | 理需求，准备晚上工作 |
| 19:00~22:30 | 加班：数据建设/SQL、专项分析、业务推进跟踪 |

**晚上三件事**：
1. 梳理数据建设（能自己做的 SQL/标签自己写，不行找数仓套近乎）
2. 专项分析（业务达成率、下滑原因、策略效果 → 云文档报告）
3. 业务推进（跟踪策略落地，跟业务拉扯目标）

> 该碰业务还是要碰——未来职业发展差距就在业务理解上。

## 三、小厂一天

- 会无明确时间规划，随时被抓开会
- 报表挂了修不好（上游数仓问题）
- 天天洗数据、做自动化建设
- 领导想看的分析数据往往没有，得跟开发撕逼
- 能力够强、数据建设搞定后越来越好

## 三、三个高频问题

### 1. 加不加班？

跟职业无必然关系，看公司文化和数据建设完善度。数据自动化程度高、老板友善 → 加班少。前期技能不熟练肯定加班。

### 2. 工作重复吗？

- 数据处理：每次应是全新思考；一直重复 = 技术能力有问题，没做自动化
- 分析报告：日报周报模板化是幸福（简单）；异常排查才棘手
- 越重复难度越低，越个性化难度越高

### 3. 30 岁会被优化吗？

- 经验老辣、一直出结果 → 不会被优化
- 以学习和成长潜力下滑为界，非年龄
- 技术能力强能搞定数据建设 → 也有一席之地
- 分析师离老板更近，往往不是第一批被优化的

## 四、转型路径

先做半年运营 → 工作中用 Excel/BI 做数据汇报 → 拉开差距 → 转行数据分析。

## 文章正文

戴师兄在大厂做数据分析已五年多，这期系统分享薪资与工作体验——大厂、小厂都待过。

**薪资参考**：2022 年入职某一线「宇宙厂」，月薪约 3 万（上下浮动不超过 5000），14～18 薪，年底一般能拿 3～4 薪，**年薪总包税前约 45 万，税后约 35～36 万**。

**大厂典型一天**从早餐开始：一边吃一边刷手机，确认报表、看板、邮件有无报错——上游数仓表没跑出来、判断条件没击中特殊情况，都可能导致下游挂零。业务上班或开会前，数据必须到位；多次延误会被投诉，领导可能降薪、打低绩效甚至劝退。报错时要尽快梳理代码逻辑、修复上线。

上午 10 点起往往是**各种日会、周会、专项会议**，一直开到下午四五点；午休时还可能拉业务同事吃午饭，高效收集信息、对接需求。五点好不容易空下来，理一理晚上要做什么——**大厂分析师白天多在开会，晚上才有时间做自己的事**，加班到 9:30 或 10:30 赶加班车是常态（也看部门和领导风格，有的团队七点多下班）。

晚上工作大致三块：**第一，数据建设**——能自己写的 SQL、标签自己梳理；做不到的，晚上拿奶茶找数仓上游套近乎推进；不合理的需求及时写文档拒绝，有的开发以季度计。**第二，专项分析**——业务达成率、某时间点明显下滑、某商家效益变差、策略问题出在哪，最终落成云文档分析报告（小公司可能要求 PPT，形式不重要）。**第三，业务推进**——策略出来后还要跟踪落地，业务会反推「目标太高实现不了」，分析师权限有限，只能反馈上级，耗时多，很多大厂分析师不愿碰；但戴师兄建议**该碰还是要碰**，未来职业差距就在业务理解上。

**小厂体验**不同：会无明确时间规划，随时被抓开会、当场写 SQL、做 BI；报表因数据建设差常常修不好；大量时间洗数据、做自动化，领导想看的分析数据往往没有，得跟开发撕。能力够强、建设搞定后会越来越好；专题分析常要周末加班自学补信息。

三个高频问题：**加不加班？** 与岗位无必然关系，看公司文化、数据建设完善度、老板风格——自动化程度高、建设好，加班就少；前期技能不熟练，各行业都会加班。**工作重复吗？** 数据处理每次应是新思考；若一直重复同一操作，是技术有问题、没做自动化。日报周报模板化反而是「幸福」——简单；异常排查才棘手。越重复难度越低，越个性化难度越高。**30 岁会被优化吗？** 经验老辣、持续出结果就不会；界限是学习和成长潜力何时不如年轻人，而非年龄本身。技术强、能搞定数据建设，也能保住一席之地；分析师离老板更近，往往不是第一批被裁的。

**他的转型路径**：先做半年运营，工作中用 Excel 和 BI 做汇报，与别的运营拉开差距，运营岗上曾涨薪三倍（6000 到 1.8 万）；偷瞄 HR 屏幕见分析师 2 万多，决定在岗位上系统自学半年多，补 SQL、Python 和 analysis 思维，踩过贵且偏工具的课坑；靠项目看板上岸分析师，18K 跳到 19K（当时较 junior）。他强调：先做业务还是先做数分不影响，关键是**工作中把数据用起来、做出作品**；女生、文科生均可，大厂团队里女生很多，高中学科划分与工作维度不同，**兴趣才是最好的老师**。

三个误区：**先堆 Python 等硬技术**——应用最重要，有需要再学（工作要自动化、求职 JD 要求、或纯粹兴趣有余力）。**回避业务**——只做取数支持是工具人；不懂业务、不为企业创造价值，老板很快会降评级；企业需要的是能落地策略、直接赚钱的人，只做标准汇报和异常分析易触天花板。**只焦虑学历和环境却不学**——担心无用，只有尝试才知道成败；限制你的往往是内心声音，职场只有前进才有收益。
