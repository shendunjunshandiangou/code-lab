## 摘要

30 分钟入门爬虫核心：**网络请求**（客户端-服务器模式）+ 开发者工具抓包 + Python `requests` 发请求 + 按响应类型选解析工具，最终 `pandas` 导出 Excel。

## 网络请求理论

- 浏览器点击 = 向服务器**发起请求**，服务器**响应**返回数据
- 发起请求三要素：**请求方式**（GET 占 90%+，其次 POST）→ **请求内容/URL** → **请求参数**
- 失败常见原因：**身份校验**——非浏览器请求会被拒，需在 headers 携带浏览器身份信息（含 Cookie、User-Agent 等）

## 爬虫五步流程

1. 确定需求（爬哪个站）
2. 构建请求（方式 + URL + 参数 + headers）
3. 发起请求（代码代替手工点击）
4. 解析响应
5. 存储（如 Excel）

## 开发者工具用法

- 右键「检查」→ **Network** 面板 → 刷新 → Ctrl+F 搜关键词定位数据包
- **标头**：请求 URL、GET/POST、请求标头（身份）
- **负载**：请求参数
- **响应**：返回数据形态

## 案例一：B 站每周必看（JSON）

```python
import requests
url = "..."  # 到 ? 为止
params = {...}
headers = {...}  # 从请求标头复制，字典键值对
res = requests.get(url, headers=headers, params=params)
data = res.json()["data"]["list"]
# pandas DataFrame → to_excel
```

## 案例二：游戏排行（HTML / 纯文本）

| 响应类型 | 解析工具 | 说明 |
| --- | --- | --- |
| JSON | `.json()` | 最常见，结构清晰 |
| HTML | **BeautifulSoup** | 按标签定位（如 tbody），可让 AI 写提取规则 |
| 纯文本 | **正则 re** | 指定起止 pattern；有时需额外请求头字段改变响应格式 |

- HTML 示例：`BeautifulSoup(response.text)` 按 XPath/标签提取表格
- 文本示例：`re` 匹配后以 `json.loads` 转字典 → 取 `data` 字段 → DataFrame → Excel

## 工具栈

- **requests**：发 HTTP 请求
- **pandas**：结构化 + 导出 Excel
- **BeautifulSoup**：HTML 解析
- **re**：正则提取文本块

> 讲者观点：分析师用 Python 可视化只是可选项，BI 工具通常更合适（下节看板课展开）。

## 文章正文

戴师兄用约 30 分钟带大家入门爬虫，核心只讲一块：**网络请求**——用代码代替手工点击，向服务器要数据、解析响应、存到本地。他会用两个网站举例：B 站「每周必看」页面（JSON 响应）和某游戏数据网站（HTML / 纯文本响应），因为不同响应形态需要不同解析工具。

**网络请求的理论**用「客户—服务员」类比：你在 B 站首页点「热门」「每周必看」，相当于向 B 站服务器发起「我要看这类数据」的请求，服务器响应后页面才显示内容。发起请求分三步：**请求方式**（GET 占绝大多数，其次 POST）、**请求内容/URL**（告诉服务器要什么）、**请求参数**（描述属性，比如「第几页」「哪一周」「哪个分区排行榜」）。响应要么成功返回数据，要么失败——失败常见原因是**身份校验**：服务器发现请求不是浏览器发的就不给数据。解决办法是在请求里携带**浏览器身份信息**（headers 里的 Cookie、User-Agent 等）。

**爬虫五步**：确定需求（爬哪个站）→ 构建请求（方式 + URL + 参数 + headers）→ 发起请求 → 解析响应 → 存储（如 Excel）。实操前要用**开发者工具**：右键「检查」→ Network 面板 → 刷新 → 用 Ctrl+F 搜关键词定位数据包。标头里有请求 URL、GET/POST、请求标头（身份）；负载里是参数；响应里看数据长什么样。

**案例一：B 站每周必看（JSON）**。在标头确认 GET 方法和 URL（到 `?` 为止），从负载复制 params 字典，从请求标头复制 headers 字典（键值对格式，注意英文引号和逗号）。Python 用 `requests`：

```python
import requests
res = requests.get(url, headers=headers, params=params)
data = res.json()["data"]["list"]
```

JSON 返回花括号结构，用 `.json()` 解析；每周必看列表在 `data` → `list` 字段。再用 `pandas` 转 DataFrame，`to_excel` 导出即可。结构化输出可用 `pprint`。

**案例二：游戏排行（HTML / 文本）**。表格数据嵌在 HTML 标签里（如 `tbody`），需 **BeautifulSoup**——像「在大锅汤里用勺捞肉」，告诉它数据在哪个标签。可复制 XPath，或让 AI 写提取规则；直播里 DeepSeek 给的代码可能报错，讲者强调**写代码报错是常态**，像抛硬币一样常见，有备用代码即可。HTML 表格可能缺字段（如段位、位置），要补齐得继续找对应标签，本课不展开。

同一站点切换段位后，响应可能变成**纯文本**——此时用**正则 re**，指定起止 pattern 提取；有时需在 headers 里加特定字段（如 `rs1`），响应才会从 HTML 变成纯字符。提取出的字符串往往仍是「字典形态」的文本，用 `json.loads` 转字典，再取 `data` 字段里列表的第 4 项等，最后 DataFrame → Excel。编码显示乱码时，英文名仍可用。

**三种响应类型总结**：JSON 用 `.json()` 最直接；HTML 用 BeautifulSoup 按标签捞；纯文本用正则 + `json.loads`。大多数情况遇到 JSON；存储统一走 Excel。课程内会涉及少量加密内容，本直播未展开。讲者还回应弹幕：MATLAB 可控性强但不太适合分析师群体，**BI 工具通常是更好方案**；Python 做可视化是「可选项」，下节课讲看板。代码可扫码进群领取。
