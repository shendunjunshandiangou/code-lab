# Excel 合并表格专题

**摘要**：本课分两大块——先用 INDEX+MATCH 解决「宽表 + 字段翻译表 + 动态列查找」的函数方案，再系统讲解 Power Query 的三种数据导入方式、追加查询（上下合并）、合并查询（左右连接）及加载策略。

## 一、课后练习：宽表字段提取（函数方案）

### 场景

- **视频信息表**：133 列全英文宽表（爬虫数据），需提取 12 个中文字段。
- **字段对照表**：133 列英文名 ↔ 中文名映射。
- **目标表**：给定达人抖音号，返回对应中文列数据。

### 核心难点

1. 查找锚点是「达人抖音号」，但列名是英文，需经对照表翻译。
2. VLOOKUP 无法直接完成（查询列不在首列、需动态确定列号）。
3. 需 INDEX+MATCH 嵌套：先翻译中文列名 → 找列号 → 再找行号 → INDEX 取值。

### 紧急方案（20 分钟交差）

在对照表旁插入辅助列，用 INDEX+MATCH 把中文列名翻译成英文列名，再复制列调整顺序后用 VLOOKUP。

### 标准方案：INDEX+MATCH 动态查找

**列号**（以「达人昵称」为例）：

```
=INDEX(视频信息表区域, 1,
  MATCH(
    INDEX(对照表区域, MATCH("达人昵称", 对照表中文列, 0), 1),
    视频信息表英文表头行, 0
  )
)
```

逻辑：中文名 → 对照表找英文名 → 在宽表表头行找该英文名的列序号。

**行号**：同理，先翻译「达人抖音号」为英文列名，再在该列中 MATCH 具体抖音号值。

**完整公式**：`=INDEX(数据区域, 行号公式, 列号公式)`

填充时注意：列号公式中列引用 `$` 锁列，行号公式中 `$` 锁行。

## 二、Power Query 入门

### 入口

「数据 → 获取和转换数据」或「查询和连接」面板。Power Query 负责**获取 → 转换 → 加载**。

### 三种常见导入方式

| 方式 | 路径 | 适用场景 |
| --- | --- | --- |
| 来自表格/区域 | 当前工作表选中区域 | 先把区域转为「智能表」再导入 |
| 来自工作簿 | 其他 Excel 文件 | 选工作簿 → 导航器选 Sheet |
| 来自文件夹 | 指定文件夹路径 | 批量合并同结构多文件；只保留 Content 列后「合并文件」 |

### 数据源管理

- 所有导入路径默认为**绝对路径**；文件移动后需「数据源设置 → 更改源」重新定位。
- 结构相同的数据可直接替换源（如 6 月数据换 4 月）。

### 加载策略

- **关闭并上载至 → 仅创建连接**：中间表不加载到 Sheet，减少冗余。
- 在「查询和连接」面板右键目标查询 → **加载到 → 表 → 现有/新工作表**，只加载最终结果。
- 误操作全部上载时，可改回「仅创建连接」再选择性加载。

## 三、追加查询（上下合并）

- **追加查询**：把多张表**纵向**拼接，要求列名一一对应。
- 列顺序不同但列名相同可合并；列顺序不同且未按列名对齐会错乱（需改 M 语言，本课不展开）。
- 从文件夹导入时，需过滤掉：打开中的缓存文件（路径含 `~`）、非目标表（如人员表、汇总表）。

### 典型流程（课后作业：4–6 月销售汇总）

1. 从文件夹导入华东/华南/华西北 6 月三张表 → 合并为「新增销售数据」。
2. **合并查询**（左外部连接）：销售数据 ⋈ 人员表，连接键=销售工号，展开区域/省份/小组。
3. 导入 4–5 月历史数据 → **追加查询**到汇总表。
4. 关闭并上载至 → 仅创建连接 → 加载最终「销售汇总数据」。

## 四、合并查询（左右连接 / 表连接）

Power Query 中的「合并查询」= SQL 的 JOIN。

| 连接类型 | 含义 |
| --- | --- |
| 内连接 | 两表匹配键都存在的行才保留 |
| 左外部 | 保留左表全部行，右表只保留匹配上的 |
| 右外部 | 保留右表全部行 |
| 全外部 | 两表所有行都保留 |
| 左反/右反 | 只保留未匹配上的行 |

合并后右表以 Table 形式嵌入，需**展开**选择需要的列；取消「使用原始列名作为前缀」可避免列名冗余。

多连接键时按住 Ctrl 分别选两表的对应列。

## 五、从文件夹合并多工作簿多 Sheet

- 对每个文件执行「示例转换」，Power Query 按**列顺序**粗暴合并。
- **注意**：各 Sheet 列顺序必须相同，否则列会错位；列名相同但顺序不同需改 M 函数。
- 隐藏文件会被读取后自动删除（Attribute Hidden = true 的行）。

## 六、Power Query vs 函数

| 场景 | 推荐 |
| --- | --- |
| 一次性、列固定、行数少 | INDEX+MATCH / VLOOKUP |
| 多文件批量、需重复刷新 | Power Query |
| 数据量级大（几十万行） | 新建空表 + PQ 加载，避免直接打开大文件卡顿 |

Power Query 存储的是**操作步骤**而非数据本身，源表更新后点刷新即可。

## 文章正文

本课接上一节数据匹配，先用**课后练习**讲函数方案，再系统讲 **Power Query** 的导入、追加与合并——直播场景是抖音达人视频种草分析：一张 **133 列全英文宽表**（爬虫下来的视频信息表）+ **字段对照表**（英文列名↔中文名）+ **目标表**（给定达人抖音号，要 12 个中文字段）。

**为什么 VLOOKUP 不够**：锚点是抖音号，但列名是英文、目标表头是中文，中间还有翻译表——要「中转」。VLOOKUP 还要求查询列在区域第一列，这里对不上。透视表也不行：透视后列仍是英文，还是要找抖音号。**INDEX+MATCH 是数据匹配大杀器**，两函数嵌套可搞定动态列号+行号。

**20 分钟交差紧急方案**：在对照表旁加辅助列，用 INDEX+MATCH 把中文列名翻译成英文列名；或笨办法复制 A 列到 C 列、删 A 列调整顺序后用 VLOOKUP——「着急时达成目标就行」。

**标准 INDEX+MATCH 动态查找**：列号逻辑——中文名「达人昵称」→ MATCH 在对照表中文列找行 → INDEX 取对应英文名 → MATCH 在宽表**英文表头行**找列序号。行号同理：先把「达人抖音号」译成英文列名，再在该列 MATCH 具体 ID。完整公式 `=INDEX(数据区, 行号公式, 列公式)`；填充时列号公式锁列、行号公式锁行。直播带着学员一步步嵌套，强调五颜六色公式不要怕，拆步写即可。

**Power Query 入门**：入口「数据→获取和转换」或「查询和连接」。三种导入：**来自表格/区域**（先转智能表）、**来自工作簿**（导航器选 Sheet）、**来自文件夹**（批量同结构文件；只留 Content 列再「合并文件」）。路径默认绝对路径，文件移动要「更改源」；结构相同可整表换源（如 6 月换 4 月）。

**加载策略**：「关闭并上载至→**仅创建连接**」——中间表不堆 Sheet；最终查询右键「加载到→表」。误全部上载可改回仅连接再选择性加载。

**追加查询（上下合并）**：纵向拼接，**列名须一一对应**；列顺序不同但列名相同可合并，顺序不同且未按列名对齐会乱（需改 M 语言，本课不展开）。文件夹导入要过滤 `~` 缓存文件和非目标表（人员表、汇总表）。

**典型作业流程（4–6 月销售汇总）**：文件夹导入华东/华南/华北 6 月三张 → 合并「新增销售数据」→ **合并查询**左外部连接人员表（键=销售工号）展开区域/省份/小组 → 追加 4–5 月历史 → 最终「销售汇总数据」仅加载结果表。

**合并查询 = SQL JOIN**：内连接、左/右外部、全外部、左反/右反；合并后右表以 Table 嵌入需**展开**列，可取消「使用原始列名作为前缀」。多键按住 Ctrl 选两表对应列。

**多工作簿多 Sheet**：示例转换按**列顺序**粗暴合并——各 Sheet 列顺序必须相同，否则错位；隐藏文件读入后会删 Hidden 行。

**PQ vs 函数**：一次性、列固定、行少 → INDEX/MATCH 或 VLOOKUP；多文件批量、要重复刷新 → PQ；几十万行 → 空表 + PQ 加载，避免直接打开大卡死。PQ 存的是**步骤**，源更新点刷新即可——这是相对函数的核心优势。
