# 数据分析师是干什么的？

**摘要**：数据分析师只做两件事——满足业务数据需求（占 60%～80% 时间）和提供决策支持。前者概括为「取数、图表、大监控」；后者通过沟通会和分析会，结合数据给出方案并算投入产出比。

## 一、相关岗位

数据运营、数据产品经理、算法工程师、数据工程师等；本视频专讲**数据分析师**。

## 二、第一件事：满足数据需求（60%～80%）

核心工具链：

| 环节 | 工具/方式 |
| --- | --- |
| 取数 | SQL（核心语法简单） |
| 无现成数据 | 爬虫 / 提需求让开发建表 |
| 加工报表 | Excel |
| 自动化处理 | Python 脚本 |
| 日常监控 | Tableau / Power BI 等 BI 搭看板，自动拉数出图 |

**七字概括：取数、图表、大监控。**

## 三、第二件事：决策支持（价值所在）

- 结合数据与业务现状，分析可行方案，**算投入产出比**
- 两类会议：
  - **沟通会**：日/周/月会，收集整合信息，宏观全面视角（周初常开一整天）
  - **分析会**：针对业务痛点/难点，深度专项分析

## 四、小结

- 看似高大上，大量时间是工具性取数做表
- 有经验分析师的价值在**第二部分决策支持**

## 文章正文

数据类岗位很多（数据运营、数据产品经理、算法工程师、数据工程师等），本视频专讲**数据分析师**——戴师兄说其实只干两件事。

**第一件事：满足业务数据需求（占 60%～80% 时间）。** 日常用 SQL 去数据库取数——「核心语法只有三个，人人都能学会」。没有现成数据则自己爬虫或提需求让开发建表；拿到数据后用 Excel 加工成各式报表，复杂一点写 Python 脚本自动处理；需每日监控的数据用 Tableau、Power BI 等搭看板，自动从数据库拉数出图。**七字概括：取数、图表、大监控。** 看似高大上，大量时间是工具性工作。

**第二件事：决策支持（价值所在）。** 结合数据与业务现状，分析可行方案，**算投入产出比**。体现在两类会议：**沟通会**——日/周/月会，收集整合信息，宏观全面视角（每周一月初常开一整天）；**分析会**——针对痛点/难点（发展方向、成本优化、竞争策略、风险控制、可行性分析等）。以成本优化为例：先提取处理成本数据，准备报表/看板/预测模型；第一轮梳理确认需求（如区域经理问上月为何亏损——结合数据搞清楚亏损情况与成因，「看看还剩几条裤子」）；沟通中发掘潜在诉求（很多业务提亏损可能是争取资源或抱怨制度）。会后独立思考或头脑风暴，第二轮给出有数据支持的结论与方案，列优劣势和投入产出比；未解决则第三轮、第四轮直到彻底解决。**七字概括：开会分析给方案**（共 14 字）。

无论哪家公司的发展都离不开这些大大小小的会议；在这些会议上提供可靠决策支持，就是分析师的价值所在。若对以上内容感兴趣，不妨自己动手深入了解数据分析。
