> **摘要**：数分岗位 JD 普遍要求 SQL。本课从兴趣驱动、求职笔试面试、日常工作三档说明学到什么程度够用，并讲解 SQL 执行顺序、学习方法，以及 Top N 等套路题解法。

## 一、SQL 是数分的最低门槛

凡是涉及数据建设、取数、分析的岗位，JD 基本都要求 SQL。这是入门必备技能，必须能用 SQL 完成取数工作。

## 二、学到什么程度：三档标准

### 1. 兴趣驱动：不设上限

若对代码和 SQL 极感兴趣，不要自我设限，能学多少学多少。多学的内容日后都会派上用场。

### 2. 求职导向：三个 Level

| Level | 考察内容 | 说明 |
| --- | --- | --- |
| 语法 | WHERE/HAVING、聚合函数、ROUND、窗口函数、日期口径 | 不单独考函数，而是结合业务场景 |
| 套路题 | 连续登录、在线人数、Top N 等 | **笔试面试最常考**，没见过就做不出 |
| 逻辑嵌套 | 多层子查询、复杂条件组合 | 考察代码组织能力 |

**笔试规律**：前几题考基础语法（聚合、筛选），越往后越接近套路题（通常第 4、5 题）。

**面试规律**：面试官没时间提前出题，往往随口问套路题（如「两个字段用户 ID + 登录日期，算连续登录」）。

### 3. 工作场景：需求翻译

工作中与面试题最大区别：

- 业务方不会直接描述「成交金额介于 X 到 Y」，而是说「想看大单用户特征」
- 你需要把业务语言翻译成 SQL
- **很多需求做不了**——数据不支持或下钻不到那个粒度

## 三、SQL 执行顺序（面试常考）

```text
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
```

逻辑：先定位表 → 筛选减少运算量 → 聚合 → 聚合后再筛选 → 输出字段 → 排序 → 限制行数。

## 四、怎么学 SQL

1. **用 Excel 迁移理解**：每个关键字对应 Excel 什么操作
2. **输入**：刷题、课程、文章、面经
3. **输出**：还是刷题——输入输出都是刷题
4. **定位问题**：做不出题不一定是 SQL 不会，可能是**对数据处理流程没思路**（Excel 也不会做）

## 五、套路题一：Top N

### 核心思路

**先打标（窗口函数排名）→ 再取出排名第 N 的行。**

### 例题 1：入职倒数第三名员工

用 `DENSE_RANK()` 对入职日期倒序排名（同日期同排名），再 `WHERE rk = 3`。

> 为什么不用 `ORDER BY ... LIMIT`？因为同日期可能有多人，LIMIT 无法全部取出。

### 例题 2：每月播放 Top3 歌曲（多表 + 多条件）

步骤：

1. 流水表关联歌曲表、用户表
2. 筛选年龄 18–25、年份 2022、歌手周杰伦
3. 按月份 + 歌曲聚合播放次数（`COUNT`）
4. 窗口函数 `ROW_NUMBER()` 按月分区排名
5. 取 `rk <= 3`

### 三种排名函数

| 函数 | 同值处理 |
| --- | --- |
| `RANK()` | 跳号（1,2,2,4） |
| `DENSE_RANK()` | 不跳号（1,2,2,3） |
| `ROW_NUMBER()` | 强制不同序号 |

## 六、学习方法总结

- 套路题像刷数学题——**见得够多才能举一反三**
- 语法基础不牢先补窗口函数等基础
- 有思路后用不同工具（Excel/SQL）都能复现，才算真正掌握

`FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT`：先定位表 → 筛选减运算量 → 聚合 → 聚合后再筛 → 输出 → 排序 → 限行。

### 怎么学

1. 用 **Excel 迁移理解**每个关键字在做什么；
2. **输入**：刷题、课程、面经；
3. **输出**：还是刷题；
4. **定位问题**：做不出题可能是 Excel 也不会做、对数据处理流程没思路，不一定是 SQL 语法不会。

### 套路题一：Top N

**思路：先打标（窗口函数排名）→ 再取第 N 名。**

**例题 1（牛客）**：入职倒数第三名员工。同日期可能多人 → 用 `DENSE_RANK()` 对入职日期倒序排名，再 `WHERE rk = 3`。不能用 `ORDER BY ... LIMIT`，因同日期需全部取出。

**例题 2**：18～25 岁、2022 年、每月周杰伦歌曲播放 Top3。流水表 `MONTH(日期)` 聚合 PV → `ROW_NUMBER() PARTITION BY 月份 ORDER BY PV DESC, song_id` → `rk <= 3`；年龄、歌手筛选可子查询或 JOIN。示例里同播放次数按 `song_id` 升序 tie-break（坑在示例而非题干）。

三种排名：`RANK()` 跳号；`DENSE_RANK()` 不跳号；`ROW_NUMBER()` 强制不同序号。

### 套路题二：最大在线人数

日志表：用户、文章/视频、进入时间、离开时间**不在同一列** → **拆成两张表 UNION**：进入打标 +1，离开打标 -1，同一时刻**先记录增加再记录减少**（百度题细节），再按时间累加求 `MAX(uv)`。京东校招变体：视频峰值 Top3 用 `ORDER BY ... LIMIT 3`。

### 套路题三：连续登录

**核心技巧：日期减排名（等差数列）**。先去重每日一条 → `ROW_NUMBER()` 按用户、日期排序 → `DATE_SUB(日期, INTERVAL rk DAY)` 得临时日期 → 同一临时日期重复次数 = 连续天数 → `HAVING COUNT >= 2`；多段连续取 `MAX`。断点分组法：判断相邻日期间隔是否 >1 天打标，再累加。

### 套路题四：网易云推荐（逻辑嵌套）

向 user_id=1 推荐「关注的人喜欢的音乐，且不在本人已喜欢列表」。分段写：`follow` 表取关注列表 → `music_likes` 中 `user_id IN (...)` → `music_id NOT IN (本人已喜欢)` → 再 JOIN `music` 取名称。逻辑题要**慢慢写、分段组合**。

### 结语

套路题是工作场景凝练，「站在巨人肩膀上」；新题仍要见过足够多才能举一反三。Hive/MySQL 是「普通话与方言」关系，分析师学 MySQL 入门即可，数仓用 Hive SQL。0～1 数据建设：先梳理数据域、跟老板定优先级，再选型（影刀取数、SQL 清洗存储、FineBI/Tableau 应用）。
