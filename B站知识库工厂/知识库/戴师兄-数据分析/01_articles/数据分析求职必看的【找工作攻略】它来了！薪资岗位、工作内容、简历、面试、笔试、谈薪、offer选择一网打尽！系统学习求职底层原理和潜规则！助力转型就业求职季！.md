# 数据分析求职必看【找工作攻略】

**摘要**：系列课知识地图（8 分P）。先判断数分是否「卷」、天花板与入行条件；再讲求职底层原理（互联网本质、岗位分工、潜规则）；转型遵循「不可能三角」与「先就业再择业」；技能按岗位分层掌握；简历/笔试/面试/谈薪/Offer 系统展开。本文为框架级重写，细节见各分P原子笔记。

## 分P 1 · 入行前必问

### 数分卷不卷？

「卷」分**就业卷**（岗位少、门槛高、薪资低）与**从业卷**（事多钱少、压榨）。数分：

- 岗位数量：公司内数分 < 产品 < 开发 < 业务；少故薪资相对高
- 入门门槛：高于业务/产品，低于开发；需 Excel/SQL/BI/Python 等
- 对比：比业务卷、与产品接近；薪资高于产品/业务、低于开发
- 从业卷度因公司/团队而异，与岗位本身关系不大

**结论**：一分内卷一分收获；选方向看兴趣而非单纯「卷不卷」。

### 天花板与出路

- 薪资/职级天花板低于开发，与产品/业务持平
- 停滞在「半工具人」约 30 万者会觉得天花板低
- 三条出路：转算法、转数据工程（开发）、接管业务线做 leader
- 优势在**方向灵活**（进可攻退可守），非单纯天花板

### 学历 / 35 岁 / 城市

- 大厂校招 985/211 起步；**3 年后看履历不看学历**
- 35 岁危机是能力与成长速度问题，非年龄本身
- 互联网/金融数分集中在一线；想留二三线需先一线镀金再降维

### 先就业再择业

不一定先转数分；**有工作场景才能成长**。可先业务岗用数据分析提效，再视情况转数分。业务岗是优秀分析师的必备扎根阶段（「宰相必起于州部」）。

## 分P 2 · 课程地图与求职本质

- 求职/转型是**重塑职业身份**，非仅学技能过笔面
- **潜规则**：社招默认不培养新人——现成即用者优先
- 课程覆盖：底层原理 → 岗位薪资 → 转型路径 → 技能程度 → 简历笔面 → Offer 选择

## 分P 3 · 求职底层原理

- **互联网本质**：用信息替代资源，提高并发效率；「互联网行业」是标签，真正要积累的是**垂直赛道**（电商/游戏等）
- 公司核心竞争力：产品力、技术力、执行力 → 打工人要在某层面做到别人做不到
- 打工人需在产品/技术/执行某一维形成差异化能力

## 分P 4 · 岗位分工与薪资（框架）

成熟互联网公司五类：运营、产品、开发、数据、职能。

- **三驾马车**：产品（设计需求）→ 开发（实现）→ 运营（增长+维护）
- **数据岗位分散**：数仓/算法归开发；数据产品归产品；分析师常配业务线（行业分析、战略、经营分析）；集团数据科学管权限与对上服务
- 数据支持（取数报表）≠ 数据分析；算法/数仓门槛高、方向深

## 分P 5 · 转型路径

### 不可能三角

换工作时**行业、岗位、公司规模**最多同时满足两个。例：游戏运营→电商大厂数分，仅行业沾边，极难。

**建议优先**：行业 + 岗位——先在互联网找能写 SQL 的数据岗；退一步先做能接触数据的运营/业务，1～2 年后再转数分。

### 两条通用路径

| 路径 | 做法 |
| --- | --- |
| 路径一 | 数据专员（工具岗）→ 干满 1 年 → 转数据分析师 |
| 路径二 | 先做业务（运营/销售/客服等）→ 用 Excel/SQL/Tableau 提效 → 再转数分/数据运营 |

**严禁伪造简历**——背调、跨厂职业污点共享，阿里系等可能永久标签。

## 分P 6 · 技能掌握程度（摘要）

| 对象 | 要求 |
| --- | --- |
| 全体互联网人 | Excel 必会 |
| 产品/开发/运营 | SQL 熟练（信息权=管理权）；Tableau 做图；用数据说话 |
| 数据专员 | Excel + SQL + Tableau + Python（工具层） |
| 数据运营 | Excel + SQL；核心是业务理解与指标优化 |
| 数据分析师 | 工具 + **工作经验**（学习难以替代） |

Excel/SQL/Tableau 按自学攻略 + 作品驱动；面试常要求「18 般武艺」多于实际工作。

## 分P 7～8 · 简历 / 笔试 / 面试 / Offer（框架）

- 99% 第一版简历不合格；按 JD 对口、职能+结果、作品集
- 笔试：数分重 SQL/业务题，勤练牛客等
- 面试：经历真实、能力匹配、Offer 选择决定下一段体验与跳板价值
