# 连续登录套路题思路

## 一句话定义

连续登录是 SQL 面试最高频套路题之一：给定用户 ID + 登录日期，判断连续 N 天登录；面试常要求口述思路而非完整代码。

## 详细说明

### 为什么常考

- 属于固定套路，面试官可随口出题
- 考察日期处理、分组标记、窗口函数综合运用
- 听完思路即知水平，甚至不需完整书写

### 通用思路框架

1. 按用户 + 日期去重/排序
2. 用日期差或 `ROW_NUMBER` 做分组标记（同一连续段标记相同）
3. 聚合统计连续天数
4. 筛选 ≥ N 天的用户

### 与 Top N 的关系

同属「套路题」——没见过很难临场想出，需刷题积累。

### 面试技巧

能清晰口述每一步逻辑即可；社会招聘尤其看重思路熟练度。
