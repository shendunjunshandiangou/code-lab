# 项目介绍主动暴露问题法

## 一句话定义

面试讲项目按「目标 → 过程 → 结果 → 优化」四要素展开，并主动暴露一个已解决的严重问题——没有困难的项目不可信，主动竖靶子既引导面试官注意力，又把问题变成展示解决能力的机会。

## 详细说明

**四要素**：

1. **目标**：真实项目一定有明确目标（如提升转化率），学习型项目最大的破绽就是说不清「做这个分析干嘛」。
2. **过程**：用了什么工具、经历什么阶段、遇到什么困难、怎么解决（自学项目也有真实困难，如「函数嵌套太复杂改用 XLOOKUP 简化」「用 BI 替代 Excel 看板」）。
3. **结果**：可以小（提升 0.2%～0.5 个点）或定性（部门评价好），但要有。
4. **优化**：如果重做会怎么改进。

**主动暴露问题的逻辑**：面试官本来就要找问题，与其被抓住薄弱点，不如主动抛出一个严重但已解决的问题——「问题越严重、解决了越加分」，同时把注意力从其他薄弱处引开。想把项目写得一帆风顺反而招致「这么顺利？我来考考你」。

**边界**：遇到用「没做出结果」全盘否定过程、态度冒昧的面试官，面试是双向筛选，这类人做领导也难受。
