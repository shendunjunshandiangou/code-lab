# 数据工程四步与工具联动

## 一句话定义

数据技术需求拆为获取→清洗→存储→应用（BI 看板）；应学工具联动与工程思维，而非单工具穷举；存储位置决定清洗方式。

## 详细说明

**四步框架**（任何数据需求可用）：
1. **获取**：RPA / 后羿 / Python
2. **清洗**：Excel / Prep / PQ / Python
3. **存储**：Excel 文件夹 / MySQL / 数仓——**常把存储前置**，决定怎么洗
4. **应用**：BI 可视化看板

**示例链路**（电商后台→看板）：RPA 获取 → Prep 清洗 → 公司现有库 → Tableau/FineBI 呈现。

**关键**：选 **成本最低** 的链路串接；单工具永远学不完（Excel/RPA/PBI/MySQL/Tableau/Python 各含无数包）。

**Power BI 注意**：Power Query 无数据输出，PBI 常被当数仓用，数据治理难——链路设计时需知晓。
