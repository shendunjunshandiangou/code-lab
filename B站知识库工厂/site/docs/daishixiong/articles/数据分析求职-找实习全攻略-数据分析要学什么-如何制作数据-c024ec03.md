# 数据分析求职与找实习全攻略

**摘要**：求职是漏斗模型——先具备实力再投简历；无业务场景用公开数据集练手并用代码/作品证明能力；简历优先写分析经验；面试是独立能力需多练；选 offer 时前四五年重成长轻薪资，电商等数据量级大的赛道优先。

## 一、求职漏斗模型

整个求职过程是典型漏斗：

1. **外围观望** → 学习投入 → **具备实力的求职者**（最大筛人环节）
2. 投简历 → 面试 → 拿 offer → 成功入职

关键数据：完整数据分析课程完课率通常只有 1%～3%，真正学到能求职的人极少。一旦具备实力，后续环节难度大幅下降。

**应对漏斗三原则**：
- 先具备准入资格，再优化后续环节
- 目标先做前 10%，不要一上来就想第一
- **多做少想**：对方向感兴趣就先学，少在「能不能学会」上纠结

## 二、无业务场景：学什么、学到什么程度、如何证明

### 2.1 五步法

| 步骤 | 要点 |
| --- | --- |
| 数据来源 | 无业务数据时用和鲸社区/Kaggle 等公开数据集 |
| Excel | 报表制作 + 数据透视表/图，需熟练到老板站背后也能改 |
| SQL | 学懂核心语句 + 牛客刷题；**每行练习代码存档**，面试可展示 |
| Tableau | 能搭简易联动仪表盘即可 |
| Python | 核心掌握 pandas + sklearn；可视化优先 Tableau，Python 主要用于建模 |

### 2.2 工具证明方式

- **SQL**：代码存档 + 牛客刷题记录，空口无凭
- **Tableau/Python**：作品集链接或截图
- 无工作场景的在岗/在校生：靠作品证明技术实力

## 三、简历呈现顺序

**在职者**：分析经验 > 工具技能 > 项目经验 > 个人作品

- **分析经验**（最重要）：写个人简介首句，体现指标体系、专项分析等真实价值
- **工具技能**：Excel/SQL/Tableau/Python 各到什么程度
- **项目经验**：实习/工作中具体做了什么
- **个人作品**：链接放最后

四项都不具备时，先学习积累，不要急着改简历。

## 四、面试常见问题

### 4.1 三类提问

1. **工具掌握**：简历挑刺或开放式「你自己说说」——需提前准备好 1～10 分自评
2. **校招式**：对公司业务、行业历程、未来方向的见解
3. **灵魂拷问**：匹配度较高时才会出现，考察对分析师角色的理解

### 4.2 回答原则

- 不会的问题：**坦言不会 + 承诺回去分析补答**（可发分析报告），不要瞎扯
- 业务场景题：用**经营者视角**回答，不要消费者视角（如批评天天做活动太营销）
- **面试时长越长越好**，说明面试官兴趣高
- **面试是独立能力**，与笔试/工具技能分开；建议多面试练手，先面次优公司找回感觉

### 4.3 校招行业准备示例（电商）

- 发展历程：淘宝→天猫、京东高质量用户、拼多多下沉
- 未来方向：品类更好匹配、更快送达、与本地生活结合
- 当前玩法：365 天活动、满减发券、扶持新商家

## 五、选 offer 与行业

- **前四五年薪资不重要**，真正分水岭在**工作 4～5 年**；年轻时多学东西比起薪 30 万天天取数值
- 理想成长路径：第 1 年取数 → 第 2 年监控 → 第 3 年策略/AB 测试 → 第 4～5 年指标体系，年薪 40～50 万
- **行业选择**：优先互联网新兴/成熟且数据量级大的赛道；不确定时选**电商**
- 小城市数据岗需求弱，尽量大城市扎根；回小城市可重仓行业经验
