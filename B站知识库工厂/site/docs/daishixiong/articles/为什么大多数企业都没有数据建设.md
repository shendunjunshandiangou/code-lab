# 为什么大多数企业都没有数据建设？

<BiliPlayer :bvid="'BV1SKomYYELq'" />


**摘要**：局长（嘉宾）直播框架——数据建设≠只看报表，须含**获取、处理、存储、应用**四环节；仅应用（下载数据看一眼）不算建设。中小厂重应用，建设重心在前三板斧；多数公司「没有数据建设」实为前几环缺失。

## 一、数据建设定义

**数据建设 = 数据获取 + 数据处理 + 数据存储 + 数据应用**

- 重点常在前三块（建设）；应用=看板/日报/取数（中小厂日常）
- **仅有应用不算数据建设**——每家公司都会下载看销售额

## 二、为何感觉「大多数企业没有」

- 小公司以为数据建设「很重」接触不到 → 其实各规模都需要，程度不同
- 常见状态：有零散 Excel/后台导出，无系统采集、清洗、数仓
- 缺获取/处理/存储 → 分析师「一没数据二没权限」

## 三、各环节要点（框架）

| 环节 | 内容 |
| --- | --- |
| 获取 | 埋点、日志、业务系统录入、爬虫等 |
| 处理 | 清洗、ETL、指标口径统一 |
| 存储 | 数据库、数仓分层（ODS/DWD/DWS/ADS） |
| 应用 | BI 看板、分析报告、策略反馈 |

## 四、对分析师的影响

- 进公司先判断**数据成熟度**：有无数仓、埋点、统一指标
- 建设不完善时：先推动最小可用数据链路，或自建练手（参见无权限练手原子）
- 理解建设逻辑有助于与数据工程/产品协作
