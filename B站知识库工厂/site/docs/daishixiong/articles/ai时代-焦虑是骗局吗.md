# AI 时代，焦虑是骗局吗？

**摘要**：直播长谈——AI 焦虑来自预期落空、世界加速、评价标准模糊与身份动摇；「AI 能否替代人」是垃圾问题，应问「我的技能值多少钱」。学 AI 主线：LLM → Harness → Skills → CLI；把 AI 当下属，目标感要强。

## 一、焦虑从哪来：OpenClaw 案例

养虾/装 Agent 热潮的心理：
- 人人都想要**属于自己的 AI 助手**（通讯软件发令、操作本地文件、主动回访）
- 自媒体鼓吹 → 预期拉高 → 安装后产出不及预期 → **归因于自己不会用** → 焦虑传导

焦虑链路：**鼓吹抬高预期 → 捣鼓安装 → 产出失望 → 自责不会用 → 焦虑**

## 二、AI 加速带来的三层后果

1. **世界变化加速**：变化速度 > 个人成长速度；大脑从众本能（掉队=被抛弃）触发报警
2. **评价标准改变**：招聘/内部都在问 AI 落地场景，但企业和面试官往往**也不知道**该干嘛——「新考题」模糊
3. **身份感动摇**：程序员/分析师怀疑自己马上被替代，沦为「Yes 型确认员」

## 三、为什么说焦虑是「骗局」

- **「AI 能否替代人」是垃圾问题**：范围太大、无法验证、答案五花八门
- **应换成好问题**：「AI 出现后，我掌握的技能值多少钱？」——基于价格讨论劳动力
- 企业裁员本质是**经济账**（AI 更便宜），但定价权在模型厂商；真到全面替代时会涨价
- 行业替代需时间，会分化大众/小众赛道；**大众群体相对安全**，技能可迁移
- 两极端博弈（「AI 毁灭人类」vs「资本炒作」）→ 极端赌赢有流量 → 中间人被拉扯

## 四、AI 学习主线（四层）

| 层级 | 含义 | 代表 |
| --- | --- | --- |
| LLM | 大语言模型 | GPT、Claude、豆包、KIMI、Gemini |
| Harness | 模型「马鞍」/壳，赋予生产力 | Claude Code、Codex、Cursor、OpenClaw |
| Skills | Markdown SOP，稳定特定任务输出 | 触发式读取，不占满上下文 |
| CLI | 替代 MCP 省上下文 | Agent 自查命令文档 |

- OpenClaw = Harness（远程通讯+记忆+定时任务），不是新模型
- 产品新闻可归入这四类判断能力边界

## 五、正确协作方式

- **当下属**：不会的活丢给它也干不好；只能给灵感，决策靠人试错
- **目标感要强**：否则对话上瘾、token 浪费、决策次数暴增更累
- **拒绝 80→95 分的完美主义**：记录每件事耗时，清楚投入产出比
- 工作本质 = **收集信息 + 决策**；AI 加速执行但增加决策负担

## 六、缓解焦虑的落脚点

- 串起主线后，新产品是否在掌控范围内一目了然
- 焦虑源于「火力不足/积累不足」
- 数据分析师应掌握主线 + 调 Skills，而非追每个热点产品

## 文章正文

戴师兄从「AI 焦虑」切入，认为多数焦虑来自预期落空、世界加速、评价标准模糊与身份动摇；而「AI 能否替代人」是**垃圾问题**——应换成「我的技能值多少钱」。学 AI 的主线是 **LLM → Harness → Skills → CLI**。

### 焦虑从哪来：OpenClaw 案例

2026 年 2 月「养虾」热潮：有人花 500 元闲鱼安装 OpenClaw，装完没用又花钱卸载。心理根源是人人想要**属于自己的 AI 助手**（通讯软件发令、操作本地文件、主动回访）。传导链路：**自媒体鼓吹抬高预期 → 安装捣鼓 → 产出失望 → 归因「自己不会用」→ 焦虑**——教程多教安装，少教使用；装完说「你好介绍一下自己」就结束，说明本无真实需求。

### AI 加速的三层后果

1. **世界变化加速**：变化速度 > 个人成长速度；大脑「掉队=被抛弃」的从众本能报警（Cloud Code 等「一天一更」，学不过模型）。
2. **评价标准改变**：招聘和内部都在问 AI 落地场景，但企业和面试官**往往也不知道该干嘛**——像从未见过「公司征集哪里能用 Python」；新考题模糊，最痛苦。
3. **身份感动摇**：程序员/分析师怀疑自己马上被替代，沦为「Yes 型确认员」。

### 为什么说焦虑是「骗局」

「AI 能否替代人」范围太大、无法验证、答案五花八门——**垃圾问题**。应换成：**「AI 出现后，我掌握的技能值多少钱？」**——基于价格讨论劳动力。企业裁员本质是经济账（AI 更便宜），但**定价权在模型厂商**；真到全面替代时会涨价。

行业替代需时间，会分大众/小众赛道；**大众群体相对安全**，技能可迁移（诺基亚员工也可去小米、华为）。两极端博弈（「AI 毁灭人类」vs「资本炒作」）——极端赌赢有流量，中间人被拉扯；OpenClaw 像又一次左派话术。

### AI 学习主线（四层）

| 层级 | 含义 | 代表 |
| --- | --- | --- |
| LLM | 大语言模型 | GPT、Claude、豆包、KIMI |
| Harness | 模型「马鞍」，赋予生产力 | Claude Code、Codex、Cursor、OpenClaw |
| Skills | Markdown SOP，稳定任务输出 | 触发式读取，不占满上下文 |
| CLI | 替代 MCP 省上下文 | Agent 自查命令文档 |

OpenClaw = Harness（远程通讯+记忆+定时任务），不是新模型。MiniMax 等产品是在 Harness 层做 Skills 创建与更新；Obsidian 类知识库解决 Harness 长期记忆外挂。

产品判断：Manus = LLM + Harness；KIMI 比豆包强在接 Skills；DeepSeek 模型强但缺 Harness 生产力——接到 Claude Code 里同样能干活。

### 正确协作与缓解焦虑

- **当下属**：不会的活丢给它也干不好；只能给灵感，决策靠人试错。
- **目标感要强**：否则对话上瘾、token 浪费、决策次数暴增更累（工作本质=收集信息+决策）。
- **拒绝 80→95 分的完美主义**：记录每件事耗时，清楚投入产出比。

串起主线后，新产品是否在掌控范围内一目了然。**焦虑源于「火力不足/积累不足」**。数据分析师应掌握主线 + 调 Skills，而非追每个热点。国内第一梯队可先用 KIMI 与 MiniMax；豆包做 PDF 易乱码因缺 Skills——理解能力边界后不再用错工具。
