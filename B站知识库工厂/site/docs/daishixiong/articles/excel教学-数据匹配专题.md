## 一、VLOOKUP 基础语法

```
=VLOOKUP(查找值, 数据区域, 返回列号, 精确匹配)
```

**示例**：在数据表中查找值 `100`，返回第 4 列（ID 列）的数据：

```
=VLOOKUP(100, $A$2:$D$6, 4, FALSE)
```

### 两个最常见错误

1. **查询列必须在数据区域的第一列**  
   若把 ID 列放进查找区域但 ID 不在第一列，匹配会出错。修正方式：调整区域使查询列位于首列。

2. **返回列号数错**  
   列号是相对于查找区域的相对位置（1、2、3…），不是工作表绝对列号。

### 找不到时的表现

查找值不存在时返回 `#N/A`，表示未匹配到数据。

## 二、查询列不在第一列怎么办

VLOOKUP 硬性要求查询列在区域首列。两种替代方案：

### 方案 1：XLOOKUP（推荐，Excel 2021+ / WPS 新版）

```
=XLOOKUP(查找值, 查找列, 返回列)
```

- 查找列和返回列可以任意位置，不必在首列。
- 后三个参数（未找到值、匹配模式、搜索模式）可选，默认精确匹配、从头到尾搜索。
- **版本限制**：Excel 2010 及以下不支持 XLOOKUP，只能用 VLOOKUP 或其他方案。

### 方案 2：INDEX + MATCH

见第三节，适用于低版本 Excel 或需要更灵活控制的场景。

## 三、INDEX + MATCH 组合

### INDEX 函数

通过行序号和列序号，在指定区域返回单元格值：

```
=INDEX(数据区域, 行号, 列号)
```

### MATCH 函数

返回查找值在指定列/行中的位置序号：

```
=MATCH(查找值, 查找列, 0)    // 0 表示精确匹配
```

### 组合示例

查找列一中值为 `100` 对应的 ID（第 1 列）：

```
=INDEX($A$2:$D$6, MATCH(100, $A$2:$A$6, 0), 1)
```

- `MATCH` 返回 `100` 在第 1 列中的行号（如第 2 行）。
- `INDEX` 用该行号 + 列号 1 取出 ID 值。

**优势**：查询列和返回列位置完全自由，是数据匹配中「最大杀器」级别的组合。

## 四、WPS 与 Excel 选择

| 维度 | WPS | Excel |
| --- | --- | --- |
| 函数支持 | 新版支持 XLOOKUP 等高级函数 | 2010+ 支持大部分函数；XLOOKUP 需较新版本 |
| Power Query | 较弱 | 编辑器非常好用，适合大批量数据处理 |
| 兼容性 | 基本能打开 Excel 文件 | 不一定识别 WPS 专有函数 |
| 实际建议 | 办公室/领导指定用什么就用什么 | 写函数 WPS 够用；大批量清洗优先 Excel + Power Query |

## 五、函数选型速查

| 场景 | 推荐函数 |
| --- | --- |
| 查询列在首列、简单查找 | VLOOKUP |
| 查询列不在首列 | XLOOKUP 或 INDEX+MATCH |
| 需要双向/多条件灵活定位 | INDEX+MATCH |
| 低版本 Excel | VLOOKUP（调整列顺序）或 INDEX+MATCH |
