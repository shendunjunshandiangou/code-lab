## P1 · 基础查询核心概念

### FROM：准备表格

- SQL 是**查询语言**，不修改原表
- FROM 本质是告诉数据库「数据在哪个表」，逻辑上复制一张表用于查询
- **执行顺序上 FROM 最先运行**，但书写顺序 SELECT 在前（以终为始）

### SELECT：取字段

- 基于 FROM 准备好的表，取出指定列
- SELECT 的字段必须来自 FROM 指定的表
- 可在 SELECT 中做计算（后续题目覆盖）

### WHERE / ORDER BY / LIMIT

- WHERE：行级筛选（聚合前）
- ORDER BY：排序
- LIMIT：限制返回行数

### 与 Excel 的对应

每个 SQL 关键字都可映射到 Excel 操作来理解，这是戴师兄推荐的入门方法。

## P2 · 表连接与子查询

### 表连接

- SQL 无 VLOOKUP，必须 JOIN 拼表再取字段
- 工作场景以 LEFT JOIN 为主

### 子查询四种情况

1. **WHERE IN**：用子查询结果做范围筛选
2. **FROM 子查询**：用 SQL 替代一张表
3. **SELECT 子查询**：在 SELECT 中嵌套标量查询
4. **相关子查询**：内外查询有关联条件

### GROUP BY / HAVING

- GROUP BY：按字段分组聚合
- HAVING：聚合**后**筛选（WHERE 是聚合前）

## P3 · 窗口函数

### 定义

在**已准备好的表格**上，按分区（PARTITION BY）对行打标排名，**不改变行数**（与 GROUP BY 缩减行数不同）。

### 三大排名函数

- `ROW_NUMBER()`：强制连续序号
- `RANK()`：同值跳号
- `DENSE_RANK()`：同值不跳号

### 不受 ORDER BY / LIMIT 影响

窗口函数在 GROUP BY 之后、最终 SELECT 阶段运行，排名独立于外层排序。

## 文章正文

戴师兄 SQL 入门免费课共 3 个分 P、配套练习题平台逐题讲解。本正文按分 P 梳理核心教学逻辑与代表性知识点，详细 SQL 解法见各题实操与原视频。

### 分 P1 · 基础查询：FROM 与 SELECT 是 everything 的地基

第一课从 `SELECT name, 州份, 人口 FROM world` 这类最简查询入手——第一题只是让你「尝试运行」，真正讲的是 **FROM 和 SELECT 各自干什么**。

**FROM 的本质是「准备表格」。** 你得先告诉数据库数据在哪个表——就像叫小孩打酱油得先给小卖部地址。SQL 是**查询语言**，不会修改原表；FROM 找到指定表后，逻辑上**复制一张一样的表**用于查询处理（教学用的方便理解；实际引擎实现可能不同，但这样理解没错）。对应 Excel：打开叫 `world` 的工作表并复制一份，因为查询不能改原表，否则就不是查询语言了。**执行顺序上 FROM 最先运行**，但书写顺序 SELECT 在前——这叫「以终为始」：人写 SQL 先想最终要哪些字段列在 SELECT，再想字段来自哪张表写 FROM；数据库则必须先知道表在哪才能取字段。语法顺序（给人看）和运行顺序（给机器）不一致，很多编程语言都这样。

**SELECT 是在已准备好的表上取字段。** 可以把它想成 Excel 里把 name、州份、人口三列复制出来；SELECT 的列必须来自 FROM 指定的表——不是「万用点菜」，得先 FROM 备好「原料」。SELECT 里还可以做计算（后续题目覆盖）。`SELECT *` 表示取所有列，可与具体列并行使用——计算了某些列又想把全部字段查出时，直接加 `*` 即可。

后续 P1 覆盖 **WHERE 行级筛选**（聚合前）、**ORDER BY 排序**、**LIMIT 限制行数**、**DISTINCT 去重**、基础**聚合函数**（COUNT/SUM/AVG 等）。戴师兄反复强调：**每个 SQL 关键字都可映射 Excel 操作**——这是入门最快的方法。代码管理推荐 **Sublime Text**（或同类编辑器）存 `.sql` 文件，后缀触发语法高亮；代码存本地文件夹比只放 MySQL 里更安全——「离职了公司电脑代码全在库里，一出问题缓存清光就完蛋」。

### 分 P2 · 多表与进阶：JOIN 替代 VLOOKUP，GROUP BY 替代透视表

**表连接：** Excel 有 VLOOKUP，SQL 没有，必须 **JOIN 拼表再取字段**。工作场景以 **LEFT JOIN** 为主——保留左表全部行，右表匹配不上显示空值，「本质上就是 VLOOKUP」。两表连接前提是**有相同关联键**（如 id）——像非诚勿扰亮灯，id 相同则「牵手成功」出现在同一行。理论上只有 LEFT JOIN 也能通过调整表顺序实现各种连接；戴师兄从业正式写代码**基本不用 RIGHT JOIN**，需要时理逻辑改成 LEFT JOIN 即可。内连接、全连接等四种语法都要知道区别，但 LEFT JOIN 是日常主力。

**子查询四种情况**（求职高频）：① **WHERE IN**——子查询结果做范围筛选；② **FROM 子查询**——用一段 SQL 替代一张表；③ **SELECT 子查询**——在 SELECT 中嵌套标量查询；④ **相关子查询**——内外查询有关联条件。理解执行顺序：先 FROM/JOIN 准备表，WHERE 筛行，GROUP BY 分组，HAVING 筛分组，最后 SELECT 投影。

**GROUP BY / HAVING：** GROUP BY 相当于 Excel **数据透视表**——要对行做「合并求和/计数」就必须新建聚合后的表布局，不能只在原表每行上 SELECT。聚合函数（SUM/COUNT/AVG）不对 GROUP BY 使用时对整个表计算。**HAVING 是聚合后筛选，WHERE 是聚合前筛选**——「不合格的行在 WHERE 就被踢掉，不合格的组在 HAVING 才被踢掉」。若脑海中没有「求和/求平均」的表格概念，应先多练 Excel 透视表，否则 GROUP BY 很难深入。

P2 还逐题讲解字符串函数（如 LEFT 截取前 n 个字符）、**IFNULL/COALESCE 处理空值**（很多函数遇 NULL 会报错，需替换默认值；与 Excel IFNA 类似但逻辑细节不同，单学 SQL 函数即可，不必跨工具硬套）、日期函数等——这类函数靠场景积累，知道名字和用途后查文档运行即可。

### 分 P3 · 窗口函数：不改变行数的「组内排名」

窗口函数是 P3 核心。与 GROUP BY **缩减行数**不同，窗口函数在**已准备好的表**上按 **PARTITION BY 分区**对行打标排名，**行数不变**——像给每组同学发学号，人还在，只是多了排名列。

三大排名函数： **`ROW_NUMBER()`** 强制连续序号（1,2,3…）； **`RANK()`** 同值跳号（1,2,2,4）； **`DENSE_RANK()`** 同值不跳号（1,2,2,3）。笔试常考「每月 Top N」「入职倒数第几」等，必背模板。

关键性质：**窗口函数不受外层 ORDER BY / LIMIT 影响**——它在 GROUP BY 之后、最终 SELECT 阶段运行，排名独立于外层排序。理解这一点能避免「排好序 LIMIT 3 却发现排名乱了」的经典坑。
