# 24年底打工人就业现状！岗位方向大盘点！

**摘要**：数据岗位品类繁多，可归纳为算法、数分、数仓、开发、产品、运营六类。选岗应基于兴趣和能力匹配，而非执念某一岗位名称。每类岗位职责和门槛不同，需了解再选择。

## 一、数据岗位六类归纳

| 类别 | 核心工作 | 特点 |
| --- | --- | --- |
| 算法 | 开发算法模型（如价格阈值报警） | 门槛高，需编程和数学 |
| 数分 | 业务数据分析、报告、策略 | 最热门，业务理解重要 |
| 数仓 | 数据仓库建设、ETL | 技术导向，基建角色 |
| 开发 | SQL Boy / 数据开发 | 取数、报表开发 |
| 产品 | 数据产品需求沟通设计 | 做需求、沟通为主 |
| 运营 | 能接触数据的运营岗 | 偏执行，数据为辅 |

> 每家企业岗位定义和名称不同，列举无法穷尽。

## 二、如何选岗

- 不要对某一岗位名称有执念
- 先了解每类岗位**实际在做什么**
- 算法 ≠ 简单提一板数据，而是开发完整算法方案
- 产品 ≠ 设计高端页面，更多是**做需求、做沟通**

## 文章正文

局长这期做**数据岗位大盘点**，先互动：数据分析师、数据工程师、算法、SQL Boy、数据产品经理、数据产品运营……品类极多，每家企业定义不同。可归纳为**六类**：算法、数分、数仓、开发、产品、运营（此处运营指能接触到数据的运营岗）。

**用仓库比喻理解分工**：开发做扫描枪系统，数据入库；产品设计扫描流程、哪些字段录入、条码规则；数仓决定数据在库里怎么摆；算法告诉你何时「爆仓」、阈值报警；数分在数仓摆好后做经营分析、入库出库量等；产品大量时间是**沟通需求**——写文档、评审、排优先级，极其依赖沟通能力，是对内对外系统的「中间人」，不只做 App 页面，BI 平台也有产品经理。

**薪资（爬取招聘平台最新数据）**：算法遥遥领先；其次开发；数仓、数分、运营、产品四者薪资接近，不必纠结谁高谁低，与个人实力相关。数分排在**中间偏上**，不是底端，大众能接受。

**为何数分门槛相对可接受？** 学历：除算法外，**本科即够**，89%～90% 岗位不卡硕士——卡的是横向对比有人比你高。经验：开发偏右（5～10 年需求大但缺口也大）；算法偏 3～5 年；数仓 3～5 年；**数分、产品、运营对 1～3 年较友好**。技能难度：算法最难，开发≈数仓，数分略高于产品运营。综合门槛不高、薪资中等，符合「不想纯销售抛头露面、又想要一点门槛」的心态。

**理想 vs 现实**：想象进数分就做洞察、写报告；进去后可能**长期洗数据、做数据治理**——小处是理表字段，大处是计算与存储治理。业务也没空替你做分析，各岗位有存在必要性。

**算法要学什么（了解即可）**：Python/R、数学（线代、概率等）、真·算法（排序、数据结构、图论、DP、贪心、双指针）、机器学习与深度学习、大模型——「看完就不想学了」。文科转算法看会不会这些，与文理科标签无关。

**产品流程（大厂）**：需求收集 → 文档 → 评审 → 开发 → 测试 → 上线 → 迭代；核心能力是需求分析与沟通。

**数仓**：ETL、分层建模、SQL 熟练、Hive/Spark 等。**开发**：后端/前端/数据开发种类多，取数报表开发即 SQL Boy。**数分**：Excel、SQL、BI、业务理解、分析报告。**运营（数据向）**：在运营框架内用数据辅助决策。

选岗建议：**不要对岗位名称有执念**；为兴趣可选，但先知道实际做什么；算法 ≠ 提一板数，产品 ≠ 画高端原型。结合 24 年底市场：存量竞争、JD 抬高、纯取数岗缩减，按**兴趣 + 能力匹配**选，而非盲追热门名。

### 产品：BRD→PRD 与大厂标准流

业务提需写 **BRD**（背景、目标、demo 样式）→ BRD 评审 → 产品内部评优先级 → 写 **PRD**（实现方案，与研发对齐）→ PRD 评审 → 实施 → 验收。技术前期多不参与，顶多评审时提质疑。核心能力：写文档、沟通、原型设计、多项目并行管理。现实里「商业本质是交换」——产品也会把自己的需求塞进业务需求里，B2D 评审是博弈场，职场上需求优先级常靠开会吵架定夺。

### 开发、数仓、数分各学什么

**开发**：Java/C++/Go 等、数据结构算法（面试八股）、网络、操作系统、数据库事务/锁/索引/分库分表——查询 SQL 相对简单。**数仓**：从开发库 ETL 到分析库、洗脏表、**建模**（管理表粒度与字段，像图书馆排架）、Hive/Spark/Hadoop 组件，Scala 写 Spark 等。**数分必学**：Excel、SQL、BI 为主，Python 视公司与岗位；直播间共识未跑偏到 SPSS/R 科研工具。理想进数分做洞察，现实常先**洗数据、数据治理**——理字段、管口径；业务也没空替你做全套分析。
