# 干好这两件事，数据分析师就让你玩明白了

**摘要**：成熟分析师的核心就两件事——①提供数据支持，让业务看到想看的数据；②基于分析为业务出谋划策。SQL/Tableau 解决取数呈现，会议沟通补齐数据发现不了的问题。

## 一、第一件事：数据支持

业务常看的报表类型有限，用 **SQL + Tableau** 即可覆盖：

| 工具 | 作用 |
| --- | --- |
| SQL | 与数据库对话取数，核心语句约七个，可快速入门 |
| Tableau | BI 可视化，比 Excel 更美观、数据量更大、支持网页协作 |

## 二、第二件事：数据分析与出谋划策

- 业务面对海量数据无力深入，但不研究又不行
- 分析师通过业务公式计算，挖掘数据背后信息（如速度 = 路程 / 时间）
- **并非所有问题都能靠数据发现**，需结合业务场景
- 需积极参会收集信息、验证假设、辅助决策（大厂分析师一天两三场会很常见）

## 三、成熟标准

做到以上两点，即具备成熟数据分析师能力：既能稳定供给数据，又能参与业务决策。

## 文章正文

戴师兄用「干好这两件事，数据分析师就让你玩明白了」概括成熟分析师的核心工作。

**第一件事：提供数据支持，让业务看到想看的数据。** 业务常看的报表类型有限，用 **SQL + Tableau** 即可覆盖大部分场景。SQL 是与数据库对话取数的核心工具，核心语句大约七个，可快速入门。Tableau 是 BI 可视化工具——比 Excel 更美观、能处理更大数据量、支持网页协作，把 SQL 取出的数做成业务随时能看的看板。

**第二件事：基于分析为业务出谋划策。** 业务面对海量数据无力深入，但不研究又不行。分析师通过业务公式计算，挖掘数据背后信息——就像速度 = 路程 / 时间，把抽象指标拆成可理解的构成。**并非所有问题都能靠数据发现**，必须结合业务场景；因此分析师需要积极参会收集信息、验证假设、辅助决策——大厂分析师一天两三场会很常见。

做到以上两点——既能稳定供给数据，又能参与业务决策——就是成熟数据分析师的标准。SQL/Tableau 解决取数与呈现，会议沟通补齐数据发现不了的问题；工具是手段，理解业务、输出可行动的建议才是分析师的价值所在。
