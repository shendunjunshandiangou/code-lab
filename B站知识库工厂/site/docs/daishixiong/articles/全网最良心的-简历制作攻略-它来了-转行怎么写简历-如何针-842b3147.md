# 全网最良心的【简历制作攻略】

<BiliPlayer :bvid="'BV1G54y1h7rZ'" />


**摘要**：简历制作四步法——先盘点「你有啥」，再明确「你要啥」，接着学「怎么说」（职能+结果），最后决定「说哪个」（对口 JD 升级）。核心动作是拆解目标岗位的职能/工具/技能，把新技能真实用到旧经历里做模拟升级；校招重学历实习，社招重技能强度。

## 一、四步通关框架

| 步骤 | 含义 |
| --- | --- |
| 你有啥 | 盘点基本信息、教育、工作/实习、项目、技能、作品集 |
| 你要啥 | 拆解目标岗位 JD，弄清用人单位真实需求 |
| 怎么说 | 用「职能 + 结果」分点写经历，尽量量化 |
| 说哪个 | 按目标岗位调整展示顺序与内容重点 |

## 二、简历模块清单

### 2.1 基本信息

姓名、电话、邮箱、现居地填全；照片选能体现职业形象的（基础修图即可，目的是过简历筛选）。

### 2.2 教育经历

学校、学历、在读时间如实填写；绩点好可写，不好可不写。专业描述可灵活——如「国民经济管理」投互联网可写大类「经济学」；有奖学金、竞赛奖项写最拿得出手的。

### 2.3 工作/实习经历（最重要）

按**公司、岗位、时间、精力**梳理；描述用**职能 + 结果**结构——做了哪些事、量化提升了多少。

- 在职/实习时要像写日记一样记录素材，端茶倒水也能总结成「茶道与商务礼仪」
- 若内容少：先写能想到的，再用后文升级法补充

### 2.4 项目经历

填项目名称、角色、起止时间；描述同样「职能 + 结果」。一般 1～2 个精品即可；若工作/实习很少，可多写项目撑场面。项目来源：导师科研、社团大型活动等。

### 2.5 专业技能

- **熟练**：「技能领域 + 具体技能」连招（如数据分析：Excel / SQL / Tableau / Python）
- **一般**：「具体技能 + 掌握程度」
- 只写**含金量高**的证书奖项；英语四级、驾照等对多数岗位无加分

### 2.6 作品集

没经验者的逆袭机会——作品质量直接反映工作能力，无需额外背书。可放墨文档链接/二维码，HR 扫码即看；作品好则面试机会大增。

## 三、拆解目标岗位 JD

结合意向岗位 JD，拆成三方面：

| 维度 | 含义 | 数分示例 |
| --- | --- | --- |
| 职能 | 要你做什么事 | 指标优化、支持业务取数、日报周报、模型预测 |
| 工具 | 用什么做 | Excel、SQL、Power BI、Tableau、Python 等 |
| 技能 | 具体怎么做 | 指标体系、异动分析、AB 测试等 |

## 四、针对新岗位升级简历（转行核心）

1. 找出**原工作经历**与**岗位拆解**相关的职能
2. **学习并掌握**目标岗位对应的工具与技能
3. 把新工具/技能**用到**原工作职能里，**模拟**目标岗位职能
4. 基于原内容，输出与目标岗位相符的**全新经历描述**

**铁律**：只有真正掌握并实操过的新工具/技能才能写进简历；胡编迟早翻车，硬实力才是转型基础。

### 4.1 除对口外的四条加分

| 要素 | 做法 |
| --- | --- |
| 稀缺 | 你会的别人做不到 |
| 权威 | 大厂/名企/重要项目经历 |
| 喜好 | 真心热爱某行业/岗位 |
| 认同 | 接触过的人都说你好 |

## 五、整合与删减

- **校招**：更看重学历、实习公司
- **社招**：更看重技能、实际工作强度
- 原则：**Less is more**——少而精，不要花里胡哨堆无关内容，否则第一面变最后一面

## 文章正文

写简历，关键就在一个「写」字——为后续制作做好准备，具体要写哪些东西，戴师兄已经帮你整理成四步框架：**你有啥、你要啥、怎么说、说哪个**。更完整的简历全攻略篇幅很长，视频简介里都有；本期用最快速度带你通关最关键的四大模块。

**第一步：盘点「你有啥」。** 基本信息：姓名、电话、邮箱、现居地通通填好；照片要体现职业形象，基础修图即可——「不怕不要脸，先要过简历筛选」。教育经历：学校、学历、在读时间如实填写，绩点好就写，不好可以不写；专业描述可以灵活，比如「国民经济管理」投互联网可写大类「经济学」；有奖学金、竞赛奖项，写最拿得出手的。工作/实习经历是整份简历最重要的部分——有没有含金量，完全看这部分积累。每次经历按公司、岗位、时间、精力梳理，用「职能 + 结果」分点写：做了哪些事、量化提升了多少。在职或实习时要像写日记一样记素材，哪怕端茶倒水也能总结成「茶道与商务礼仪」；若发现没啥好写，多半是之前没认真积累——以后长点心，在职就把做过的事记下来，下次写简历保证有素材。若已经来不及找新实习、工作内容有限，先写能想到的，后面还有升级法。项目经历填项目名称、角色、起止时间，描述同样「职能 + 结果」；一般一到两个精品即可，若工作/实习很少，就多准备项目撑场面。项目来源可以是导师科研、社团大型活动等——「只要思想不滑坡，办法总比困难多」。专业技能：熟练的话用「技能领域 + 具体技能」连招，比如数据分析写 Excel / SQL / Tableau / Python；一般的话用「具体技能 + 掌握程度」。只写含金量高的证书奖项，英语四级、驾照等对多数岗位无加分。作品集是没经验者最后的逆袭机会——作品质量直接反映工作能力，不需要额外背书；可放墨文档链接或二维码，HR 扫码即看，作品好则面试机会大增。

**第二步：明确「你要啥」。** 结合意向岗位 JD，拆成职能、工具、技能三方面：职能是「要你做什么事」（如指标优化、支持业务取数、日报周报、模型预测）；工具是「用什么做」（Excel、SQL、Power BI、Tableau、Python 等）；技能是「具体怎么做」（指标体系、异动分析、AB 测试等）。上一期戴师兄还分享过如何爬取意向岗位信息并提炼 JD——拆解清楚后，才能有的放矢调整简历。

**第三步：学「怎么说」——针对新岗位升级（转行核心）。** 找出原工作经历与岗位拆解相关的职能；学习并掌握目标岗位对应的工具与技能；把新工具/技能用到原工作职能里，模拟目标岗位的职能；基于原内容，输出与目标岗位相符的全新经历描述。理论上听起来复杂，举个例子就懂：小白把原工作中与数据相关的部分，用新学的 SQL、BI 工具重新描述成「支持业务取数、搭建看板、输出分析报告」——句式对齐 JD，但内容必须真实。这条铁律要记住：**只有真正掌握并实操过的新工具/技能才能写进简历**；胡编迟早翻车，硬实力才是转型基础。除对口外，四条加分要素是：稀缺（你会的别人做不到）、权威（大厂/名企/重要项目）、喜好（真心热爱某行业/岗位）、认同（接触过的人都说你好）。

**第四步：「说哪个」——整合与删减。** 校招更看重学历和实习公司，社招更看重技能和实际工作强度；展现顺序和注意事项不同，按攻略排版、逐一检查即可。原则是 Less is more——少而精，不要花里胡哨堆一堆毫无重点的内容，否则 HR 的第一面可能就是最后一面。教程不管多详细，简历还要自己写——不如抓紧时间，赶快去改简历。
