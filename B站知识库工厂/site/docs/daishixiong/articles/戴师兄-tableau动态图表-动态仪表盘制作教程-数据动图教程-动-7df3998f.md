> **摘要**：介绍动态可视化/动态仪表盘的主题选择与数据准备、Tableau Public 安装，并在 Public 社区用 SQL Query 取数。**字幕仅含 P1 与 P3，P2 动态图表/仪表盘实操部分缺失。**

## 一、动态可视化主题与价值

- 随时间变化的过程都可用动态可视化展现（如汉斯·罗林的 Gapminder）。
- **在职**：结合业务指标的时间趋势，直观呈现贡献。
- **求职/实习**：可选时事热点主题（如疫情发展、综艺人气排行），更易产生代入感。

## 二、数据准备

**数据来源**：
- 天池、Kaggle（科赛）等数据科学社区
- Tableau Public 官方社区（本教程数据，含日期字段且每日更新）
- 也可手动整理（如 UP 主自录 B 站播放量）

**要求**：数据须包含 **时间 + 对应数值** 方可做动态图表。

**取数方式**：社区页面可直接下载，或点击 **Query** 用 SQL 查询后下载 CSV/XLSX。

## 三、Tableau Public 安装

- 从官网下载，默认安装；不推荐破解版（易出现图标不显示等 bug）。
- Public **无法保存到本地**，但足够练习 Tableau 用法。
- 正式版见 Tableau 安装教程；本教程全程用 Public 演示。

## 四、Public 社区 SQL Query 取数（P3）

### 4.1 工作场景

分析师在实际工作中对着 Web 界面写 SQL 取数，**不需要**自行搭建 Hive/Spark 等大数据框架，但需了解框架特性以避坑。

### 4.2 基础语句演示

以新冠数据为例，核心语句：

```sql
SELECT *
FROM 数据库名
LIMIT 5000
```

- `SELECT` 读取字段；`*` 表示全部（不熟时产品经理也常用，再 Excel 处理）。
- `FROM` 指定数据源。
- `LIMIT` 限制返回行数，**调试代码时常用**，避免大数据量长时间加载。

**习惯**：SQL 不区分大小写；实际工作中小写即可；良好换行便于阅读；须用英文输入法。

### 4.3 字段理解与数据结构探查

通过 `DISTINCT` 查县名、州名等字段，发现：
- 美国有县/州级数据，加拿大有州级，多数国家仅国家级日数据。
- 熟练分析师可通过去重、字段对比快速判断数据结构；也可下载后在 Excel 透视或 Tableau 拖拽验证。

### 4.4 国家日汇总查询

取国家、汇报日期、大洲及四个指标字段，加 `GROUP BY` 按国家+日期汇总（加拿大等同日多行需 SUM）。

**GROUP BY 规则**：SELECT 中非聚合字段都须出现在 GROUP BY；数据库按 **最细维度** 汇总。

**别名**：字段后空格直接写别名（不必用 `AS`）；中文别名需单引号且并非所有库支持。

### 4.5 筛选与排序

- `WHERE country = 'China'` 单国筛选
- `WHERE country IN ('China', 'Canada')` 多国
- `ORDER BY date DESC` 降序；多字段排序：`ORDER BY country, date DESC`

### 4.6 下载

Query 完成后 Download → CSV（大容量）或 XLSX（给业务）。

**掌握 SELECT / FROM / WHERE / GROUP BY / ORDER BY / LIMIT 六语句 + 聚合**，SQL 已掌握约一半。
