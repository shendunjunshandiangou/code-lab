> **摘要**：在陌生领域用「需求驱动 + 查官网」主动选工具；演示 PyGWalker 拖拽出图与 Shiny 搭建 Web 看板。无硬性工具要求时 BI 拖拽仍最快。

## 一、被动学习的问题

跟教程被动学 Python 可视化常见问题：动力衰退、只会教程内图表不会扩展、工具版本过时跟不下去。

## 二、工具选择原则

- **无硬性工具要求时**：BI 工具（Tableau/Power BI）拖拽 **比写代码快、学习成本低**，是效率首选。
- Python 可视化生态工具极多（Matplotlib、ECharts、PyGWalker、Shiny 等），应 **按自身需求主动查官网** 而非只跟 B 站教程。

**需求示例**：熟悉 BI 拖拽，希望 Python 里也有类似 Tableau/Power BI 的操作 → 找到 **PyGWalker**。

## 三、PyGWalker：Tableau 式拖拽

- 官网：一行代码将数据集转为交互式可视化；拖拽字段分析。
- 与 Tableau 差异：度量需拖到 **X 轴** 等轴上才显示（非直接拖到「行/列」）。
- **AI 作图**：自然语言描述图表（如面积图展示不同期数成交金额随日期变化）；筛选器部分需手动补；AI 功能需 CMD 安装 + 官网订阅（有免费档）。
- **局限**：能作单图，**难以做多图仪表盘**。

## 四、Shiny：Web 看板框架

- 官方展示即为 **仪表盘** 形态；需配合独立作图工具（如 Plotly）嵌入。
- Shiny 提供 Web 应用框架与布局，作图库负责图表本身。

**搭看板四步**（类比做饭）：
1. **构思轮廓**：标题、指标卡、图表、布局（不必过细）
2. **确认组件**：对应 Shiny 的 `titlePanel`、`valueBox`、图表组件等
3. **逛官方文档**：在组件分类里找示例语法（如 `ui.valueBox`）
4. **照示例改参数**：KPI 改 GMV、换 logo 路径等；不会改可用 AI 兜底

**代码结构**：
- `ui`：布局（`layout_columns` 等）+ 组件（日期筛选器、valueBox、图表）
- `server`：响应筛选器，计算 GMV 等指标并返回图表，实现联动

## 五、主动学习的意义

从 0 到 1 自己探索官方文档产出第一个看板，比被动跟课更能建立迁移能力；面对 Web 需求时不只会想到 Flask/Django。
