# 数据看不懂，分析全白搭：三步读懂数据

<BiliPlayer :bvid="'BV1LwF5zEE32'" />


**摘要**：分析师最匮乏的往往是「读懂数据」而非新工具。读懂数据三步：明确每一行描述什么（粒度）→ 做假设 → 用交叉计数验证。维度是分析核心，粒度错了结论必错。

## 一、为什么先学「读懂数据」

年末学东西应满足：**最匮乏** + **投入产出比高**（耗时短、见效快）。

很多人觉得自己会 SQL/BI，但说不清「数据能力是什么」——本质是**读取 → 处理 → 应用**。不会描述问题时，就拆流程；本课聚焦**读取数据**。

## 二、读懂数据三步法

### 步骤 1：明确粒度——每一行描述什么

读任何表先问：这一行代表什么事件/对象？

### 步骤 2：做假设

根据字段名和业务常识，先猜粒度（如「每部电影一条」「每个游客一条」）。

### 步骤 3：技术验证

用透视表/交叉表对维度做**计数**：
- 计数唯一 → 假设可能正确
- 出现重复 → 假设错误，找「哪里不同」（日期、地点、ID 等）
- 重复消除后计数唯一 → 得到真实粒度

> 业务主键 ≠ 技术主键；验证比字段名更可靠。

## 三、案例要点

| 数据集 | 错误假设 | 真实粒度 |
| --- | --- | --- |
| 电影票房 | 每部电影一条 | 每部电影在特定城市/日期的播放数据 |
| 旅游消费 | 每个游客一条 | 每个用户在特定日期去特定景点的花费 |
| TikTok 视频 | （需验证） | 每个 video_id 一条 |
| 携程美食 | 每个店名一条 | 每个 shop_id 一条（同址关店重开 id 会变） |
| 美食评论 | 每店每用户一条 | 每用户每店**每次评论**一条（同用户可多次评论） |

## 四、维度是分析核心

- 分析思路从**维度**出发，再拆度量
- 电商数据若只有日期+度量、缺维度，只能看趋势，无法深入
- 读懂粒度后，分析方向自然浮现（如景点客流、菜系均价、用户复评）

## 五、与「会不会」的元认知

「什么是桌子」——脑子里分得清，但说不清。面试/交流时需能**重新定义**概念；数据同理：以为会分析，其实读不懂粒度就会算错。
