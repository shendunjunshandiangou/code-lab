# 40分钟戴你梳理【数据分析】核心知识点

**摘要**：从思维到工具梳理数据分析核心知识——流程分析（侦探式了解业务）、时序分析（同环比找参照物）、行为漏斗（AARRR 各环节优化）、矩阵分析。学会三大分析方向，月薪上万轻轻松松。

## 一、分析基础：流程分析

把自己当**侦探**——竞品为什么做得好？业务下滑要往前倒推：定目标 → 策划 → 准备 → 执行，每一步发生了什么。

> 连业务信息都不了解，不可能有分析结论。像谈恋爱不知道对方喜好就无法吸引。

## 二、三大分析方向

### 1. 时序分析

所有基于时间的同比、环比、预测都是时序分析。

**核心目的**：在时间上找到有效参照物——现在比之前好就总结为什么好，差就总结为什么差。只有一天数据无法分析好坏。

### 2. 行为漏斗

AARRR 等增长模型本质是梳理用户各时间点做了什么：注册→激活→登录→领礼包→体验→活跃→付款→推荐。每个业务实际流程不同。

**做法**：先拆流程漏斗 → 再看每个环节跟之前比何时好何时坏 → 60% 基本盘就清楚了。

### 3. 矩阵分析

（原文讲解用户分群、产品矩阵等二维交叉分析，找到优化杠杆点。）

## 三、工具层（概要）

- Excel：取数、透视、基础报表
- SQL：数据库查询
- Tableau/BI：可视化看板
- Python：自动化、高级分析

> 思维先于工具。有分析思维后工具是执行手段，月薪上万轻轻松松。

## 四、学习路径建议

1. 先建立流程分析习惯（侦探思维）
2. 掌握时序+漏斗+矩阵三大分析
3. 再学工具匹配分析需求
4. 在工作中用数据汇报拉开差距

## 文章正文

这期「学完就涨薪的分析秘技」直播，先讲分析框架再点到 Excel、SQL、Tableau、Python 四门工具应学到什么程度。

**一切分析的基础是流程分析**——把自己当**侦探**。销售额下滑、竞品做得更好，第一件事是研究**如何发生**：双 11 下滑就要往前倒——定目标、策划、准备、执行各步发生了什么。不熟悉业务就不可能有结论；「连对方喜欢什么都不知道，怎么吸引」——恋爱类比。有天赋或经验者可速成，多数人靠积累。

掌握流程分析后，**三个方向足够月薪上万**：时序分析、行为漏斗、用户分析（外加矩阵等交叉）。

**时序分析**：所有同比、环比、基于时间的预测均属此类。**核心目的：在时间上找有效参照物**——现在比以前好就总结为何好，差就总结为何差。只给一天数据无法判断好坏。预测也是为了有对比、说服业务执行策略。做同环比要想清目的，别乱看。

**行为漏斗**：AARRR 等增长模型本质是梳理用户在各时间点做了什么——注册、激活、登录、领礼包、体验、活跃、付款、推荐，每业务不同。**做法**：先拆漏斗 → 再看各环节与历史比何时好坏 → 「60% 基本盘」就清楚。能为业务提价值的分析，最终都源于**引导用户行为**。

**用户分析**：RFM 很基础，正式互联网场景用得少；需更新到天猫八大人群、科勒 5A 人群、字节云图等分层打法——用户「可爱、多变、丰富」，反复只讲 RFM 说明业务经验有限。卖水果要看小区人群：干部多卖礼盒，小孩多卖当季新鲜，打工者多卖便宜量贩——**画像在中间，需求在左，方案在右**。

**工具层要点**：SQL 学到能规范取数、注意写法即可涨薪；Tableau 看板要有**分析逻辑**，不是业务说啥图就堆啥图；Python 可封装成业务直接用的界面（如 POI 选址：输入迪士尼、酒店类型，一键抓上海含关键词的酒店），不必永远 Jupyter——**自动化是定时跑、全封装**；若 Pandas 只是导表给别人，未形成生产力。Excel 连接数据源更新、代码规范等细节在直播中展开。

**学习路径**：先流程分析（侦探思维）→ 时序 + 漏斗 + 用户/矩阵 → 再匹配工具。**思维先于工具**；有分析思维后，工具是执行手段——「月薪上万轻轻松松」指的是思维到位后的可达区间，非空喊口号。

### 矩阵分析与工具层补充

**矩阵分析**：二维交叉（如用户×产品、渠道×时段）找优化杠杆——哪象限该加资源、哪象限该止损。SQL 层强调代码规范、可维护写法，直接关系到协作与涨薪；Tableau 看板须自带分析叙事，而非堆图。Python 可封装 POI 选址等**数据应用**（输入关键词→自动抓酒店列表→落盘），让业务无需配环境；真·自动化是定时任务、全流程封装，而非 Jupyter 导 CSV 人工转发。
