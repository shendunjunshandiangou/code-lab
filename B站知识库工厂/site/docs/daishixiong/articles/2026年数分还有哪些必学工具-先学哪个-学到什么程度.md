> **摘要**（框架级，侧重可视化/看板相关）：大厂日程以会议为主；学工具应学 **联动与工程思维** 而非单工具穷举。数据链路四步：获取→清洗→存储→应用（BI 看板）。

## 大厂工作与看板

- 日程排满：分析报告、**看板 demo 确认**、看板需求、取数 review、数据对齐等。
- Excel 在大厂多用于格式微调；日常协作用 **飞书/钉钉在线表**。

## 工程思维：四步链路

任意数据技术需求可拆为：

1. **数据获取**：RPA / 后羿 / Python 等
2. **数据清洗**：Excel / Prep / Power Query / Python
3. **数据存储**：Excel 文件夹 / MySQL / 数仓——**存储位置决定清洗方式**，常把存储前置考虑
4. **数据应用**：**BI 可视化看板**

示例（电商后台→看板）：RPA 一条龙获取最快 → Prep 清洗 → 公司有什么库用什么 → Tableau/FineBI 出看板。

## 2026 BI 选型（戴师兄观点）

| 优先级 | 工具 | 理由 |
| --- | --- | --- |
| 首选 | **Tableau** | 快速出结果 |
| 其次 | **FineBI** | 免费、可局域网部署 |
| 不推荐初学 | **Power BI** | 难用、协作麻烦；Power Query 无数据输出，数据治理难（医药公司常把 PBI 当数仓） |

**核心**：公司用什么学什么；自学首选 Tableau 快速出作品集。

## 与 2023 版差异

2023 视频推 Power BI 首选；2026 直播更强调 **工程链路与 Tableau 快速出结果**，反映不同场景（求职作品集 vs 企业微软生态）的选型差异。

## 其他

- 单工具永远学不完；应掌握 **工具协作** 与最低成本链路串接。
- 课程含看板项目（UP 主看板可换抖音直播监控看板等）。
