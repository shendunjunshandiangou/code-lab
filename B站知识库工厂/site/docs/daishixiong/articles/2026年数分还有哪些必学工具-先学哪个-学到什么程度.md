> **摘要**（框架级，侧重可视化/看板相关）：大厂日程以会议为主；学工具应学 **联动与工程思维** 而非单工具穷举。数据链路四步：获取→清洗→存储→应用（BI 看板）。

## 大厂工作与看板

- 日程排满：分析报告、**看板 demo 确认**、看板需求、取数 review、数据对齐等。
- Excel 在大厂多用于格式微调；日常协作用 **飞书/钉钉在线表**。

## 工程思维：四步链路

任意数据技术需求可拆为：

1. **数据获取**：RPA / 后羿 / Python 等
2. **数据清洗**：Excel / Prep / Power Query / Python
3. **数据存储**：Excel 文件夹 / MySQL / 数仓——**存储位置决定清洗方式**，常把存储前置考虑
4. **数据应用**：**BI 可视化看板**

示例（电商后台→看板）：RPA 一条龙获取最快 → Prep 清洗 → 公司有什么库用什么 → Tableau/FineBI 出看板。

## 2026 BI 选型（戴师兄观点）

| 优先级 | 工具 | 理由 |
| --- | --- | --- |
| 首选 | **Tableau** | 快速出结果 |
| 其次 | **FineBI** | 免费、可局域网部署 |
| 不推荐初学 | **Power BI** | 难用、协作麻烦；Power Query 无数据输出，数据治理难（医药公司常把 PBI 当数仓） |

**核心**：公司用什么学什么；自学首选 Tableau 快速出作品集。

## 与 2023 版差异

2023 视频推 Power BI 首选；2026 直播更强调 **工程链路与 Tableau 快速出结果**，反映不同场景（求职作品集 vs 企业微软生态）的选型差异。

## 其他

- 单工具永远学不完；应掌握 **工具协作** 与最低成本链路串接。
- 课程含看板项目（UP 主看板可换抖音直播监控看板等）。

## 文章正文

局长（戴师兄团队）这期从**同事真实飞书日历**切入：大厂一个普通下午的日程排满——分析报告、看板 demo 确认、项目进度梳理、内审自查、取数 review、竞品研究、部门培训、数据对齐、跨部门周会、洗数据、看板新增需求、检查周报数据等。除非极闲部门，**大部分人都是这个节奏**；看不到领导很正常，「领导根本就不管你的」。想进大厂，应对这种忙有心理预期；中场略轻，但不可能轻松。

日程里**会议居多**——部门大、协作多，被内外部拉会，自己的活只能堆到晚上，这是大厂加班的重要原因之一。技巧：两个设备同时参会；一旦会议突然安静，很可能在 call 你；可话术「不好意思刚在另一个会上，能否重复一下问题」。精力应集中在**本部门、高优先级**的会，垃圾会要学会拒绝。

**用得最多的「工具」首先是脑子**——双周会、需求评审、项目同步、被私下约的半小时「私会」等，都要带脑子。不带脑子只会 OK，需求锅全甩你头上，且有会议纪要、视频为证，事后反悔显得不靠谱。信息差即话语权：领导知道更多，反应更快；分析师的价值之一就是**别人取不到的数你能取到、别人做不成的分析你能做成**。第二重要的是**文档**——周报、分析报告、BRD（业务需求：背景、目标、demo 样式、预期效果，写不清就驳回）、PRD（产品方案：数据能否搭、维护方、更新频率、容灾等）。理想协作：业务提 BRD → 评审通过 → 产品写 PRD → 分析师按 PRD 落地。

周报万能框架：**报表 + 概览 + To-do 进展**，口诀「向上向下、对内对外、对平行」。向上给管理层核心图表，不能太复杂；大领导旁听时**概览最重要**，团队内部周会则 to-do 进展最重要——「要知道这场会议主角是谁」。40%～50% 分析灵感来自业务主动发现异常，分析师很难监控所有指标。

**Excel 在大厂很简单**——会 VLOOKUP、调格式 often 够用；日常协作用**飞书/钉钉在线表**，不是单机 Excel。SQL 做取数开发，BI 做看板。

**2026 该学什么？** 单工具永远学不完：Excel、RPA、Power Query、Power BI、MySQL、Hive、SQL、Tableau、Prep、Builder、FineBI、Python 各包……必须建立**工程思维**：任意数据需求拆成四步——**数据获取 → 数据清洗 → 数据存储 → 数据应用（BI 看板）**。例：电商后台数据上墙——RPA/后羿/Python 获取 → Prep/PQ/Python 清洗 → 公司有什么库用什么（存储位置决定清洗方式）→ Tableau/FineBI 出看板。

**BI 选型（2026 观点）**：自学求职作品集首选 **Tableau**（快速出结果）；企业有微软生态再学 Power BI；**FineBI** 免费、可局域网部署也推荐。不推荐初学 Power BI——难用、协作麻烦，Power Query 无数据输出、数据治理难（医药公司常把 PBI 当数仓）。与 2023 版「PBI 首选」差异来自场景：求职作品集 vs 企业生态。**核心原则：公司用什么学什么**；掌握工具联动比单点精通更重要。课程含看板项目，UP 主看板可换成抖音直播监控等场景。

### 工程链路细节与 Power BI 为何不首推

四步链路中间的关键：**把「存储」放在清洗之前想清楚**——RPA 爬下来先存哪，才决定怎么洗。存 Excel 文件夹 → Prep 清洗输出新文件 → BI 读文件夹；存 MySQL → Prep 洗完写入库 → BI 连库。Power Query 无数据输出，严格意义上不算完整 ETL，数据只能留在 PBI 数据集里，导致「把 Power BI 当数仓」、建模混乱、治理难——医药公司常见此坑。FineBI 优势是免费、可局域网部署；Tableau 优势是快速出作品集。

### 现场演示：直播间每秒在线人数折线图

需求拆解：获取（影刀 RPA 打开网页→捕获右上角在线人数元素→写入自带表格）→ 默认 Excel 存储 → 清洗可省略 → Tableau 连表出折线图。难点在「每秒爬一次」的频率控制。局长强调：脑子里能拆出四步，就知道该补哪块技能；沉默的大多数往往完全没有思路——这正是工程思维要练的。
