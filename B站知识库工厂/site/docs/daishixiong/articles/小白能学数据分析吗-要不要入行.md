# 小白能学数据分析吗？要不要入行？

<BiliPlayer :bvid="'BV1R3QoYpEH3'" />


> 摘要：一场 92 分钟的系列直播（第二讲），围绕小白入行的五大难点——学什么、学不会、学了忘、没人要、用不上——逐个给解法；核心干货是按公司规模讲透数据链路（从纯 Excel 到数仓分层 + OLAP），从底层解释为什么要学 Excel/SQL/BI、为什么越大的厂越不需要 Python；后半场避雷天坑岗位（付费内推、数据标注、数据录入、专职爬虫）并答疑。本文为框架级重写。

## 一、入行五大难点总览

小白（零基础或转行者）入行必然依次撞上五个问题：

1. **学什么**——不知道该学哪些东西
2. **学不会**——觉得太难、学不进去
3. **学了忘**——学完记不住
4. **没人要**——简历没响应、写不出来、面试答不上
5. **用不上**——已上岸，但学的东西工作里用不到（幸福的烦恼）

## 二、学什么：用数据链路推导必学工具

必学的内容有且仅有一个：**SQL**。推荐学习路线：先 Excel → BI → SQL → Python（SQL 和 BI 顺序可换），先学会一个，再用已知的去理解新的、融入自己的学习体系。

为什么是这几样？把不同规模公司的数据链路画出来就明白了：

| 公司规模 | 数据链路 | 对应工具 |
| --- | --- | --- |
| 无数据建设 | 平台/后台下载数据 → Excel/WPS 处理 → Excel/PPT 向老板汇报 | Excel 是基础中的基础 |
| 小厂 | 前端数据写入数据库（OLTP），清洗、取数都在数据库里；可能没权限、找开发帮忙取数；或用 ERP 系统 | SQL |
| 中小厂 | 业务库要支撑前端网页（每次查询背后都是一段 SQL），取数会拖垮它，于是同步到另一个库专门查数（解耦）；固定取数塞进 BI 看板自动化呈现，协作也在 BI 上避免本地 Excel 版本不一致 | SQL + BI |
| 中型/大厂 | 前端数据 → OLTP 库 → ETL 同步到**数仓**（数据量大、要求准确性稳定性，做**数仓分层**管理，像仓库管理员把表按规范摆好）→ 加工好的数据再同步到 **OLAP 库**（擅长查询）→ 支撑取数和 BI 应用 | SQL + BI，分工明确（ETL 工程师、前后端开发、数仓、数分各司其职） |

几个推论：

- **Excel 必学**：SQL 取出来的数总要有承接，处理和汇报离不开它，且免费课就能掌握到够用。
- **BI 必学**：大厂汇报标配；把 SQL 塞进看板即可自动完成每天固定的取数并自动呈现，还便于观测异常值和分布。
- **Python 不是必学**：越大的厂越不需要 Python（从数据分析角度）——数据获取大厂不用你做，清洗和存储 SQL 搞定，应用 BI 看板搞定。
- 任何数据技术问题都可以从**四个环节**回答：数据的获取、清洗、存储、应用。
- 越小、数据建设越差的公司，数据分析师承担的职能越多（大厂分给多个岗位的活，小公司只有你一个数据人员）。老板懂的话可能还要求你会点爬虫。

### 用 JD 验证

现场拆了拼多多的社招 JD：数据监控预警 = BI 的活；定期数据报告 = 分析报告；业务数据模型 = 指标体系；配合算法迭代 = 分工协作。岗位要求里 BI、SQL（Hive 本质就是 SQL）、AB 测试和分析框架全都能对上，Python 大概率是为了和算法同学对接。没写 Excel 是因为 SQL 取数默认要用 Excel 承接。

**校招 JD 的差异**：校招对业务要求更低，但会考数理统计、线性代数等理论和八股文——因为校招生没有产出可聊，只能用专业、笔试（SQL、概率论、AB 测试）筛人；有两段大厂实习的人则会被围绕实习经历提问。社招不考八股，因为有工作经历和项目可聊。

## 三、学不会：费曼学习法，关键在输出

**有时间的情况**：用费曼学习法——定计划、找资料、快速学（输入），核心在**输出**。两个检验标准：

1. **做出作品**：做不出作品 = 没学会。作品带来正反馈（"我学了之后立马能做出来，太爽了"），没有正反馈很难坚持，也不会得出"我喜欢数据分析"的结论。
2. **讲得出来**：脑子里清楚但讲不出来 = 没融入自己的知识体系。典型例子：问"指标异动飙升怎么排查"，很多学员只能蹦出"业务流程""向前看向下钻"这几个词——这是**知识的诅咒**，你默认面试官也懂这套口诀，但你无法用自己的话把它解释明白。检验方式：开口讲、录下来听，或直接用面试实战检验。

**没时间的情况**：快速熟悉一个行业/业务的三步法（来自冯唐成事心法，也是咨询公司的经典面试题）：

1. **找 100 个关键词**搞懂它（AI 生成、自己搜都行，不懂的马上搞懂）；
2. **找行业专家聊 1~2 小时**（想去哪个大厂就找那条业务线的人），一两天就能达到与业内人交流无障碍的水平；
3. **补专业知识**：找一本专业书或一门专业课，花一到两周学会——这一步逃不掉。

顺带讲了点线面体：从点连成线、再到面、再到体（初级 → 中高级 → 行业专家）。成为专家没有捷径，需要在喜欢的领域深耕八到十年，标志是具备**从零到一**的能力（比如"把一个账号做到十万粉"这种没有框架可依的事），面试可以吹，真做是另一回事。

关于学历低：大专以下确实很难，但不是绝对——有厨师出身、大专以下学历做到数据总监的真实样本（"不货哥"（存疑））。人生是概率游戏，要尊重市场：简历投出去有响应就不要自我设限。

## 四、学了忘：要有忘的觉悟

- 学东西一定会忘，就像家里整理完一定会乱。大脑像电脑系统（类比 Windows 任务计划程序每天自动运行清理任务），会把当前用不上的东西丢进"回收站"清理内存。
- 现在用不上而忘掉没关系，要用时能从回收站里拉出来——只要学过，回忆起来很快。
- 真不想忘只有两条路：**赶紧上岸每天在工作里用**，或**每天复习保持学习状态**。学一天休六天还问为什么忘，是态度问题。
- 好习惯附赠：写代码要写注释、变量别命名 ABCDEFG，否则自己的代码跟天书一样。

## 五、没人要：留到简历面试专场

简历写不满、投出去已读不回、面试磕磕碰碰——这是后两讲（简历专场、面试专场 + 大厂同事模拟面试）要重点解决的内容，本场不展开。

答疑中相关要点：
- 找不到实习先**诊断原因**（投递→已读不回→追问反馈），定位不出问题就盲目试措施，不是合格分析师的做法。
- 五家没过不代表什么，招聘是赛马、优中选优的概率游戏，只能尽量提高自己的概率。
- 简历行业对口的包装：HR 先看**公司名**（不知名公司可括号注明类型）、**项目名称**（"风险管控"想投电商会被秒 pass，做成本方向的可改写为"成本经营分析"）、工作内容逐条向目标岗位靠——要么往行业靠，要么往细分岗位（经营分析/业务分析/商业分析）靠。
- 作品展示什么取决于简历想表达什么：展示分析能力放报告（指标体系、异动、归因、AB 测试），展示技术放数仓分层、ETL，展示 SQL 放代码。

## 六、用不上：幸福的烦恼

学的东西全用上才可怕——意味着所有工作都堆给你。用不上就先放回收站，要用时现查现捡即可。

## 七、天坑岗位避雷

1. **付费内推 / 上缴工资骗局**：钓鱼实测有中介声称多家大厂"两轮面试全部包进、一周走完流程"，代价是上缴工资的 60%~80%。前不久某大厂 OD 裁员裁的就是这批人。凡是让你掏钱找工作的都是诈骗，"保证入职"的合同没有任何法律效力；追问具体业务线，对方直接删好友。
2. **数据专员 / 数据运营：可以去**。转行一步到位很难，拆成两步就丝滑：土木转电商数分，可以"先数分再转电商"（先去传统行业做测算等数据工作）或"先电商运营再转数分"。**只要能接触到数据，就有操作空间**。
3. **专职数据标注：坚决不去**。每天打几十万张图的标，一年下来毫无积累；大厂标注基本都外包，专职岗位没有任何价值。（工作中零星打标逃不掉，但不能是主要内容——判断标准是看岗位考核指标：是标注量还是背 GMV/KPI。）
4. **数据录入 / 校对 / 文员：不要主做**。每天把手写表录进电脑、逐表核对，积累不到东西。
5. **专职爬虫：不要去，容易坐牢**。爬虫的合法边界：不破解、不商用、不高频（像正常访问一样的频率，等同复制粘贴）就没有风险；出事的都是**破解加密**（现场演示了 B 站接口的 32 位 MD5 加密参数如何被逆向复现）、绕过前端抓包批量请求（一秒请求十次，直接消耗对方服务器资源）、把数据或爬虫程序拿去卖。影刀这类模拟人工翻页的 RPA 方式和抓包逆向截然不同。专职爬虫岗一定会涉及逆向，不建议做。
6. **软考等证书**：对数据分析求职没用，团队里没有任何人有证。

## 八、答疑要点摘录

- **实习信息差**：实习做了可以不写在简历上（履历不好、不对口就不写）；入职后一个月内、公司还没交社保时可以跑路（不留流水记录），一旦交了社保就要待住，尽可能待满一年到一年半，别暴露不稳定性（有些公司"五年三跳"是红线）。
- **暑期实习**：三四五月是投递最高峰，现在投、现在面、暑假去；实习必须是应届身份。
- **降薪转行（倒挂）**：在职就再找找，不建议主动降薪；实在没办法可以接受倒挂入行——代价是下一跳的 base 基数变低（跳槽涨幅一般 20%~30%），但也能讲出"我对这个行业爱得深沉"的职场故事。
- **专业优势**：有没有优势看和谁比——商业世界全是博弈，和同批投递者对比才有意义（这也是分析师思维：数据好不好一定要有对照）。
- **薪资**：各城市、去不去大厂标准不同；每次跳槽涨幅约 20%~30%。
- **BI 工程师三个方向**：做 BI 看板、做 BI 平台部署运营、偏前端开发做数据展示；前两者数分技能可以覆盖，第三种偏开发做不了。
- **做过数据标注且交了社保**：只能接受事实，包装成"因表现出色被拉进项目组做项目"，但背景、协作、产出必须提前想好。
