> **摘要**：用「芝麻开门」类比解释 SQL 与数据库的关系，提炼 SQL 的「一大能力 + 两大属性」，并介绍七大核心语句与书写顺序 vs 执行顺序的差异。

## 一、SQL 是什么

SQL 是你对数据库说的「芝麻开门/芝麻关门」——用标准化语言明确告诉数据库要做什么。

主流数据库几乎都支持 SQL 查询（需有权限）。

## 二、一大能力 + 两大属性

记住这三个点即可彻底搞清楚 SQL：

### 一大能力：对数据库发号施令

- 创建数据库
- 查询、优化数据
- 与数据库对话的标准语言

### 两大属性

#### 1. 结构化

**含义一：只处理结构化数据**

- 数据库里的数据都是行列规整的
- 无法处理合并单元格、图片等非结构化 Excel 表格
- SQL 改数据是**整行整列**操作，不能单独改某一个单元格

**含义二：语法有严格结构和运行顺序**

- 七大核心语句：`SELECT / FROM / WHERE / GROUP BY / HAVING / ORDER BY / LIMIT`
- 各数据库用法一致，**顺序不能错**（如 LIMIT 不能写在 SELECT 前面）
- SQL 是**单向执行**语言——运行到后面不会回头执行前面

#### 2. 标准化

- 语法标准由 ANSI 统一制定
- 主流数据库都与 ANSI 标准兼容
- 学会 SQL 就像学会「数据库界的英语」，走到哪都能用

## 三、书写顺序 vs 执行顺序

| 书写顺序（人写） | 执行顺序（数据库跑） |
| --- | --- |
| SELECT → FROM → WHERE → ... | FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT |

**为什么书写顺序把 SELECT 放前面？** 以终为始——先想清楚要什么字段，再想从哪张表取。数据库则必须先知道表在哪。

## 四、复杂 SQL 的优化提示

见过 400–500 行的 SQL。超过这个长度应优化：

- 用临时表拆分子查询
- 避免反复调用同一子查询
- 否则查询太慢
