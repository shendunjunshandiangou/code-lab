# 第四章 · 数据可视化与报表

> 你在整个学习路径中的位置：第二章你学会了用 Excel 处理和汇总数据，第三章你学会了用 SQL 从数据库里把数取出来。到这里，你已经能"拿到数、算出数"了。但数据分析的最后一步，是**把数据讲给别人听**——老板不会去看你的 SQL 代码，业务方也没耐心翻你的几万行明细表。本章解决的就是这个问题：怎么把算好的数变成一眼能看懂的图表和看板。同时，本章也顺带回答一个前置问题——图表的原料（数据）如果不在公司数据库里、而在网页上，该怎么合法地采回来。
>
> 本章会带着你**从零安装 Tableau 到画出五张基础图**，每一步都对应原视频里的鼠标操作顺序；爬虫部分则补全无代码采集器的完整操作流程。

---

## 4.1 先校准认知：AI 时代，可视化技能还值得学吗

在动手学工具之前，先回答一个很多转行同学都会犹豫的问题：都 2026 年了，AI 都能自动画图了，我还有必要学 BI 工具吗？

戴师兄把技能按 AI 时代的命运分成了四层金字塔（详见第一章），其中和本章直接相关的是最底层的**淘汰区**和第二层的**重塑区**：

### 重塑区：Excel 和 BI 看板不会死，但用法会变

**生活化类比**：汽车普及之后，马车夫消失了，但"把人从 A 送到 B"这个需求没消失——它变成了司机这个职业。工具会换，需求恒在。

**技术定义**：所谓[重塑区技能](/daishixiong/atoms/数据分析-工具技能-重塑区技能清单.html)，指 Excel、BI 看板这类不会被淘汰、但使用方式会被 AI 重构的技能——从"手动操作"变为"指挥 AI 完成"。

具体来说：

- **Excel** 是全世界业务的通用数据载体。你的上游同事、你的老板、大家电脑里存的，很长一段时间内都还会是 Excel。变化在于你会指挥 AI 去做，重构工具使用习惯。
- **BI 看板**（BI 即 Business Intelligence，商业智能，指 Tableau、Power BI 这类拖拽出图、组装数据看板的工具）：未来可能不再手动用 BI 工具做，而是直接让 AI 生成一个 HTML 格式的看板再让它改。但"看图看表"的需求还在，企业还是需要人去看、去解读。

**对你的意义**：如果你学会了 BI 和 Excel，再更新一下技能（学会用 AI 驱动这些工具），你依然能做，企业依然需要。分析师在企业管理中的位置——掌握信息就掌握决策权、解释权和监督权——AI 能写 SQL、出图、写报告，但替代不了这个结构性角色（详见第一章 1.2）。

### 淘汰区：纯执行的机械操作会被替代

与之对照的是[淘汰区技能](/daishixiong/atoms/数据分析-工具技能-淘汰区技能清单.html)：手写 SQL、手写 Python 代码、死记函数语法、人肉取数做表——这些**纯执行动作** AI 写得比你快多了。

**你可能会问**：那我第二、三章学的 Excel 和 SQL 不是白学了？——不是。淘汰的不是 SQL 和 Python 本身，而是"只靠机械执行、不知道为什么要取这个数、不知道口径"的工作方式。你要成为那个知道"为什么画这张图、给谁看、看什么口径"的人，工具执行层交给 AI 加速。这正是本章的学习姿势：理解可视化的逻辑，同时**亲手走通一遍操作**，以后指挥 AI 时你才知道它在干什么、哪里会出错。

## 4.2 Tableau 入门：从安装到画出五张基础图

Tableau 是最主流的 BI 可视化工具之一，特点是**拖拽出图**——不写代码，把字段拖到不同位置，图就出来了。原视频称柱状图、条形图、饼图、折线图、散点图为可视化领域的"四大天王"（本课实际演示五张图）。

### 4.2.1 下载、安装与激活

**生活化类比**：Tableau 的密钥管理有点像视频网站的会员账号——一个账号最多同时登两台设备，想在新电脑上用，得先把旧设备踢下线。

**技术定义**：Tableau 从官网下载安装后可免费试用 14 天；正式使用需产品密钥激活；在读学生可申请一年免费密钥。详见 [数据分析-Tableau-下载安装与密钥管理](/daishixiong/atoms/数据分析-tableau-下载安装与密钥管理.html)。

**最小可运行步骤（跟着做）**：

**第 1 步：下载**

1. 从视频下方 Tableau 下载网址进入版本列表，选择**最新版本**；
2. 滚动到页面底部，按 Win / Mac 对应链接下载；
3. **注意**：大陆站点不支持 Win32，需走**台湾站点**或独立老版本链接；
4. 下载慢时，复制链接到**迅雷**加速。

**第 2 步：注册账户**

1. 下载页右上角「登录」→「创建账户」；
2. 填写姓名、邮箱、密码；
3. 邮箱收到激活邮件，点击链接完成激活。

**第 3 步：安装**

1. 双击安装程序；
2. **勾选默认两项，不要自定义安装路径**（自定义易出现路径问题）；
3. 系统盘需预留 **2–3 GB** 空间；
4. 安装完成后 Tableau 通常自动打开；首次打开点「立即开始试用」，输入登录信息即可（每台电脑 **14 天试用**）。

**第 4 步：密钥激活（试用结束后）**

1. 菜单栏「帮助 → 管理产品密钥 → 激活 → 使用产品密钥激活」；
2. 粘贴产品密钥；
3. **每个有效密钥最多激活两台电脑**，超出可能被追责；换电脑前需先在旧设备上停用。

**第 5 步：学生免费（在读生）**

1. 凭学生证/学历证明申请**一年免费**激活码；
2. 填写信息必须与证明文件**完全一致**（姓名、学校、在读期限）；
3. 提交后等待 **1–3 个工作日**审核；成功则邮件发送密钥，失败可按要求重新上传。

**你可能会问**：试用期过了、又不是学生怎么办？——原子笔记中只提到试用与学生两条正式渠道，其余方式原视频未展开（原文此处不清晰），建议以官网政策为准。

### 4.2.2 连接数据：实时 vs 提取

**生活化类比**：「实时连接」像每次做饭都现跑一趟菜市场；「数据提取」像先把一周的菜买回来放冰箱——做饭（分析）的时候直接从冰箱拿，快得多。

**技术定义**：Tableau 可连接 Excel/CSV 文件或数据库；[「数据提取」模式](/daishixiong/atoms/数据分析-tableau-数据连接与提取模式.html)会把源数据预加载为 `.hyper` 格式文件，大数据量下比「实时」连接快，一般推荐使用。

**最小可运行步骤（用疫情数据练习）**：

原视频使用 Tableau Public 社区下载的「全中国各省疫情数据」Excel 做演示。你可以从视频下方链接获取同款练习数据。

**第 1 步：选择连接类型**

打开 Tableau 后，左侧「连接到文件」：

| 文件类型 | 选什么 |
| --- | --- |
| `.xlsx` / `.xls` | Microsoft Excel |
| `.csv` | 文本文件 |
| 数据库 | 连接到服务器（SQL 直连等，入门阶段可先跳过） |

**第 2 步：选择连接模式**

拖入数据后，右下角二选一：

| 模式 | 行为 | 推荐场景 |
| --- | --- | --- |
| 实时 | 每次操作直接读取源文件/数据库 | 数据量小、源数据频繁变动 |
| 数据提取 | Tableau 预加载为 `.hyper` 副本，处理更快 | **一般推荐**，尤其数据量大时 |

**第 3 步：确认字段**

连接成功后，左侧出现字段列表：

- **蓝色** = 维度（分类字段：省份、日期、渠道……）
- **绿色** = 度量（数值字段：新增确诊、销售额、人数……）

右侧默认只显示前 **1000 行**预览，可改大数字查看总行数。

**第 4 步：刷新数据**

提取模式下，如果后来改了 Excel 原文件，Tableau 里的图不会自动变——点数据源页的「**刷新**」重新提取即可。

**你可能会问**：提取之后我改了 Excel 原文件，Tableau 里的图为什么没变？——因为提取模式读的是预加载的副本，不是原文件。记得点「刷新」。

### 4.2.3 工作簿格式与三种工作区

**技术定义**：Tableau 的[工作簿有两种保存格式](/daishixiong/atoms/数据分析-tableau-工作簿格式与工作区类型.html)：

| 格式 | 特点 | 什么时候用 |
| --- | --- | --- |
| `.twb` | 仅保存图表与连接配置，**不含数据**；文件小、便于保密 | 工作中涉及敏感数据 |
| `.twbx` | 数据打包进工作簿；便于练习、导出、离线操作 | **学习练习推荐** |

界面底部有三种工作区图标，层级关系像「单页 → 拼版 → 演示」：

1. **工作表**（Sheet）：制作单一图表的地方——所有图都先在这里画好；
2. **仪表盘**（Dashboard）：把多个工作表拼在一起展示；**不能在仪表盘内新建图表**，只能拖入已画好的工作表；
3. **故事**（Story）：类似 PPT，把工作表和仪表盘组合成有顺序的交互展示，适合项目汇报。

**操作提示**：第一次保存时选 `.twbx`，换台电脑打开也有数据，不用重新连源文件。

### 4.2.4 五大基础图表快速上手

**生活化类比**：Tableau 画图就像玩冰箱贴——把「维度」（分类字段，如省份、日期）和「度量」（数值字段，如确诊数、销售额）这两类磁贴，贴到「行」「列」「颜色」「大小」这些不同的架子上，图自动就变出来了。

**技术定义**：[拖拽维度与度量到行列、颜色、大小等「架子」即可出图](/daishixiong/atoms/数据分析-tableau-四大基础图表快速上手.html)；可视化领域最常用的是柱状图、条形图、饼图、折线图、散点图。

练习数据字段说明：省份（维度）、新增确诊/新增治愈/新增死亡/累计确诊（度量）、日期（维度，需改类型）。

---

#### 图表 1：柱状图（手把手 6 步）

1. 新建工作表，把「省份」拖到 **列**（Columns）；
2. 把「新增确诊」拖到 **行**（Rows）——柱状图自动出现；
3. 点击工具栏**排序按钮**（降序），一眼看出哪个省最高；
4. **排除离群值**：湖北确诊数远高于其他省份，右键图中「湖北」→ 选「**排除**」（或拖入筛选器取消勾选），其余省份对比才看得清；
5. 把「新增确诊」拖到「标记」卡的 **标签**（数值常显在柱顶）；或拖到 **工具提示**（悬停才显示）；
6. 菜单「视图」→ 选「**适合宽度**」，让所有柱子显示在一屏内。

---

#### 图表 2：条形图（手把手 5 步）

柱状图和条形图的区别就是**行列互换**：

1. 新建工作表（或清空当前工作表）；
2. 「省份」→ **行**，「新增确诊」→ **列**；
3. 排序（降序），同样排除湖北；
4. **全局筛选技巧**：在筛选器上右键 → 「**应用到此数据源的所有项**」——之后所有用这个数据源的工作表都自动排除湖北，不用每张图重复操作；
5. 也可以直接点工具栏的「**转置**」按钮，一键把柱状图和条形图互换——建议自己点一下，思考背后原理。

---

#### 图表 3：饼图（手把手 7 步）

1. 「新增确诊」→ **大小**（Size）；
2. 「省份」→ **颜色**（Color）；
3. 「标记」卡左下角显示形式下拉框 → 选「**饼图**」；
4. 菜单「视图」→ 选「**整个视图**」，饼图放大铺满（原视频调侃：饼一定要又大又圆）；
5. 按住 **Ctrl** 拖拽「省份」到 **标签**（复制字段，不用从左侧重新拖）；
6. 同样 Ctrl+拖拽「新增确诊」到标签，扇区旁显示数值；
7. 饼图也支持升序/降序排序。

**课后思考 1**（原视频布置）：饼图能否用「累计确诊」替代「新增确诊」做占比？

**参考答案提示**（观察数据结构得出，非虚构）：各省「累计确诊」是随时间单调递增的存量，各扇区之和没有"占比"的统计意义；而「新增确诊」是同一统计周期内的增量，各扇区之和才有整体概念，所以占比图用新增而非累计。你可以打开数据亲自验证。

---

#### 图表 4：折线图（手把手 5 步 + 粒度变换）

折线图必须基于**时间**字段：

1. 找到「日期」字段——左侧图标显示为 **Abc**（文本）时，先点击图标改为 **日期** 类型；
2. 「日期」→ **列** → 在弹出的日期层级中选「**精确日期（天）**」；
3. 「新增确诊」→ **行**——最简单的日粒度折线图完成。

**课后思考 2**：如何改成周/月/季度/年粒度？

**操作提示**：拖到列上的日期字段，点击下拉箭头可以切换「年/季度/月/周/日」——类比 Excel 里把日期拖到透视表后选分组粒度。原视频故意放慢这步，就是希望你课后自己摸索。

---

#### 图表 5：散点图（手把手 4 步）

散点图看两个度量之间的分布关系：

1. 「新增死亡」→ **列**；
2. 「新增治愈」→ **行**；
3. 「省份」→ **详细信息**（Detail）；
4. 按住 Ctrl 把「省份」复制到 **标签**——每个点旁边显示省名。

原视频点评：浙江在「低死亡、高治愈」区域表现突出，一眼看出分布差异。

**课后思考 3**：如何让散点图各省份像饼图一样用颜色区分？

**操作提示**：把「省份」再拖到「标记」卡的 **颜色** 架上，每个省一个颜色。自己试一下，比看答案记得牢。

**你可能会问**：维度和度量怎么区分？——粗略地说，回答"是什么/属于哪类"的字段是维度（省份、渠道、日期），回答"有多少"的数值字段是度量（销售额、人数）。Tableau 导入数据后会自动帮你分好，蓝色是维度、绿色是度量。

### 4.2.5 筛选器：排除离群值与全局应用

**生活化类比**：全班身高图里混进一位篮球运动员，其他人的差异全被压扁看不出来了——先把他"请出去"，剩下的人才能比出高矮。

**技术定义**：[Tableau 可通过右键排除离群值](/daishixiong/atoms/数据分析-tableau-筛选器排除与全局应用.html)，或将筛选器「应用到此数据源的所有项」，让同数据源下所有工作表默认继承筛选条件。

**最小可运行步骤**：

**方式 A：单图快速排除**

1. 右键图中的维度成员（如「湖北」）→ 选「**排除**」；
2. 或把维度拖入筛选器架，在弹窗中取消勾选不需要的项。

**方式 B：全局筛选（推荐，避免重复劳动）**

1. 在筛选器上右键 → 「**应用到此数据源的所有项**」；
2. 之后所有用这个数据源的工作表都自动应用同一筛选——做柱状图、条形图、饼图时不用每张图都手动排除湖北。

原视频场景：疫情数据中湖北确诊数远高于其他省份，排除后其余省市的对比才看得清。这个操作在真实业务中极其常见——某个头部渠道/大区数值碾压，不处理就看不到长尾差异。

**你可能会问**：排除掉离群值算不算"篡改数据"？——不算，前提是你在汇报时说清楚"此图已排除湖北，用于对比其余省份"。可视化的目标是让人看清结论，而不是隐藏事实。

### 4.2.6 标签与工具提示：让图会说话

**生活化类比**：标签像商品价签——常驻展示；工具提示像扫码看详情——凑近（悬停）才显示，不占版面。

**技术定义**：[「标记」卡](/daishixiong/atoms/数据分析-tableau-标记卡标签与工具提示.html)可以把度量拖到「标签」（数值常显在柱顶/扇区旁）或「工具提示」（鼠标悬停时才显示）。

**最小可运行步骤**：

**标签（常显数值）**

1. 把度量（如新增确诊）拖到「标记」卡的「**标签**」架；
2. 数值直接显示在柱顶/扇区旁；
3. 右键标签 → 「**旋转标签**」：竖排改横排，密集时更易读。

**工具提示（悬停详情）**

1. 把多个度量（新增确诊、新增治愈、新增死亡）拖到「**工具提示**」架；
2. 鼠标悬停任意柱子/扇区时，弹出显示全部指标——版面干净，细节按需查看。

**三个提效快捷键**（原视频强调）：

| 操作 | 怎么做 |
| --- | --- |
| 一键显示/隐藏标签 | 顶部 **T** 按钮 |
| 复制字段到另一架子 | **Ctrl + 拖拽**字段（不用从左侧重新拖） |
| 旋转标签方向 | 右键标签 → 旋转标签 |

**组合使用示例**：柱状图里「新增确诊」放标签常显，「新增治愈 + 新增死亡」放工具提示——老板看大图看确诊数，鼠标移上去看完整三项指标。

#### 界面区域速查（第一次打开不迷路）

打开 Tableau 连接数据后，记住四个区域的分工：

| 区域 | 位置 | 干什么 |
| --- | --- | --- |
| 数据窗格 | 左侧 | 所有维度（蓝）和度量（绿），拖从这里出发 |
| 功能区（架子） | 顶部中间 | 列、行、筛选器、标记、颜色、大小、标签、工具提示 |
| 视图区 | 中间大画布 | 图表实时渲染的地方 |
| 标记卡 | 功能区下方 | 控制图形类型（自动/饼图/方形等）和标签/颜色细节 |

原视频说"拖拽出图就像玩冰箱贴"——本质就是**把左侧字段贴到顶部不同架子上**，架子组合决定图表类型。第一次练习建议严格按 4.2.4 的顺序做五张图，不要跳步，每张图建一个新工作表，避免字段架子互相干扰。

## 4.3 图表的原料从哪来：爬虫与采集选型

公司内的数据走 SQL 取（第三章），但有时你需要的数据在公开网页上——比如做求职分析要抓招聘网站的岗位数据，或像原视频短教程那样抓 B 站排行榜做练习项目。这就涉及"爬虫"：用程序自动化地批量获取网页数据。对分析师来说，关键不是技术多深，而是**选对工具、守住边界**。

### 4.3.1 选型：别迷信手写代码

**生活化类比**：搬家你会叫搬家公司，而不是自己研究怎么开货车。爬数据同理——现成的采集工具是搬家公司，手写 Python 爬虫是自己考货车驾照，费时且容易翻车。

**技术定义**：[招聘站等场景自写 Python 爬虫常因反爬直接失效](/daishixiong/atoms/数据分析-工具-爬虫选型与反爬现实.html)（反爬即网站为阻止程序批量抓取而设置的防御机制）；分析师应优先用**后羿采集器 / RPA**（RPA 即 Robotic Process Automation，机器人流程自动化，模拟人工点击操作的工具），把精力留给数据清洗与分析。

手写爬虫在真实招聘站上的常见死法：

| 平台 | 典型问题 |
| --- | --- |
| Boss 直聘 | 频繁请求 → **封 IP**（访问行为异常） |
| 拉勾/智联 | **必须登录**才能翻页 |
| 前程无忧等 | **URL 加密**，无法批量构造链接 |

推荐路径：

- **后羿采集器**：字段智能识别、预登录、多站分组
- **影刀 RPA**：模拟人工操作取数
- Python requests：适合学习原理（第八章会讲），**生产环境慎用**

#### 无代码采集器完整操作流程（B 站排行榜示例）

原视频 2 分半短教程演示了零代码爬 B 站排行榜的全流程（功能与后羿采集器一致）：

**第 1 步：安装**

到官网下载采集软件（链接在视频简介），安装并打开。

**第 2 步：智能采集（列表页）**

1. 复制目标页面网址（如 B 站排行榜），粘贴到软件的网址框；
2. 点击「**智能采集**」——软件自动识别网页上所有可爬取的字段；
3. 稍等识别完成，预览区出现表格。

**第 3 步：修改字段名**

有些字段自动命名不准，右键「**修改字段名称**」（例如把某字段改名为「播放量」），点确定。

**第 4 步：深入采集（详情页，可选）**

需要每条记录的详情信息时：

1. 点击「**深入采集**」；
2. 要新增字段时点「**添加字段**」，出现笔状指针；
3. 直接指到想爬的数据上点击，识别后修改字段名（例如「关注数」），确定。

**第 5 步：启动与导出**

1. 点保存 → 开始采集 → 启动，软件自动爬数据；
2. 爬完后点停止（或等自动提示完成）；
3. 点「**导出数据**」，设置导出文件夹，点导出；
4. 点「查看导出的文件」，双击即可用 Excel 打开。

**注意事项**：

- 只爬列表页很快；需逐条点进详情页的深入采集较慢，免费版速度一般；
- 导出后的 Excel/CSV 可直接接入 Tableau（4.2.2 的连接步骤）。

#### 企业级项目全流程

完整的企业爬虫项目不是"粘贴网址就完事"，而是：

1. **需求澄清**：爬什么平台、哪些字段、用途、频率——需求方应说清背景，而非扔一句话"帮我爬一下"；
2. **可行性验证**：先测能否稳定取数（工具优先于手写代码）；
3. **字段梳理**：多数据源字段对齐清单；
4. **分源爬取** → **分源清洗** → **合并去重/对齐**；
5. **交付**：统一宽表 → Excel/数据库 → Tableau 等可视化。

记住原则：**爬虫只是取数手段，分析师的价值在清洗、口径、分析，不在攻防。**

#### 后羿采集器 vs 影刀 RPA：怎么选

| 维度 | 后羿采集器 | 影刀 RPA |
| --- | --- | --- |
| 适合场景 | 网页结构规整、字段可智能识别 | 需要模拟人工点击、翻页、登录 |
| 招聘站 | 支持预登录（智联等） | 模拟人工操作，绕过简单反爬 |
| 学习成本 | 粘贴网址即可开始 | 需录制操作流程 |
| 分析师定位 | 快速验证可行性 | 流程固定后的自动化取数 |

企业级爬虫课的建议是：**先用后羿验证能不能取到**，取不到再考虑 RPA 或提需求换数据源——不要一上来就写 Python。

### 4.3.2 合法边界：三不原则

**生活化类比**：进超市抄价签没问题；撬开仓库门、把货批量搬走去卖，就是另一回事了。

**技术定义**：[爬虫的安全边界是"不破解、不商用、不高频"](/daishixiong/atoms/数据分析-工具-爬虫三不原则合法边界.html)——像正常访问一样的频率抓公开展示的数据，等同复制粘贴，没有风险；出事的都是逆向破解加密、抓包批量请求、卖数据卖程序的。

具体对照：

**安全的**：
- 数据是页面上公开展示的，抓取等同复制粘贴
- 频率像正常用户访问（正常人一秒最多请求一个页面）
- 只自用，不拿数据或程序去卖

**危险的**（真实被抓案例的行为）：
- **破解加密**：逆向 API 加密算法、身份指纹——这是绕过了对方的保护（原视频直播演示过 B 站接口的 32 位 MD5 加密参数如何被断点定位并复现，用于说明"这类行为才是危险的"）
- **抓包批量请求**：绕过前端限制一秒请求十次，直接消耗对方服务器资源（每个请求都是对方的钱）
- **商用获利**：把破解方案或抓来的数据卖给别人，侵害对方核心数据

**你可能会问**：影刀 RPA 模拟人工翻页，算不算爬虫违规？——不算。RPA 是模拟人工操作，与抓包逆向截然不同，可以用。另外提醒一句：**专职爬虫岗位**一定会涉及逆向破解，不建议入行时选这个方向。

## 4.4 从数据到看板：串一次完整链路

把本章内容串成一条可复用的工作流，下次做项目直接照着走：

| 步骤 | 做什么 | 本章对应 |
| --- | --- | --- |
| 1. 取数 | 公司内 SQL/看板；公开数据用采集器 | 第三章 + 4.3 |
| 2. 清洗 | Excel 或 Power Query 整理口径 | 第二章 |
| 3. 连接 | Tableau 选 Excel/CSV，推荐数据提取 | 4.2.2 |
| 4. 探索 | 五大基础图快速看分布、趋势、离群值 | 4.2.4 |
| 5. 精修 | 排除离群、加标签/工具提示、全局筛选 | 4.2.5–4.2.6 |
| 6. 交付 | 仪表盘拼版或故事汇报；敏感数据用 `.twb` | 4.2.3 |

#### 第一次做仪表盘（5 步预览）

原视频本讲只到工作表，但 4.2.3 提到了仪表盘——学完五张基础图后，你可以用同一份疫情数据试一次拼版：

1. 底部点「**新建仪表盘**」；
2. 右侧「工作表」列表里，把已画好的柱状图、折线图**拖进画布**；
3. 拖入时选「**作为筛选器**」可以让点击柱状图某省份时，折线图跟着联动；
4. 调整每个工作表容器大小，对齐排版；
5. 保存为 `.twbx`，双击即可演示。

仪表盘里**不能新建图表**——这是新手最常犯的错误：发现仪表盘里拖字段出不了图，其实得回工作表先画好再拖进来。

**新手三个常见坑**（原视频直播弹幕高频）：

1. **饼图太小**：忘了「视图 → 整个视图」，饼图缩在中间一小块；
2. **折线图日期不对**：忘了先把日期字段图标从 Abc 改成日历图标；
3. **改 Excel 后图没变**：提取模式忘了点「刷新」——不是 Tableau 坏了，是读的是副本。

原视频最后强调：本讲目标是**快速上手**——先会用、先出图，拖拽出图的底层原理和四大图表的业务选型（什么时候用柱状图 vs 折线图），在后续 Tableau 课程里展开。你现在的任务是把五张图各做一遍，三道课后思考题自己想通，比背操作步骤重要得多。
