## 2.1 基础操作与效率

动手之前，先把"手"练顺。这一节解决的是：拿到陌生表格后，如何在 30 秒内摸清数据边界、确认数据类型没坑、用快捷键把重复动作压缩到肌肉记忆。

### 2.1.1 分析师常用快捷键

**生活化类比**：厨师上灶之前先把刀磨快、调料摆顺手。快捷键就是分析师的"摆调料"——单个动作只省一两秒，但你一天要做几百次。

**技术定义**：快捷键是用键盘组合替代鼠标操作的指令。分析师最高频的一批覆盖文件访问、窗口管理、单元格导航、筛选、填充和求和。详见 [数据分析-Excel-分析师常用快捷键](/daishixiong/atoms/数据分析-excel-分析师常用快捷键.html)。

#### 拿到表格的「默认三连」（截图式操作）

假设你刚收到同事发来的 `销售明细.xlsx`，打开后光标在 A1：

1. **按 `Ctrl+Shift+L`**：表头行立刻出现下拉小三角——筛选已开启。这是戴师兄说的"拿到表格默认动作之一"。若再按一次，筛选关闭。
2. **单击 C 列任意有数据的单元格，按 `Ctrl+↓`**：光标瞬间跳到 C 列最后一行有数据的单元格——你立刻知道这张表大约有多少行（看行号即可，比如跳到第 1658 行说明约 1657 条明细 + 1 行表头）。
3. **选中一列数字（如「成交金额」），看 Excel 窗口右下角状态栏**：会显示"平均值、计数、求和"——不用写公式就能粗看这一列的规模。若要正式求和，选中数字区域后按 **`Alt+=`**，Excel 自动在下方或右侧插入 `=SUM(...)` 并回车。

#### Windows 通用快捷键（文件与窗口）

分析师不只在 Excel 里干活，这些 Windows 快捷键同样高频，来自戴师兄《24 个快捷键》课：

| 快捷键 | 功能 | 典型场景 |
| --- | --- | --- |
| `F2` | 重命名文件/文件夹 | 比双击名称再编辑更不容易误触"打开"或"删除" |
| `Ctrl+Z` | 撤销 | 误删文件、误改公式，第一时间撤回 |
| `Win+E` | 打开「此电脑」 | 快速找数据文件 |
| `Win+D` | 显示/恢复桌面 | 老板走过来时一键藏窗口（再按一次恢复） |
| `Win+方向键` | 窗口分屏/贴边 | **左边原报表、右边动手复现**——2024 报表课的标准姿势 |
| `Alt+F4` | 关闭当前窗口 | |
| `Win+Shift+S` | 系统截图到剪贴板 | 做文档、发飞书时用 |
| `Win+V` | 剪贴板历史 | 连续复制多段内容后依次粘贴 |

#### Excel 核心快捷键速查

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl+Shift+方向键` | 从当前位置选中到数据区域末端 |
| `Ctrl+;` / `Ctrl+Shift+;` | 输入当前日期 / 当前时间 |
| `Alt+Enter` | 在同一个单元格内换行（做表头换行很实用） |
| `Ctrl+T` | 将选中区域转为**智能表**（后面透视表、PQ 都会用到） |

#### 填充柄的四个细节

- 选**两个**数（如 1 和 2）往下拖 → **等差数列**；
- 只选**一个**数拖 → **复制**同一数字；
- 按住 **`Ctrl`** 再拖 → **强制复制**（即使看起来像序列）；
- 按住 **`Shift`** 拖动列边界 → **插入列**（把后面的列往右挤），而不是覆盖。

**最小可运行示例**——分析师打开陌生表格的标准流程：

```
打开文件 → Ctrl+Shift+L 开筛选 → Ctrl+↓ 探行数 → 扫一眼状态栏/Alt+= 粗看规模
```

**你可能会问**：24 个快捷键记不住怎么办？——不用背。先把 `Ctrl+Shift+L`、`Ctrl+方向键`、`Alt+=` 这三个用成肌肉记忆，其余的在遇到对应场景时再查、再用，用三次自然就记住了。

#### 练手清单：5 分钟快捷键热身

找一张任意明细表（没有就用课程数据），按顺序完成：

1. `Ctrl+Shift+L` 开关筛选两次，确认表头小三角出现/消失；
2. 在「日期」列按 `Ctrl+↓`，记录最后一行行号；
3. 选中「金额」列，看状态栏求和，再按 `Alt+=` 插入 SUM 公式；
4. `Win+方向键` 把窗口贴左，再开第二个 Excel 窗口贴右，模拟报表对照；
5. 在 A1 输入长标题，用 `Alt+Enter` 在单元格内换行。

全部做完，你就完成了分析师「开表第一眼」的标准动作——后面每一节都会在此基础上叠加。

### 2.1.2 Excel 的日期本质是数字

**生活化类比**：酒店房卡上印的是"888 房"，但门锁系统里存的其实是一串编码。Excel 的日期也一样——你看到的是"2020-05-31"，Excel 内部存的是一个数字。

**技术定义**：Excel 中日期本质是从 1900 年 1 月 1 日开始计数的序号（1900-01-01 = 1），比如 **43982** 就对应 **2020-05-31**。显示成什么样，由"单元格格式"决定。详见 [数据分析-Excel-Excel日期存储与格式](/daishixiong/atoms/数据分析-excel-excel日期存储与格式.html)。

#### 截图式操作：从数字变回日期

2024 报表课里的真实场景——你要在报表标题处显示"最新数据日期"：

1. 在空白单元格（如 C1）输入 `=MAX(A:A)`，假设 A 列是「成交日期」；
2. 回车后单元格可能显示 **43982** 而不是日期——**别慌，公式没错**；
3. **右键该单元格 → 设置单元格格式 →「数字」选项卡 → 分类选「日期」→ 类型选「短日期」→ 确定**；
4. 43982 立刻显示为 `2020/5/31`。

配套提取函数（后面 SUMIF 按月份汇总会用到）：

```
=YEAR(A2)    → 提取年份，如 2020
=MONTH(A2)   → 提取月份，如 5
```

**常见报错与误区**：

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 日期参与计算结果是很大整数 | 单元格格式是「常规」 | 设成短日期或自定义格式 |
| 输入 `2020-05-31` 后无法 SUM | 被存成**文本** | 用「分列」或 DATE 函数转真正日期 |
| 手动在数字前打 `¥` 或日期文字 | 单元格变**文本型** | 应通过「单元格格式」设显示样式 |

**你可能会问**：我想让金额显示"¥"符号，直接在单元格里打个 ¥ 行不行？——不行。手动输入 ¥ 会让单元格变成**文本型**，从此无法参与计算。正确做法：**右键 → 设置单元格格式 → 货币**，或自定义格式 `¥#,##0`——数据保持数字本质，只是"换皮肤"。

### 2.1.3 公式引用方式：相对、绝对与混合引用

**生活化类比**：你跟朋友说"我家楼下的便利店"，你搬家了这句话指的店也跟着变——这是**相对引用**；说"人民路 88 号的便利店"，无论你搬到哪它都不变——这是**绝对引用**；"我家所在那条街的 88 号"则一半跟着变一半锁死——这是**混合引用**。

**技术定义**：Excel 公式在向下、向右填充时，单元格引用默认会跟着偏移（相对引用）；加 `$` 符号可锁定行或列（绝对/混合引用）。详见 [数据分析-Excel-公式引用方式混合引用](/daishixiong/atoms/数据分析-excel-公式引用方式混合引用.html)。

| 引用类型 | 写法 | 向下填充 | 向右填充 |
| --- | --- | --- | --- |
| 相对引用 | `A11` | 行变 | 列变 |
| 绝对引用 | `$A$11` | 不变 | 不变 |
| 混合引用（锁列） | `$A11` | 行变 | 列不变 |
| 混合引用（锁行） | `A$11` | 行不变 | 列变 |

#### 截图式操作：F4 循环切换引用

1. 在编辑栏里**用鼠标选中**公式中的某个引用（如 `E:E`）；
2. **按 F4 键**：每按一次循环切换 `$E:$E` → `E$:$E$` → `$E$:$E$` → `E:E`……
3. 2024 报表课建议：条件区域、求和列、筛选器单元格多用绝对引用；行内变化的省份名用 `$A11`（锁列不锁行）。

**最小可运行示例**——做省份明细表，用 SUMIFS 向下填充每个省的数据时：

- 求和列 `$E:$E`、条件区域 `$A:$A`：整列锁死，用绝对引用；
- 报表顶部的月份筛选器 `$H$4`：一个固定单元格，用绝对引用；
- 当前行的省份 `$B11`：要随着往下填充逐行变化，但往右填充时列不能跑偏，用 **`$B11`**（混合引用，锁列不锁行）。

**常见报错**：

| 现象 | 原因 |
| --- | --- |
| 往下拖公式后，某几行结果突然为 0 | 条件区域没加 `$`，填充后指向了错误列 |
| 向右拖公式后，月份条件跑偏 | 筛选器 `$H$4` 写成了 `H4` |

**你可能会问**：`$` 要一个个手打吗？——不用。选中公式里的引用，按 **F4** 键循环切换四种引用方式。

## 2.2 数据匹配

基础操作顺了，下一关是**跨表拼数据**。数据匹配解决的问题是：手头两张表，想按"工号""订单号"这样的共同字段，把 B 表的信息带到 A 表里来。这是分析师使用频率最高的操作之一——后面的 Power Query 合并查询做的是同一件事，只是规模更大。来源文章 [【Excel教学】数据匹配专题！](/daishixiong/articles/excel教学-数据匹配专题.html) 把函数体系分为三层：VLOOKUP（基础）→ XLOOKUP（灵活）→ INDEX+MATCH（万能保底），并补充了 WPS 与 Excel 的版本兼容说明——公司电脑是什么版本，决定你优先练哪一层。

### 2.2.1 VLOOKUP：最基础的查找函数

**生活化类比**：查字典。你拿着一个字（查找值），到字典（数据区域）里找到它所在的那一页，然后抄下第几行的解释（返回列号）。

**技术定义**：VLOOKUP 是 Excel 最基础的单向查找函数，在数据区域**第一列**中查找目标值，返回同一行中指定列号的值。详见 [数据分析-Excel-VLOOKUP基础与常见陷阱](/daishixiong/atoms/数据分析-excel-vlookup基础与常见陷阱.html)。

#### 截图式操作：第一次写 VLOOKUP

假设 **A 列**是待补全的「订单号」，**F:H 列**是参考表（F=订单号，G=客户名，H=金额）：

1. 在 B2 输入公式，**从英文输入法**开始打：
   ```
   =VLOOKUP(A2, F:H, 3, FALSE)
   ```
2. 含义拆解：
   - `A2`：要查的订单号；
   - `F:H`：只在 F 列（区域第一列）里找 A2；
   - `3`：找到后返回 F:H 区域的**第 3 列**（即 H 列金额）——注意是**相对区域**的列号，不是工作表绝对列号；
   - `FALSE`（或写 `0`）：精确匹配，日常取数几乎永远写它。
3. 回车 → **双击 B2 右下角填充柄**（或双击填充柄），整列匹配完成。

**找不到时**：单元格显示 **`#N/A`**，意思是"这个值在首列里不存在"——不是公式语法错，是数据里真的没有。

**两大常见错误**（新手 90% 的 VLOOKUP 报错都在这）：

1. **查询列不在首列**：你想按「订单号」查，但区域选成了 `G:H`（G 列是客户名不是订单号）→ 结果全错或大量 `#N/A`。**修正**：调整区域使查询列位于第一列，如改为 `F:H`。
2. **列号数错**：区域是 `F:H` 共 3 列，要返回 G 列应填 `2` 不是 `7`（7 是工作表第 7 列的绝对编号）。

**你可能会问**：要查的列偏偏不在区域第一列怎么办？——别硬搬列顺序，直接换函数，往下看。

#### 教学用最小数据表（可自建练手）

数据匹配专题课里的示例结构——你可以在 Sheet1 自己建这 5 行练 VLOOKUP：

| 列一(A) | 列二(B) | 列三(C) | ID(D) |
| --- | --- | --- | --- |
| 100 | ... | ... | ID001 |
| 200 | ... | ... | ID002 |
| ... | | | |

在 F 列写查找值 `100`，G 列公式 `=VLOOKUP(100, $A$2:$D$6, 4, FALSE)` 应返回 `ID001`。故意把区域改成 `$B$2:$D$6`（查询列不在首列）→ 观察 `#N/A` 或错值；把返回列号写成 `5`（超出区域宽度）→ 观察 `#REF!`。**亲手踩一次坑，比看十遍讲解记得牢。**

### 2.2.2 XLOOKUP 与 INDEX-MATCH：更灵活的查找方案

**生活化类比**：VLOOKUP 像只能"从左往右看"的字典；XLOOKUP 和 INDEX-MATCH 像带全文检索的电子词典——从哪列找、返回哪列，随你指定。

**技术定义**：当 VLOOKUP 查询列不在首列或需要更灵活的定位时，用 XLOOKUP（新版 Excel / 新版 WPS）或 INDEX+MATCH（全版本通用）替代。详见 [数据分析-Excel-XLOOKUP与INDEX-MATCH查找方案](/daishixiong/atoms/数据分析-excel-xlookup与index-match查找方案.html)。

#### 方案 A：XLOOKUP（推荐，Excel 2021+ / WPS 新版）

截图式操作——查询列在 G 列、返回列在 F 列（VLOOKUP 做不了）：

1. 在目标单元格输入：
   ```
   =XLOOKUP(A2, G:G, F:F)
   ```
2. 查找列 `G:G`、返回列 `F:F` 可以任意位置，**不必在首列**。
3. 后三个参数（未找到时显示什么、匹配模式、搜索方向）可选，默认精确匹配、从头到尾搜。

**版本限制**：Excel 2010 及以下没有 XLOOKUP，会报 **`#NAME?`**（函数名无法识别）——此时只能用 INDEX+MATCH 或调整列顺序后用 VLOOKUP。

#### 方案 B：INDEX + MATCH（低版本保底）

```
=INDEX(数据区域, MATCH(查找值, 查找列, 0), 列号)
```

截图式操作——在 `$A$2:$D$6` 区域中，找「列一」里值为 `100` 对应的 ID（第 1 列）：

1. 行号：`=MATCH(100, $A$2:$A$6, 0)` → 返回 100 在第 1 列中排第几行（如第 2 行）；
2. 取值：`=INDEX($A$2:$D$6, 2, 1)` → 取第 2 行第 1 列，即 ID；
3. 合并为一行：
   ```
   =INDEX($A$2:$D$6, MATCH(100, $A$2:$A$6, 0), 1)
   ```

**选型建议**：

| 场景 | 推荐 |
| --- | --- |
| 新版 Excel/WPS、查询列不在首列 | XLOOKUP |
| 低版本 Excel | INDEX+MATCH |
| 列号需要动态确定（宽表翻译） | INDEX+MATCH 嵌套 |

**WPS 与 Excel 怎么选**（数据匹配专题课补充）：

| 维度 | WPS | Excel |
| --- | --- | --- |
| 函数 | 新版支持 XLOOKUP | XLOOKUP 需较新版本 |
| Power Query | 较弱 | 编辑器好用，适合大批量 |
| 实际建议 | 公司指定用什么就用什么 | 写函数 WPS 够用；大批量清洗优先 Excel + PQ |

**你可能会问**：XLOOKUP 这么好用，为什么还要学 INDEX+MATCH？——因为 Excel 2010 及以下版本不支持 XLOOKUP，而很多公司电脑用的正是老版本；INDEX+MATCH 是走到哪都能用的保底方案，也是下一节动态查找的基础。

### 2.2.3 INDEX-MATCH 动态列查找（宽表翻译）

**生活化类比**：你在国外图书馆帮朋友找书，朋友给的是中文书名。你得先查"中文书名 → 英文书名"对照表，再拿英文书名去检索系统找位置——两步翻译，一步取书。

**技术定义**：面对列名需要翻译、列号不固定的宽表（列非常多的表），用 INDEX+MATCH 嵌套实现「中文列名 → 英文列名 → 列号 → 行号 → 取值」的动态查找。详见 [数据分析-Excel-INDEX-MATCH动态列查找](/daishixiong/atoms/数据分析-excel-index-match动态列查找.html)。

#### 真实场景：133 列英文宽表 + 对照表

合并表格专题课的课后练习：

- **视频信息表**：133 列全英文（爬虫数据）；
- **字段对照表**：133 行，第 1 列英文名、第 2 列中文名；
- **目标**：给定「达人抖音号」，提取「达人昵称」等 12 个中文列。

VLOOKUP 直接做不了——查询列不在首列，且列号随需求变化。

#### 标准方案：三步嵌套（截图式）

**第一步：翻译列名**——中文「达人昵称」→ 英文列名：

```
=INDEX(对照表!$A$2:$B$134, MATCH("达人昵称", 对照表!$B$2:$B$134, 0), 1)
```

**第二步：定位列号**——拿英文列名，在宽表**第 1 行表头**里找是第几列：

```
=MATCH(上面那个翻译结果, 视频信息表!$1:$1, 0)
```

**第三步：定位行号**——同理，先把「达人抖音号」翻译成英文列名，再在该列 MATCH 具体 ID 值。

**完整公式**：

```
=INDEX(视频信息表!$A$2:$Z$1000, 行号公式, 列号公式)
```

填充时注意：**列号公式里锁列 `$`**，**行号公式里锁行 `$`**，避免填充时行列一起漂移。

#### 紧急方案（20 分钟交差）

1. 在对照表旁插入辅助列，用 INDEX+MATCH 把中文列名批量翻译成英文；
2. 把宽表里需要的列**复制出来、调整顺序**，让查询列排到第一列；
3. 用最熟的 VLOOKUP 完成匹配——先拿分，再练优雅。

**你可能会问**：公式套三层，笔试时脑子转不过来怎么办？——有个紧急替代方案：插入辅助列先把列名翻译好，把需要的列复制出来调整顺序，然后用你最熟的 VLOOKUP。先拿分，再练优雅。

## 2.3 条件汇总与报表复现

匹配会把散落的信息拼到一起；**条件汇总**则是分析师出报表的核心——按月份、区域、省份等维度，把明细里的数字加总成老板看得懂的指标。本节先讲 SUMIF/SUMIFS 和环比，再用 2024 版 Excel 课里的**完整案例**把 2.1 的引用方式和 2.3 的函数串起来。

### 2.3.1 SUMIF 与 SUMIFS：按条件求和

**生活化类比**：班主任统计成绩，"只算女生的总分"是单条件求和；"只算三班的、女生的总分"是多条件求和。

**技术定义**：SUMIF 对满足单一条件的行求和；SUMIFS 支持多个条件同时判断。它们是报表中按月份、区域、省份等维度做汇总的核心函数。详见 [数据分析-Excel-SUMIF与SUMIFS条件汇总](/daishixiong/atoms/数据分析-excel-sumif与sumifs条件汇总.html)。

#### 参数顺序（新手第一坑）

```
=SUMIF(条件区域, 条件, 求和列)          ← 求和列在最后
=SUMIFS(求和列, 条件区域1, 条件1, ...)  ← 求和列在第一个
```

两个函数参数顺序**相反**，写反了结果全错或报 **`#VALUE!`**。戴师兄建议：熟练后**统一用 SUMIFS**，哪怕只有一个条件，将来加条件不用改函数名。

#### 截图式操作：月累计成交金额

2024 报表课场景——原数据 15 列、1657 行，H4 是月份筛选器（4 或 5）：

1. 在 B6 输入：
   ```
   =SUMIF(N:N, $H$4, E:E)
   ```
   （假设 N 列是「成交月份」，E 列是「成交金额」——以你实际列为准）
2. 含义：找出成交月份 = H4 筛选器里选的月份的所有行，对 E 列求和。
3. **选中 B6 公式里 `N:N` 和 `$H$4`，按 F4 锁成 `$N:$N` 和 `$H$4`**，再向右/向下填充时月份条件不会跑偏。
4. 成交客户数（D6）：同样结构，只把求和列改成客户数列。

**你可能会问**：SUMIF 和筛选后手动求和有什么区别？——筛选求和是一次性的，SUMIF 是公式，数据更新后自动重算，而且能一次性填充出几十行的汇总报表。

#### SUMIFS：省份 × 区域 × 月份三条件

省份明细需要同时判断**月份 + 区域 + 省份**，在 A11 行起填充：

```
=SUMIFS($E:$E, $N:$N, $H$4, $A:$A, $A11, $B:$B, $B11)
```

| 引用对象 | 引用方式 | 原因 |
| --- | --- | --- |
| 求和列 `$E:$E`、条件区域 `$N:$N`、`$A:$A`、`$B:$B` | 绝对引用 | 填充时列范围不能变 |
| 筛选器 `$H$4` | 绝对引用 `$H$4` | 固定在一个单元格 |
| 当前行区域 `$A11`、省份 `$B11` | 混合引用 `$A11` | 向下填充行号变，向右填充列不变 |

### 2.3.2 环比计算与 IFERROR 容错

**生活化类比**：这个月工资比上个月涨了多少？拿本月工资除以上月工资再减 1，就是"环比"。但如果你上个月刚入职没有工资，这道除法就没法算——需要一个兜底显示。

**技术定义**：环比 = 本期值 ÷ 上期值 − 1，反映相邻两期的变化幅度。算环比前必须确认上期数据存在；用 IFERROR 函数包裹公式，可以在分母缺失报错时显示替代内容。详见 [数据分析-Excel-环比计算与IFERROR容错](/daishixiong/atoms/数据分析-excel-环比计算与iferror容错.html)。

#### 完整案例：2024 版「销售汇总报表复现」

下面把 2.1–2.3 串成一条可照着做的链路（数据来自 2024 Excel 升级课，2020 年 4–5 月借贷销售明细）。

**Step 0 · 认识原数据**

| 字段 | 说明 |
| --- | --- |
| 成交日期 | 2020 年 4–5 月 |
| 销售工号 / 产品类型 / 期数 | 组织与产品维度 |
| 成交客户数 / 成交金额 | 核心度量 |
| 区域 / 省份 / 小组 / 业务组 | 组织架构 |
| 成交年份 / 成交月份 | `=YEAR()`、`=MONTH()` 从日期提取 |

打开后：`Ctrl+Shift+L` 开筛选，`Ctrl+↓` 探行数，状态栏看选中列汇总。

**Step 1 · 分屏对照**

1. 新建工作表，命名为「报表复现」；
2. **视图 → 新建窗口**（或 `Win+方向键` 把两个窗口左右贴边）；
3. 左边对照目标报表样式，右边动手写公式。单元格定位：列用字母（A、B…），行用数字，区域如 `A2:H3`。

**Step 2 · 标题与筛选器**

1. 合并 A1:B1，输入「最新数据时间」，右对齐；
2. C1 输入 `=MAX(A:A)`（A 列为成交日期），设**短日期**格式 → 显示最后一条成交日；
3. G4 输入「月份筛选」，H4 设**数据验证**：
   - **数据 → 数据验证 → 允许：序列 → 来源填 `4,5`**（英文逗号分隔）；
4. 合并 A2:H3 作标题，公式 `=H4&"月销售数据汇总"` → 选 4 月显示「4月销售数据汇总」，选 5 月自动变。

**Step 3 · 核心指标（SUMIF）**

| 单元格 | 公式逻辑 | 示例 |
| --- | --- | --- |
| B6 月成交金额 | SUMIF(月份列, H4, 金额列) | `=SUMIF($N:$N,$H$4,$E:$E)` |
| D6 月客户数 | 同上，换求和列 | |
| F6 客单价 | 金额 ÷ 客户数 | `=B6/D6` |

**Step 4 · 环比（SUMIF 复制 + 改条件）**

1. 在环比区域先算**本期**（条件 = H4）和**上期**（条件 = H4−1）：
   - 上期金额公式：把 SUMIF 里 `$H$4` 改成 `$H$4-1`；
2. 环比：`=本期/上期 - 1`；
3. 套容错：
   ```
   =IFERROR(本期/上期-1, "-")
   ```
4. **数据前提**：算 5 月环比需要 4 月数据存在；原数据只有 4–5 月时，4 月环比（缺 3 月）会 `#DIV/0!`——这不是公式错，是**数据边界**，IFERROR 显示 `-` 即可。

**Step 5 · 多维度明细（SUMIFS + 混合引用）**

从第 11 行起填区域、省份明细。2024 课里报表左侧先列出区域名（如华东北、华西北…），右侧是各指标列。以 **A11 =「华东北」、B11 =「山东」** 为例，C11 写月成交金额：

```
=SUMIFS($E:$E, $N:$N, $H$4, $A:$A, $A11, $B:$B, $B11)
```

**截图式填充**：

1. 在 C11 写好公式并确认结果合理（切换 H4 的 4/5 月，数字应随之变化）；
2. **选中 C11 单元格**，拖动右下角填充柄**向下**到所有省份行——`$A11` 会变成 `$A12`、`$A13`…，区域名逐行变；
3. 若还要向右复制到「客户数」列，确认 `$A11` 没有变成 `B11`（列被锁死）；
4. 若某行结果为 0，先筛原数据看该「区域+省份+月份」组合是否真有成交，再查公式引用是否写错列字母。

**Step 5 常见报错**：

| 报错 | 常见原因 |
| --- | --- |
| `#VALUE!` | SUMIFS 参数个数不成对，或多写了文本条件 |
| 整列都是同一个数 | 区域/省份条件写成了绝对引用 `$A$11`，填充时不随行变 |
| 结果比筛选手工求和偏大 | 条件区域没锁列，填充后指向了错误列 |

**Step 6 · 产品期数占比**

- 各期成交金额：SUMIFS，条件 = 期数 + 月份；
- 占比 = 该期金额 ÷ 总金额（分母 `$B$6` 绝对引用）；
- 总计行：选中数字区域按 **`Alt+=`** 快速求和。

**Step 7 · 格式美化（可选但职场常用）**

- 金额：**千分位、0 位小数**；环比/占比：**百分比**；
- **条件格式**：环比 >0 绿色↑，&lt;0 红色↓；金额**数据条**；占比**色阶**；
- 自定义格式 trick：`[Green]0.0%↑;[Red]0.0%↓` 在显示层加箭头，**不改变原值**；
- **视图 → 取消网格线**，加边框分区。

**关键习惯**（原视频反复强调）：

- 所有括号、逗号用**英文输入法**；
- 函数嵌套多层时，保持清晰逻辑——复杂公式是一层层简单组合；
- 做环比前先确认分母月份数据是否存在。

**同比**同理：把"上期"换成"去年同期"，如 2020 年 5 月 ÷ 2019 年 5 月 − 1。

**你可能会问**：IFERROR 是不是把所有错误都藏起来了，会不会掩盖真正的公式错误？——会有这个风险，所以正确的顺序是：先把公式调试正确、确认报错只可能来自"上期数据缺失"这种合理边界，再套 IFERROR 做美观容错。

## 2.4 Power Query 合并表格

前面的函数都在"一张表"里干活。但真实工作中你经常收到的是：12 个月的销售表、每个分公司一个文件——几十张表怎么合成一张？手动复制粘贴既慢又容易错，这时候就轮到 **Power Query**（简称 PQ，Excel 内置的数据获取与清洗工具）出场。

PQ 的合并查询（JOIN）和追加查询（UNION），就是下一章 SQL 的预告片——在这里先混个脸熟，第三章就好学了。

### 2.4.1 Power Query 数据导入方式

**生活化类比**：PQ 像一个"数据中转站"：原材料（各处的表格）先运进中转站加工，加工完再决定送到哪个仓库（工作表）。

**技术定义**：Power Query 通过三种常见入口获取数据：当前表区域、外部工作簿、文件夹批量导入。它是 Excel 处理大批量数据清洗与合并的基础设施。详见 [数据分析-Excel-PowerQuery数据导入方式](/daishixiong/atoms/数据分析-excel-powerquery数据导入方式.html)。

#### 入口在哪里

Excel 顶部：**数据 → 获取和转换数据**（或右侧「查询和连接」面板）。PQ 负责 **获取 → 转换 → 加载** 三步。

#### 三种导入方式（截图式对照）

| 方式 | 菜单路径 | 适用场景 | 操作要点 |
| --- | --- | --- | --- |
| 来自表格/区域 | 数据 → 从表格/区域 | 当前 Sheet 里已有表 | 会先提示转为**智能表**再进编辑器 |
| 来自工作簿 | 数据 → 获取数据 → 来自文件 → 工作簿 | 单个外部 xlsx | 导航器里勾选具体 Sheet → 转换/加载 |
| 来自文件夹 | 数据 → 获取数据 → 来自文件 → **文件夹** | 批量同结构多文件 | 列出文件 → 保留 Content 列 → **合并文件** |

#### 截图式操作：从文件夹批量导入 12 个月销售表

1. **数据 → 获取数据 → 来自文件 → 从文件夹**；
2. 浏览选中存放 12 个 xlsx 的文件夹 → **确定**；
3. PQ 列出文件夹内所有文件；在预览里**只保留 Content 列**（或先过滤再合并）；
4. 点击 Content 列右侧 **「合并文件」** 按钮（或「合并二进制」）；
5. 若结构一致，12 张表纵向合成一张 → 进入 **Power Query 编辑器** 做后续清洗。

**从文件夹导入的三个坑**：

| 坑 | 现象 | 处理 |
| --- | --- | --- |
| 路径含 `~` | 多出一个奇怪文件 | 正在打开的 Excel 缓存，**过滤掉** Name 列含 `~` 的行 |
| 混进人员表/汇总表 | 合并后多出无关行 | 按文件名或扩展名**排除** |
| 隐藏文件 | 莫名多几行 | 读到后删除，或过滤 Attribute Hidden |

**数据源路径**：默认是**绝对路径**。文件挪位置后 → **数据 → 查询和连接 → 右键查询 → 数据源设置 → 更改源** 重新定位。结构相同的表可以直接**替换数据源**（如 6 月表换成 4 月）。

**大文件技巧**：几十万行 xlsx 直接打开会卡死 → **新建空工作簿**，用 PQ **连接**大文件再加载，绕过 Excel 直接打开。

**你可能会问**：几十万行的大文件，Excel 一打开就卡死怎么办？——不要直接打开它。新建一个空表，用 PQ 连接那个大文件再加载，绕过 Excel 直接打开的卡顿。

### 2.4.2 追加查询与合并查询

**生活化类比**：**追加**像把两摞同样规格的试卷摞在一起（上下拼接）；**合并**像把学生成绩单和学生档案按姓名对齐后拼成一张宽表（左右连接）。

**技术定义**：追加查询把多张结构相同的表纵向拼接（对应数据库里的 UNION）；合并查询把两张表按共同字段左右连接（对应数据库里的 JOIN）。详见 [数据分析-Excel-PowerQuery追加与合并查询](/daishixiong/atoms/数据分析-excel-powerquery追加与合并查询.html)。

#### 完整案例：4–6 月销售汇总（课后作业逐步操作）

透视表课 + 合并表格课共用的真实作业——把 4–6 月销售明细合成一张「销售汇总数据」。

**数据结构**：

- **4–5 月历史表**：已含区域、省份、小组；
- **6 月**：分华东 / 华南 / 华西北三个 xlsx，**缺**区域/省份/小组（需从人员表补）；
- **人员表**：销售工号 → 区域 / 省份 / 小组。

**操作流程（建议新建空表「销售汇总数据」跟着做）**：

**① 从文件夹导入 6 月三区域表**

1. 数据 → 从文件夹 → 选放三个 6 月文件的文件夹；
2. **过滤**：Name 列 **不包含** `~`；**排除** 人员表、汇总表等非目标文件；
3. Content 列 → **合并文件** → 得到查询「新增销售数据」（仅 6 月）。

**② 合并查询补全维度（左外部连接）**

1. PQ 编辑器 **主页 → 合并查询**；
2. 左表选「新增销售数据」，右表选「人员表」；
3. 两表各点选 **销售工号** 列（多键时 **按住 Ctrl** 依次点两表对应列）；
4. 连接类型选 **左外部**（保留全部销售行，带上匹配到的人员信息）→ 确定；
5. 新列 `人员表` 右侧点 **展开** → 勾选区域、省份、小组 → **取消勾选「使用原始列名作为前缀」** → 确定。

**③ 追加 4–5 月历史**

1. 先 **关闭并上载至 → 仅创建连接**（见 2.4.3），导入 4–5 月表也为连接；
2. 在「新增销售数据」查询上：**主页 → 追加查询** → 选「4–5 月历史」→ 确定；
3. 列名需一一对应；列顺序不同但**列名相同**可对齐；列名也对不上会错乱（需改 M 语言，入门先避开）。

**④ 加载最终结果**

关闭并上载至 → 仅创建连接 → 在查询面板右键最终查询 → **加载到 → 表 → 现有工作表**「销售汇总数据」。

**追加 vs 合并速查**：

| 操作 | 菜单 | 结果 | SQL 对应 |
| --- | --- | --- | --- |
| 追加查询 | 主页 → 追加查询 | 上下拼接 | UNION |
| 合并查询 | 主页 → 合并查询 | 左右连接 | JOIN |

**合并连接类型**：

| 类型 | 含义 |
| --- | --- |
| 内连接 | 两表匹配键都存在的行才保留 |
| 左外部 | 保留左表全部，右表只带上匹配到的行 |
| 右外部 | 保留右表全部 |
| 全外部 | 两表所有行都保留 |
| 左反 / 右反 | 只保留**没**匹配上的行 |

**常见报错**：

| 现象 | 原因 |
| --- | --- |
| 展开后列名变成 `人员表.区域` | 没取消「使用原始列名作为前缀」 |
| 追加后某列数据串行 | 两表列名不一致或列顺序错位 |
| 合并后大量 null | 连接键格式不一致（如工号一端是文本一端是数字） |

**从文件夹合并多 Sheet 注意**：PQ 按**列顺序**粗暴合并各 Sheet——列顺序必须相同，否则列会错位；列名相同但顺序不同需改 M 函数（入门阶段先统一模板）。

**你可能会问**：合并查询和 VLOOKUP 是不是干同一件事？——目的相同（按键匹配带字段），但量级不同：VLOOKUP 适合一张表内的少量匹配；几万行以上、多表反复合并，用 PQ 更稳、可刷新、不用拖公式。这套连接思想到第三章学 SQL 的 JOIN 时会原样重现——在这里先混个脸熟，SQL 就好学了。

### 2.4.3 Power Query 加载策略

**生活化类比**：装修完不会把施工的脚手架也留在客厅。PQ 的中间步骤（脚手架）不该全部塞进工作簿，只把最终成品加载出来。

**技术定义**：PQ 处理完成后，用「仅创建连接」让中间查询不落地成 Sheet，再选择性地把最终结果加载到指定位置。详见 [数据分析-Excel-PowerQuery加载策略](/daishixiong/atoms/数据分析-excel-powerquery加载策略.html)。

#### 截图式操作：仅创建连接 + 选择性加载

1. PQ 编辑器右上角 **关闭并上载至** 下拉 → 选 **仅创建连接**；
2. 回到 Excel，右侧 **查询和连接** 面板列出所有查询（带「仅连接」图标）；
3. 找到最终那个查询（如「销售汇总数据」）→ **右键 → 加载到…**；
4. 选 **表** → **现有工作表** 或 **新工作表** → 确定。

**误操作恢复**：手快点了「关闭并上载」，中间表全变成 Sheet 铺满工作簿——

1. 查询面板里把中间查询逐个 **右键 → 加载到 → 仅创建连接**；
2. **删除**工作簿里多余的中间 Sheet；
3. 只对最终查询执行「加载到 → 表」。

**理解 PQ 的本质**：

- 通过 PQ 加载的表，Sheet 上是**连接关系**，数据还在源文件里；
- PQ 记住的是**操作步骤**（M 语言），不是静态副本；
- 源表更新后 → 查询上 **右键 → 刷新**，所有步骤自动重跑。

**你可能会问**：那我把源文件删了会怎样？——连接失效，刷新会报 **`DataSource.Error`** 或找不到路径（已加载出来的数据还在 Sheet 里，但无法再更新）。所以用 PQ 的工作簿要和源文件一起妥善保管，挪动位置后记得改数据源路径。

## 2.5 数据透视表

如果说函数是"手动挡"，数据透视表就是 Excel 的"自动挡"——不写一个公式，拖拽几下字段就能完成多维度汇总。来源文章 [Excel中最被低估的功能，学会数据透视表，你的Excel速度直接翻3倍](/daishixiong/articles/excel中最被低估的功能-学会数据透视表-你的excel速度直接翻3倍.html) 的标题不是夸张：透视表 + 切片器 + GETPIVOTDATA 的组合，就是很多分析师日常看板的雏形——第四章学 Tableau/Power BI 之前，你已经在 Excel 里体验过"交互式可视化"的核心逻辑了。下面用 2.4 合并好的「销售汇总数据」继续练手，从创建到透视图完整走一遍。

### 2.5.1 创建透视表与数据源设置

**生活化类比**：透视表像一台"自动分拣机"：把一大袋混装的快递（明细数据）倒进去，指定按城市分堆、按重量称总数，它咔咔几下就分好了。

**技术定义**：数据透视表通过拖拽字段对明细数据做多维汇总。创建效率可以用快速访问工具栏 + Alt 快捷键提升；数据源推荐使用智能表，以便新增数据自动纳入。详见 [数据分析-Excel-数据透视表创建与数据源](/daishixiong/atoms/数据分析-excel-数据透视表创建与数据源.html)。

#### 截图式操作：一键插入透视表

**设置快速访问工具栏（做一次，永久受益）**：

1. **插入 → 数据透视表**（先点一下菜单，不用真的创建）；
2. 在按钮上 **右键 → 添加到快速访问工具栏**；
3. 看快速访问栏它是第几个图标（比如第 4 个）→ 以后按 **`Alt+4`** 一键弹出「创建数据透视表」对话框。

**创建步骤**：

1. 选中数据区域（推荐先 **`Ctrl+T`** 转智能表）；
2. `Alt+4` 或 插入 → 数据透视表；
3. 选「新工作表」或「现有工作表」→ 确定；
4. 右侧出现**字段列表**，把字段拖进行、列、值、筛选四个区域。

**数据源三种选法的差别**：

| 数据源方式 | 新增行处理 | 推荐度 |
| --- | --- | --- |
| 框选连续区域 | 需手动「分析 → 更改数据源」 | 一般 |
| 整列选中（如 A:H） | 刷新后自动纳入新行 | 较好 |
| 智能表（Ctrl+T） | 自动检测新行并扩展（角标提示） | **推荐** |

**复制透视表**：必须选中**完整**透视表（含行标签、列标签、值区所有部分）再复制，粘贴后才有「数据透视表工具」选项卡；只框选一部分复制，得到的只是静态数值，无法刷新。

**你可能会问**：想复制一张透视表改改再用，怎么复制出来的是死数据？——必须选中**完整**的透视表（包含所有组成部分）再复制，粘贴出来的才带「数据透视表工具」；只框选其中一部分复制，得到的只是静态数值。

### 2.5.2 行/列/值字段规则

**生活化类比**：切蛋糕。行字段是横着切一刀（按区域分），列字段是竖着切一刀（按月份分），值字段是问每一块"有多重"（金额求和）。

**技术定义**：行/列区域放**维度**（离散的分类字段，如区域、省份、期数）；值区域放**度量**（可聚合的连续数值，如成交金额，默认求和）。详见 [数据分析-Excel-透视表行列值字段规则](/daishixiong/atoms/数据分析-excel-透视表行列值字段规则.html)。

#### 截图式操作：零公式汇总

1. 把「区域」拖到**行**；
2. 把「成交金额」拖到**值**；
3. 立刻得到各区域成交金额汇总——**零公式**。

**三个高频细节**：

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 把「省份」拖进值区，数字很小 | 离散字段默认**计数**不是求和 | 值字段设置改为「求和」或拖回行区 |
| 销售工号显示 1.23E+10 | 被识别成数值 | 源数据先把该列转成**文本** |
| 想核对某个汇总数字 | — | **双击**该汇总单元格 → 新开 Sheet 展示明细行 |

**常见报错**：刷新时报 **「无法覆盖」**——同一张 Sheet 上多张透视表，前面的扩展撞上了后面的表。解决：表之间**预留空行**，或一 Sheet 一透视表。

**你可能会问**：刷新时报「无法覆盖」错误是怎么回事？——同一张 Sheet 上放了多张透视表，前面的透视表加字段后要扩展空间，撞上了后面的表。给每张透视表之间预留足够空行，或干脆一张 Sheet 只放一张透视表。

### 2.5.3 筛选、排序与布局

**生活化类比**：分拣完的货物还要"上架陈列"：哪些货架要展示（筛选）、按销量摆放顺序（排序）、货架样式选哪种（布局）。

**技术定义**：透视表支持维度值筛选、按度量排序和多种报表布局；日期字段拖入后会自动组合成年/月/日层级。详见 [数据分析-Excel-透视表筛选排序与布局](/daishixiong/atoms/数据分析-excel-透视表筛选排序与布局.html)。

#### 按成交金额排序（截图式）

1. 在「区域」列任意单元格 **右键 → 排序 → 其他排序选项**；
2. **排序依据**选「成交金额」，**降序** → 确定；
3. 若有「区域 + 省份」两层，**内外层维度要分别设置排序**——只排外层，内层省份仍可能是拼音序。

#### 字段筛选 vs 值筛选（易混）

- **维度值筛选**：作用在该维度的**汇总层面**。筛「区域成交金额 > 1000 万」→ 筛掉的是**整个区域**，不是区域下的省份行。
- **值筛选灰色不可用**：先点击透视表**旁边的空白单元格**（让焦点离开字段），再添加值筛选器。

#### 格式：值字段设置 vs 开始栏

- **右键值字段 → 值字段设置 → 数字格式**：对整个值字段统一生效；
- **开始 → 数字格式**：只对当前选中单元格生效——改一个漏一堆。

#### 布局与默认选项

- **设计 → 报表布局**：压缩 / 大纲 / 表格形式；可 **重复所有项目标签**；
- **分类汇总**：每组底部或顶部显示小计；
- **固定默认布局**：**文件 → 选项 → 数据 → 编辑默认布局 → 导入当前设置**；
- 常用：**空单元格显示为 `-`**；**更新时自动调整列宽**；勾选 **启用显示明细数据**（双击看明细的前提）。

**日期字段**：拖进行区会自动组合年/月/日。前提是源列是**真日期**——**文本型日期**无法组合也无法设格式（回顾 2.1.2）。

**你可能会问**：日期字段拖进去怎么自动变成了年、月？——这是日期字段的自动组合特性，方便按年月下钻。但前提是这一列是真正的日期型；**文本型日期**既无法组合也无法设置数字格式——回顾 2.1.2，数据类型永远是地基。

### 2.5.4 切片器与日程表

**生活化类比**：遥控器。不用趴到电视机（透视表）跟前按按钮，拿着切片器点一下，一台甚至几台"电视"同时换台。

**技术定义**：切片器是透视表的可视化筛选控件，一个切片器可以同时控制多张透视表；日程表是专用于日期字段的切片器，以日历形式选择连续时间范围。详见 [数据分析-Excel-切片器与日程表](/daishixiong/atoms/数据分析-excel-切片器与日程表.html)。

#### 截图式操作

1. 选中透视表 → **插入 → 切片器** → 勾选「区域」→ 确定；
2. 点击切片器上「华西北」，透视表立即只显示该区域；
3. **一控多**：右键切片器 → **报表连接** → 勾选其他透视表名称 → 确定。

**实践建议**：

- 透视表一多，务必 **分析 → 数据透视表名称** 改成可读名字（如「区域汇总_销售」），否则报表连接列表里认不出；
- **日程表**适合连续月份；断选（只看 4 月和 6 月、跳过 5 月）不符合它的设计——戴师兄建议自定义「成交月份」**普通切片器**更灵活。

**你可能会问**：切片器和普通筛选器选哪个？——给自己用、追求快，普通筛选够了；要做给老板看的交互式报表、或要一次控制多张透视表/透视图，用切片器。

### 2.5.5 计算字段与计算项

**生活化类比**：分拣机自带"称重"和"计数"，但你想要"平均每单重量"这种它没有的指标——得自己写个小算式装上去。

**技术定义**：**计算字段**是不同列（字段）之间的运算，如客单价 = 成交金额 ÷ 成交客户数；**计算项**是同一字段内不同数据项之间的运算，如 18 期 − 12 期。详见 [数据分析-Excel-计算字段与计算项](/daishixiong/atoms/数据分析-excel-计算字段与计算项.html)。

| 类型 | 运算对象 | 示例 | 入口 |
| --- | --- | --- | --- |
| 计算字段 | 不同列之间 | 客单价 = 金额 ÷ 客户数 | 分析 → 字段、项目和集 → 计算字段 |
| 计算项 | 同列的不同数据项 | 18 期 − 12 期 | 先**点击维度内具体数据项**（如「12期」）再进同一菜单 |

#### 截图式操作：添加「总客单价」

1. 选中透视表 → **分析 → 字段、项目和集 → 计算字段**；
2. 名称填 `总客单价`，公式填 `=成交金额/成交客户数`（从下方字段列表双击插入，避免拼写错）；
3. 确定 → 新字段出现在值区域。

**组合技巧**：选中多个维度项（如几个省份）→ 右键 **组合** → 生成「分组1」等大区字段；日期字段可组合年/季/月/周，**按周组合**时设置「起始于」对齐周一到周日。

**你可能会问**：值汇总方式里没有"不重复计数"（比如统计去重后的客户数）怎么办？——普通透视表做不到，需要 **Power Pivot + DAX** 的 `DISTINCTCOUNT`（本章不展开，知道方向即可）；中位数同理可用 MEDIAN 或 Power Pivot。

### 2.5.6 值显示方式与环比

**生活化类比**：同一份成绩，可以报"考了 85 分"（绝对值），也可以报"占满分的 85%"（占比），还可以报"比上次多了 5 分"（差异）——数字没变，"说法"切换了。

**技术定义**：在求和结果的基础上，透视表的「值显示方式」可以一键切换成占比（总计/行/列/父级百分比）或差异（环比）。详见 [数据分析-Excel-透视表值显示方式与环比](/daishixiong/atoms/数据分析-excel-透视表值显示方式与环比.html)。

#### 占比：分母区别

| 显示方式 | 分母 |
| --- | --- |
| 总计的百分比 | 全表总和 |
| 列汇总的百分比 | 所在列总和 |
| 行汇总的百分比 | 所在行总和 |
| 父行/父列汇总的百分比 | 上一级维度的汇总 |

饼图/环形图常配合 **列汇总的百分比** 看结构占比。

#### 截图式操作：一键月环比

1. 值区域**再拖入一次**「成交金额」（出现「成交金额2」）；
2. 右键「成交金额2」→ **值显示方式**；
3. 选 **差异百分比**；
4. **基本字段**选「成交月份」（或「月份」），**基本项**选 **「上一个」** → 确定；
5. 等价于 `(本月 − 上月) / 上月`，**一个公式都不用写**。

日环比：基本字段选「成交日期」+「上一个」；年环比：选「成交年份」+「上一个」。

**前提条件**与 2.3.2 手写环比完全一致：透视表里必须**包含**上一个月的数据——把前月筛掉了，当月环比就算不出来（参见 [数据分析-Excel-环比计算与IFERROR容错](/daishixiong/atoms/数据分析-excel-环比计算与iferror容错.html)）。

**差异**（非百分比）：指定某个维度值为基准，显示当前值 − 基准值的绝对差。

**你可能会问**：函数版环比（2.3.2）和透视表版环比选哪个？——快速探索、临时看数用透视表版，三下搞定；要嵌入固定格式的正式报表、对样式有要求时用函数版。两者原理完全一样。

### 2.5.7 GETPIVOTDATA 提取规则

**生活化类比**：透视表是一面自动更新的"成绩公告栏"，GETPIVOTDATA 就是派一个人盯着公告栏，把指定的那个数字抄到你的正式报表里。

**技术定义**：GETPIVOTDATA 函数按条件从透视表中提取汇总值，常用于"透视表做计算 + 正式报表做展示"的架构。使用时必须满足**可见**且**唯一**两个条件。详见 [数据分析-Excel-GETPIVOTDATA提取规则](/daishixiong/atoms/数据分析-excel-getpivotdata提取规则.html)。

#### 截图式操作

1. **分析 → 选项 → 勾选「生成 GETPIVOTDATA」**（For formulas...）；
2. 在报表空白单元格输入 `=`，**点击**透视表中想要的数值 → 公式自动生成，例如：
   ```
   =GETPIVOTDATA("成交金额",$A$3,"区域","华西北","期数","6-7")
   ```
3. 把写死的 `"华西北"` 改成引用报表筛选单元格（如 `$B$2`），报表就动态化了。

**两大硬性要求**：

| 要求 | 违反时的现象 |
| --- | --- |
| **可见** | 行/列被折叠隐藏 → 返回 **`#REF!`** 或错误值 |
| **唯一** | 条件定位到多个单元格（如期数在行内且无列汇总）→ 无法取「期数总计」 |

不需要的维度参数可以**整对删除**（如 `"业务组","xxx"` 这一对），但剩下条件仍须唯一定位。

**你可能会问**：为什么不直接在透视表上做正式报表？——透视表的样式、行列结构会随数据变动，正式报表往往有固定格式要求；「透视表算数 + GETPIVOTDATA 抄数」是兼顾自动化和格式稳定的常见架构。

### 2.5.8 数据透视图基础用法

**生活化类比**：数字表格像文字版天气预报，图表像天气地图——一眼看出哪里下雨。透视图就是"随透视表自动更新的天气地图"。

**技术定义**：数据透视图与透视表强联动，可快速生成柱状图、折线图、饼图和组合图，配合切片器实现动态可视化。详见 [数据分析-Excel-数据透视图基础用法](/daishixiong/atoms/数据分析-excel-数据透视图基础用法.html)。

#### 截图式操作：柱状 + 折线组合图

1. 透视表值区域放「成交金额」（求和）和「成交金额2」（差异百分比环比）；
2. 选中透视表 → **插入 → 组合图**（或插入柱形图再改类型）；
3. 成交金额用**簇状柱形图**（主轴）；
4. 环比勾选 **次坐标轴** + **折线图**——否则环比是 0.x 的小数，成交金额是千万级，折线会贴地看不见。

**常用搭配**：

| 图表 | 适用 |
| --- | --- |
| 柱状图 | 各区域/省份对比 |
| 折线图 | 每日/每月趋势 |
| 饼图/环形图 | 配合列汇总百分比看占比 |

**联动规则**：透视图行列与透视表一致，透视表切换行列，图表结构跟着变；切片器筛选后透视图同步更新——**切片器 + 透视图 = 小型交互看板**。

#### 进阶案例：单月 Top3 + 月环比排名图

**难点**：月环比需要**多个月同时在透视表**里算，但展示只要**单个月**的 Top3。

**解法**：

1. 透视表保留全部月份，用 2.5.6 算好各区域环比；
2. 展示层用 **GETPIVOTDATA** 或切片器锁定单月，取 Top3 区域做图；
3. 或在辅助区域用公式引用 GETPIVOTDATA 排出前三名再制图。

**你可能会问**：透视图和普通图表选哪个？——数据来自透视表、需要随筛选动态变化，用透视图；一次性展示静态数据，普通图表即可。

### 2.5.9 综合练手：从合并表到 mini 看板（一条龙）

把 2.4 课后作业和 2.5 各节合成一条**验收清单**——全部打勾，说明第二章核心技能已落地：

| 步骤 | 操作 | 验收标准 |
| --- | --- | --- |
| 1 | PQ 文件夹导入 6 月三区域表，过滤 `~` | 查询预览行数 ≈ 三表之和 |

### 2.5.10 本章 Excel 函数与透视表报错速查

初学阶段会把时间大量花在「公式怎么写对」上。下面这张表汇总了本章各节最高频的报错——下次遇到可以先对号入座，再回对应小节：

| 报错/现象 | 最可能原因 | 回哪一节 |
| --- | --- | --- |
| `#N/A` | VLOOKUP 查无此值；或 INDEX+MATCH 找不到 | 2.2.1 / 2.2.3 |
| `#NAME?` | 函数名拼错；或低版本用了 XLOOKUP | 2.2.2 |
| `#REF!` | VLOOKUP 返回列号超出区域；GETPIVOTDATA 目标被折叠 | 2.2.1 / 2.5.7 |
| `#DIV/0!` | 环比分母为 0 或上期无数据 | 2.3.2 |
| `#VALUE!` | SUMIF/SUMIFS 参数顺序写反；条件类型不匹配 | 2.3.1 |
| 日期显示为 43982 等大整数 | 单元格格式是常规而非日期 | 2.1.2 |
| 金额无法 SUM | 单元格是文本（手动输入了 ¥） | 2.1.2 |
| 填充后公式结果全变 0 | 引用没加 `$`，条件区域漂移 | 2.1.3 / 2.3.1 |
| PQ 刷新 `DataSource.Error` | 源文件移动或删除 | 2.4.1 / 2.4.3 |
| 透视表刷新「无法覆盖」 | 同 Sheet 多透视表布局冲突 | 2.5.2 |
| 透视表环比整列空白 | 筛掉了计算环比所需的上期数据 | 2.5.6 |

**调试顺序建议**（戴师兄 2024 报表课习惯）：① 先看数据类型（日期/文本/数值）→ ② 再看引用 `$` 是否锁对 → ③ 最后查函数名和参数顺序。类型错了，后面全白搭——这也是 2.1.2 放在全章最前面的原因。

## 本章小结

回顾本章的完整链路，正是分析师处理一份数据的真实动线——也是戴师兄 5 期 Excel 课串联起来的工作流：

1. **拿到表**：快捷键快速摸底（`Ctrl+Shift+L` 筛选、`Ctrl+↓` 探边界），确认日期/数值的数据类型没坑（43982 不是错，是日期序号）；
2. **补数据**：VLOOKUP / XLOOKUP / INDEX-MATCH 跨表匹配，133 列宽表用动态列查找三步嵌套，把散落的信息拼到一起；
3. **算汇总**：SUMIF/SUMIFS 按条件求和，2024 报表课完整复现 1657 行 → 带 H4 月份筛选器的多维度报表，加环比、套 IFERROR；
4. **多表合一**：Power Query 从文件夹批量导入、过滤 `~` 缓存、左外部合并查询补维度（销售工号连接人员表）、追加 4–5 月历史、仅创建连接后按需加载；
5. **出报表**：透视表拖拽出多维汇总，`Alt+4` 一键插入，切片器加交互，GETPIVOTDATA 喂给正式报表，组合图（主轴柱形 + 次轴折线环比）可视化。

**两条完整案例的对照**：

| 案例 | 来源视频 | 核心技能 | 数据规模 |
| --- | --- | --- | --- |
| 销售汇总报表复现 | 2024 Excel 升级课（上） | SUMIF/SUMIFS + 混合引用 + 环比 + 条件格式 | 15 列 × 1657 行 |
| 4–6 月销售合并 + 透视看板 | 合并表格专题 + 透视表课 | PQ 文件夹导入 + 合并/追加 + 透视表全套 | 3 区域 6 月表 + 历史 |

其中 Power Query 的合并查询（JOIN 思想）和追加查询（UNION 思想），就是下一章 SQL 的预告片——第三章我们把同样的思想搬进数据库，解决"数据不在 Excel 里，在公司数据库里"的取数问题。
