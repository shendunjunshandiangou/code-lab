# 爬虫三不原则合法边界

## 一句话定义
爬虫的安全边界是"不破解、不商用、不高频"：像正常访问一样的频率抓公开展示的数据等同复制粘贴，没有风险；出事的都是逆向破解加密、抓包批量请求、卖数据卖程序。

## 详细说明

**安全的**：
- 数据是页面上公开展示的，抓取等同复制粘贴
- 频率像正常用户访问（正常人一秒最多请求一个页面）
- 只自用，不拿数据或程序去卖

**危险的（真实被抓案例的行为）**：
- **破解加密**：逆向 API 加密算法、身份指纹（直播中演示了 B 站接口 32 位 MD5 加密参数如何被断点定位并复现）——绕过了对方的保护
- **抓包批量请求**：绕过前端一秒请求十次，直接消耗对方服务器资源（每个请求都是对方的钱）
- **商用获利**：把加密参数破解方案或抓来的数据卖给别人，侵害对方核心数据

**工具区别**：影刀这类 RPA 是模拟人工翻页，与抓包逆向截然不同，可以用。专职爬虫岗位一定会涉及逆向，不建议做。
