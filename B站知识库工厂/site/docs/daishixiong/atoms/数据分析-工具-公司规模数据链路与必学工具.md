# 公司规模数据链路与必学工具

## 一句话定义
把不同规模公司的数据链路画出来，就能从底层推导出为什么必学 Excel/SQL/BI、为什么越大的厂越不需要 Python。

## 详细说明

| 公司规模 | 数据链路 | 工具推论 |
| --- | --- | --- |
| 无数据建设 | 平台/后台下载 → Excel 处理 → Excel/PPT 汇报 | Excel 是基础中的基础 |
| 小厂 | 数据写入数据库（OLTP），清洗取数都在库里；可能没权限要找开发帮忙；或走 ERP | SQL |
| 中小厂 | 业务库要支撑前端（每次网页查询背后就是一段 SQL），取数会拖垮它 → 同步到另一个库专门查数（解耦）；固定取数塞进 BI 看板自动化 | SQL + BI |
| 中型/大厂 | 前端数据 → OLTP 库 → ETL 同步到数仓（做数仓分层，像仓库管理员按规范摆表）→ 加工好的数据同步到 OLAP 库（擅长查询）→ 支撑取数与 BI | SQL + BI，岗位分工明确 |

**关键推论**：
- 越大的厂越不需要 Python（数据分析视角）：获取不用你做、清洗存储 SQL 搞定、应用 BI 搞定。Python 出现在 JD 里多半是为了和算法同学对接。
- 任何数据技术问题都可从四个环节回答：**获取、清洗、存储、应用**。
- 越小/数据建设越差的公司，数分承担职能越多（大厂拆给 ETL、数仓、开发的活全归你），老板懂行还会要求爬虫。
- 用大厂 JD 可验证此框架：数据监控=BI、数据报告=分析、业务数据模型=指标体系、Hive=SQL；没写 Excel 是因为取数默认用 Excel 承接。
