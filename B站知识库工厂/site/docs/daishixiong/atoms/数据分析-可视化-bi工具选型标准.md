# BI 工具选型标准

## 一句话定义

选 BI 要「入门简单 + 深度足够」；价值是回答领导决策问题，而非炫技；须支持结构化分析，且「可以美观但不可没有美观能力」。

## 详细说明

**喜乐君标准**：
1. 入门简单，但 **不能五天就「学完」**——否则无竞争壁垒
2. 国内 BI 常：基础图极易，**高级定制图做不出** → 需 Python 补位，割裂流程
3. **反对**个人过度追求炫酷/复杂图形；分析师解决 **领导决策问题**
4. 最好 BI：简单图形也能回答深刻问题（多维/结构化分析）
5. **美观能力**：「可以做得美观我可以不用，但你不能没有」——领导要求美化时不能尴尬

**对业务分析师**：Tableau 仍是最好选择（易用 + 深度）；开发者可能偏好 Power BI 定制。

**RFM 测试**：好 BI 能在 **明细表上直接做 RFM**；必须中间表则割裂下钻。
