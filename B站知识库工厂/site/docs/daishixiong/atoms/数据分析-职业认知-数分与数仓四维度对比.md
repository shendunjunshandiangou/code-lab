# 数分与数仓四维度对比

## 一句话定义

数仓下限高、入门难、工作枯燥；数分上限高、体验好、入门易——分析师长远应懂数仓，数仓开发应懂业务分析。

## 详细说明

| 维度 | 数分 | 数仓 |
| --- | --- | --- |
| 薪资下限 | 低（中小厂五六千） | 高 |
| 薪资上限 | 商分/战略/策略+项目红利 | 架构负责人，因人而异 |
| 入门 | 工具多但难度低 | Hadoop/Spark/分层/调度，技术深 |
| 体验 | 能分析、能聊业务 | 建表写 SQL，少接触业务 |
| 技术前景 | 业务经验+AI 提效 | 实时数仓；方法论成熟，国内常因投入不足做不好 |

**交叉能力**：分析师懂数仓可小团队搭指标体系；数仓不懂分析则需求做不完。
