# 算法工程部署边界

## 一句话定义

工程部署要么一点都不做，要么做到底；模型调优+业务沟通已占满时间，兼工程是无底洞且投入产出比低。

## 详细说明

**常见陷阱**：
今天部署代码 → 明天维护平台 → 后天对接业务 → 同事离职被迫接手（不涨薪、无成长）

**两条路线（30 岁+须选）**：

| 路线 | 特点 |
| --- | --- |
| 纯算法 | 对接大模型等前沿，配资源/实习生做工程 |
| 算法+工程 | 适合数据建设不完善的小团队，**次优选** |

**工程为何不必恐惧**：
- 算法答案无限可无限优化；工程跑通一条路线即搞定
- Linux 本质是服务器 OS + 脚本流水账，文档/GitHub 成熟

**入职策略**：先本职交付 → 与领导画清边界 → 不必过度焦虑。
