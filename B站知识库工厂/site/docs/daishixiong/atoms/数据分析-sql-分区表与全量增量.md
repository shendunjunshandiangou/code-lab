# 分区表与全量增量

## 一句话定义

分区表逻辑上是普通筛选列、物理上是文件夹；日期分区最常用，配合全量（DF）或增量（DI）后缀控制数据更新策略。

## 详细说明

### 分区本质

- **逻辑**：分区列用法同普通 WHERE 筛选
- **物理**：每个分区值对应一个文件夹

### 常见分区

1. **场景分区**：按平台（抖音/B站）分文件夹
2. **日期分区**（最常用）：每天新数据入当天分区，取数时限制日期范围减少扫描、加速查询

### 全量 vs 增量

| 类型 | 后缀 | 含义 |
| --- | --- | --- |
| 增量 DI | DI | 新数据放入新分区文件夹 |
| 全量 DF | DF | 每个分区存截至当天的完整快照 |

### 使用要点

取数时把分区当普通字段 WHERE 限制即可，无需关心底层文件夹实现。面试常问是否用过 Hive 分区表。
