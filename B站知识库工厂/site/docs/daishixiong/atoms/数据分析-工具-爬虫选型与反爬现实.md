# 爬虫选型与反爬现实

## 一句话定义

招聘站等场景自写 Python 爬虫常因反爬直接失效；分析师应优先**后羿采集器/RPA**，并规范需求后再做 multi-source 清洗合并。

## 详细说明

**手写爬虫常见失败**：
- Boss 直聘：频繁请求 → 封 IP
- 拉勾/智联：需登录翻页
- 前程等：URL 加密，无法批量构造

**推荐路径**：
- **后羿采集器**：字段智能识别、预登录、多站分组
- **影刀 RPA**：模拟操作取数（见数据工程课）
- Python requests：适合学习原理，**生产慎用**

**企业流程**：需求澄清 → 可行性验证 → 字段梳理 → 分源爬取/清洗 → 合并宽表 → 可视化。

**无代码采集器最简操作流（短教程补充）**：粘贴网址 → 智能采集自动识别字段 → 右键改字段名 → 深入采集抓详情页（笔状指针点选新增字段）→ 启动采集 → 导出数据。只爬列表页很快，需逐条点击的深入采集较慢，免费版速度一般。

**原则**：爬虫是取数手段；分析师价值在清洗、口径、分析，非攻防。
