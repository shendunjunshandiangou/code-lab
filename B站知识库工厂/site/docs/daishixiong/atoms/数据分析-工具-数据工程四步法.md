# 数据工程四步法

## 一句话定义

任何数据技术需求可拆解为：**获取 → 存储 → 清洗 → 应用**；分析师应学工具联动而非死磕单点。

## 详细说明

| 步骤 | 含义 | 可选工具 |
| --- | --- | --- |
| 获取 | 把数据拿下来 | 后羿采集器、影刀 RPA、Python 爬虫 |
| 存储 | 存到哪决定怎么洗 | Excel、MySQL、数仓（常**先于清洗**确定） |
| 清洗 | 格式/口径处理 | Excel、Prep Builder、Power Query、Python |
| 应用 | 呈现与协作 | Tableau、Power BI、FineBI 等 |

**工程思维**：脑子里能串成链路（如 RPA 抓取 → 存 Excel → Prep 清洗 → BI 读文件夹）。

**选型原则**：测试成本最低优先；公司有什么用什么。

**反误区**：单工具（含 Python 全包）永远学不完；AI 时代更要会「选工具组方案」。
