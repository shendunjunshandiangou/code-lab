# 工作中问题驱动学习

## 一句话定义
不要为缓解知识焦虑而"为学而学"；技能型知识只能靠练习，应在工作中遇到问题再学、遇到兴趣点再深挖，保证学到的都被实际运用。

## 详细说明

- **反面模式**：强迫自己下班后学数据分析/编程/运营，结果不是放弃就是学会了也不会用——本质是为缓解知识焦虑的低效学习。
- **转折**：《刻意练习》的启发——学到的东西很少被实际运用，技能型知识光看绝对看不会，和学骑自行车一个道理。
- **正确模式**：工作中不会做活动运营就上网课快速提升，不会数据分析就自学，然后不停在实际工作中应用和总结，形成"遇到问题 → 学 → 用 → 总结"的闭环。
- **时间来源**：通勤时间（戴师兄当时每天 3 小时），不够再压缩休息。
- **效果佐证**：戴师兄用此模式四五个月成长为合格的全栈运营（独立策划科技活动，几百元预算做出 3 万+曝光、10% 转化率）。
