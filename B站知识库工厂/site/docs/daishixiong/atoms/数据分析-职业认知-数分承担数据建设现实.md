# 数分承担数据建设现实

## 一句话定义

理论上数据建设应由数据团队（研发+数仓+数分+产品）协作，实际上中小厂甚至大厂里大量清洗、落表、存储工作落在分析师身上。

## 详细说明

**标准团队配置（大厂）**：
- 研发：数据获取
- 数仓：清洗 + 存储 + 分层设计
- 数分：数据应用
- 产品：链路设计

**现实**：
- 除大厂外，中厂/小厂常无完整数仓团队
- 分析师被要求洗数据、写表、建看板一条龙
- 这也是「一没数据二没权限」困境的根源之一

**应对**：进公司先判断数据成熟度；建设不完善时推动最小可用链路，或自建练手库。
