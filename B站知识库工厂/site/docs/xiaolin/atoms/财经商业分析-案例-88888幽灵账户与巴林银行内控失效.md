# 88888 幽灵账户与巴林银行内控失效

## 一句话定义
里森利用自己做过后台、熟知系统漏洞的优势，建立了总部报表看不见的 88888「幽灵账户」隐藏亏损，并靠幽灵账户与主账户对倒做出虚假利润；巴林银行的内控在每个环节全部失效。

## 详细说明
### 幽灵账户的运作
- 里森曾在摩根士丹利和巴林银行长期做后台清算，深知巴林后台系统「漏洞百出千疮百孔」。
- 他新建账户并修改代码，使其不出现在总部报表中，成为幽灵账户，代码 88888。
- 亏损进账户后，还需保证金：他以「与客户交易需要保证金」为幌子向伦敦总部要钱，总部从不追问细节。
- 后期升级为**对倒交易**：用 88888 账户与主账户互为对手方交易，把主账户利润做高（1994 年做出 2800 万英镑虚假利润），总部看到高利润反而更愿意供给保证金。

### 内控失效的四个环节
1. **系统层**：一个交易员可以随意隐藏账户；
2. **资金层**：总部对巨额保证金申请「眼都不眨」地放款，不见回头钱也长期不查；
3. **审计层**：1994 年审计到场，被里森现编的「复杂新型客户交易协议」忽悠，未调阅报告即离开（与 SEC 审麦道夫如出一辙——人性爱偷懒）；
4. **管理层**：不理解衍生品业务，只看利润不问来源；资深同行已质疑其夏普比率高得不可能，管理层反斥为「嫉妒」。

### 为什么交易记录也看不出问题
衍生品交易员每天做成百上千笔交易，记录里有赚有亏，外人无法还原交易逻辑——事后审查交易流水并不能替代实时的权限与账户管控。
