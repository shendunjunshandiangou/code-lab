# 中美Agent落地路线差异

## 一句话定义

美国 Agent 以 B 端企业应用为主（PC/SaaS 生态成熟）；中国可能跳过 PC Coding 直达移动端 C 端「许愿编程」，类比跳过信用卡直达移动支付。

## 详细说明

| 维度 | 美国 | 中国 |
| --- | --- | --- |
| 主战场 | B 端（企业应用占 AI 市场 90%+） | C 端移动端 |
| 基础设施 | PC、SaaS 中小企业支付意愿高 | 手机普及率远高于 PC，很多行业不需 PC |
| 产品路径 | Claude Code 等 B 端渗透 | 灵光等 C 端 Wish Coding / 闪应用 |
| 历史类比 | 信用卡阶段 | 跳过信用卡 → 移动支付；跳过 PC Coding → 移动端 |

**中国选 C 端理由**（蚂蚁灵光）：
- 普惠基因：让更多普通人采纳新技术
- 中国 SaaS/企业软件生态与美国本质不同
- 大量小微企业可用移动端低成本解决问题
- 真正用过 Claude Code 的中国用户极少

各路径殊途同归，探索解决复杂、长程、开放问题；终极目标一致，场景与路径不同。
