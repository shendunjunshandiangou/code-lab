# TRS 总收益互换机制

## 一句话定义
TRS（Total Return Swap，总收益互换）本质是一份对赌合同：客户与投行约定"标的股票每涨 1% 投行付 X、每跌 1% 客户付 X"，客户只需缴纳一部分保证金即可获得远超融资融券上限（美国法定最多 2 倍）的杠杆（Archegos 案约 5 倍）；投行通过收保证金对冲信用风险、买入等额正股对冲市场风险，旱涝保收赚服务费——副作用是**股票登记在投行名下，真实庄家彻底隐身**。

## 详细说明
- **杠杆算术**：100 万保证金对赌 500 万名义仓位——涨 20% 赚 100 万（本金翻倍），跌 20% 本金归零。
- **投行的"零风险"拼图**：①信用风险→收保证金，标的下跌则要求追加，否则变卖保证金退出；②市场风险→在市场买入等额正股，对赌赔付与持股盈亏对消；③盈利→服务费（Archegos 一年给六家投行贡献上亿美元服务费）。
- **隐匿属性**：持仓人是投行而非客户，客户即使成为某股票实际第一大股东也无需披露、无人知晓——这既是客户的"隐性福利"，也是监管盲区和系统性风险来源：多家投行可同时对同一客户敞口而互不知情。
- **风险的真面目**：投行的"零风险"只在客户不违约时成立——客户破产瞬间，对冲用的正股变成投行自己的裸头寸，被迫踩踏抛售。
