# 阿里去 IOE 与算力瓶颈

## 一句话定义

2007 年前后阿里电商业务暴涨，每天早高峰服务器使用率飙到 98%，标准化的 IOE 方案（IBM 服务器 + Oracle 数据库 + EMC 存储）成本可能拖垮淘宝且技术迭代慢，倒逼阿里自研云计算；2013 年 7 月淘宝最后一个 Oracle 数据库下线，IOE 彻底成为历史。

## 详细说明

- IOE 两大问题：①业务量增加就要买更多设备，成本压力巨大；②算力需求极大时 Oracle 跟不上，技术更新慢。
- 王坚在 2007 年阿里网侠大会上一句"没有技术，阿里未来必死"戳中马云，加上阿里确有底层架构刚需、两人又是杭州老乡，一拍即合成立阿里云，王坚任首席架构师。
- 去 IOE 的第一道阻力来自内部：换新架构要拆老架构，原数据库架构师反对（抢饭碗 + 云不安全谁负责）；王坚以"数据库已病入膏肓，不砸自己饭碗阿里撑不下去"说服，2009 年秋工程师含泪拆掉自己建的数据库。
- 通用启示：大公司的技术转型，组织阻力往往先于技术难题。
