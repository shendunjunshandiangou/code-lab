# AOSP、安卓与厂商 UI 三层架构

## 一句话定义
手机操作系统分三层：底层开源的 AOSP（安卓开源项目）→谷歌基于 AOSP 开发的安卓系统→各厂商基于安卓定制的 UI（MIUI、Flyme、EMUI 等）；2019 年美国对华为的行政命令切断的是第二层——AOSP 开源仍可用，但安卓系统不能再升级，对大厂等于事实断供。

## 详细说明
- 三层关系：
  1. **AOSP**（Android Open Source Project）：谷歌开发的底层开源项目，任何人可用；
  2. **安卓系统**：谷歌基于 AOSP 开发、附带谷歌服务的操作系统；
  3. **厂商定制系统**：小米 MIUI、魅族 Flyme、华为 EMUI 等在安卓之上做调整优化。
- 制裁的精准打击点：2019 年 5 月 15 日特朗普行政命令后，谷歌停止与华为合作——华为可继续用开源的 AOSP，但无法获得安卓系统更新。长期停留在老版本，对华为这种体量的厂商基本等于不能用。
- 含义：开源许可 ≠ 供应链安全；生态控制权掌握在上一层的所有者手里，「系统不在自己手里」与「芯片不在自己手里」同为终端厂商的两大命门——这直接催生了鸿蒙。
