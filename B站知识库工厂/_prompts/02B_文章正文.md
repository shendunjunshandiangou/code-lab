# 阶段 1B · 文章正文：字幕 → 润色全文（追加）

> 本阶段是阶段 1（`_prompts/02_重写.md`）的**可选追加**，不替代重写。
> 触发方式：实例 `项目配置.md` 启用阶段 1B，或人类说「给逐视频文章追加正文」；
> 任务队列阶段列写 `1B`，读 `_pipeline_tasks.md` 执行所有 ⏳ 的 1B 任务。
>
> **不影响**已完成的阶段 2–4（拆解 / 大纲 / 成文不需重跑）。

## 输入 / 输出

| 项 | 说明 |
| --- | --- |
| 输入（主） | 实例 `00_raw/` 中任务指定的字幕 Markdown |
| 输入（辅） | 实例 `01_articles/` 中同名文章（只读上半部分，对照是否遗漏） |
| 输出 | 原地更新 `01_articles/` 同名文件，在文末追加 `## 文章正文` |

## 任务目标

在**保留现有结构化板块不动**的前提下，基于字幕原文写出一段**经润色、可独立成文**的全文，追加到文章末尾。

## 文章最终结构（固定顺序）

```markdown
---
来源: 00_raw/<原始文件名>
视频标题: <标题>
UP主: <从项目配置或 frontmatter 获取>
视频链接: <如有>
处理日期: <YYYY-MM-DD>
阶段: 01B_文章正文完成
正文字数: <N>          # 可选，便于批量进度追踪
---

# 标题

**摘要**：...

## 一、...          ← 现有结构化板块，禁止改动
...

---

## 文章正文          ← 本阶段新增

[连续段落组成的润色全文，可独立阅读]
```

## 执行规则

### 1. 上半部分不动

- 从 `# 标题` 到 `---` 分隔线之间的内容（frontmatter、摘要、结构化 `##` 板块）**只读不写**。
- 若 `01_articles/` 中尚无结构化文章（无摘要或无 `##` 分节），**停止**，提示人类先跑阶段 1。
- 若文章末尾已有 `## 文章正文`：通读并与 `00_raw/` 对照，若已覆盖主要段落则**跳过**，任务日志注明「已有正文，跳过」；若明显缺段（如仍写「详见字幕原文」、或正文远短于 raw 的 40%），则**补写缺段**而非整篇覆盖。

### 2. 主取材 `00_raw/`

- 以字幕原文为**信息上限**；结构化上半部分仅作「是否遗漏要点」的对照。
- **不得**因上半部分已写而省略原文中的案例、数字、步骤、Q&A、原话。
- 多 P 字幕（`## 分P N`）按顺序合并叙述，保留分 P 之间的逻辑转折。

### 3. 润色标准

| 操作 | 内容 |
| --- | --- |
| **删** | 时间戳、语气词（「啊」「就是说」「对吧」）、重复口头禅、无关寒暄、抽奖、恰饭 |
| **留** | 观点、案例、数据、操作步骤、原话金句（引号标注）、Q&A 往返、讲者个人判断 |
| **改** | 口语 → 书面语；补全/修正专业术语（拿不准保留原文并标 `（存疑）`）；段间加过渡句 |
| **禁** | 新增原视频没有的知识、案例、结论；主观评价讲者 |

### 4. 文体要求

- 写成**可独立成文**的连续叙述；默认不用 `##` 小标题。
- **超长直播 / Q&A**（raw 去噪后 > 8000 字）：可按话题用 `###` 分大段，但**不做**要点列表化，**不得**用「详见字幕原文」敷衍。
- **直播 Q&A**：保留问—答可读性（叙述式引用或明确「问：… / 答：…」块均可）。
- **教程 / 操作类**：步骤融入叙述；关键命令、快捷键、函数名保留 `` `代码` `` 或代码块。

### 5. 篇幅

- 正文应**明显长于**结构化上半部分（时长 < 3 分钟的短视频除外）。
- 字数参考：`00_raw/` 去噪后字符数的 **40%–80%**；框架级长直播、万字 Q&A 取**上限**。
- 短视频若 structured 部分已接近全文信息量，正文仍须写成**连贯段落**，而非复制上半部分。

### 6. 质量自检（完成后逐条核对）

- [ ] 通读 `## 文章正文`，不读上半部分也能读懂主线
- [ ] 与上半部分结构化要点**无矛盾**（细节可更丰，不可相反）
- [ ] 无时间戳、无「详见字幕」类偷懒表述
- [ ] frontmatter `阶段` 已更新为 `01B_文章正文完成`

### 7. 与后续阶段的关系

- 阶段 2 拆解**不**扫描 `## 文章正文` 节（见 `_prompts/03_拆解.md`）。
- 阶段 4 成文回溯 `01_articles/` 时，**可**引用正文节中的案例与话术。

## 完成标准

- 文件含完整双板块：结构化要点 + `---` + `## 文章正文`
- 正文满足上述自检清单
- 任务日志记录：正文字数、是否跳过、是否补写
