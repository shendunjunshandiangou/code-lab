# 阶段 0B · 导入书籍：书 → 知识库实例 →（可选）上线网站

> 触发方式：人类说"**把《<书名>》导入知识库**"或"我想把一本书做成知识库"。
> 本流程是 `_prompts/01_采集.md` 的书籍版替代：内容源从「视频字幕」换成「书籍章节」，
> 阶段 2-4（拆解 / 大纲 / 成文）与视频流水线完全相同，不再重复定义。
> 任何模型（Cursor / Claude Code / Codex / 其他）接手时，先读完 `AGENTS.md`，再按本文件执行。

---

## 第 1 步：确认配置（缺什么问什么，最多问 3 个问题）

1. **书**：书名、作者、原文文件路径（md / txt / epub / pdf）。
2. **目标读者**：给谁看？（默认：零基础学习者）
3. **是否上线网站**：只建知识库，还是同时发布到 `site/` 网站？（默认：先建库，人类确认质量后再上线）

## 第 2 步：创建实例目录

1. 实例名：`<书名>-<领域>`（如 `纳瓦尔宝典-财富与幸福`），放在 `知识库/` 下。
2. 创建五个阶段目录：`00_raw/`、`01_articles/`、`02_atoms/`、`03_outline/`、`04_knowledge/`。
3. 复制 `_template/` 下的 `项目配置.md` 和 `_pipeline_tasks.md` 进实例目录（书籍来源不需要 `_采集清单.md`，用「章节清单」代替，见第 3 步）。
4. `项目配置.md` 中注明：来源类型 = 书籍，书名、作者、出版信息、原文格式、目标读者。

## 第 3 步：拆章（替代视频采集）

1. 把原书按章拆成 `00_raw/` 下的 Markdown 文件，一章一个，命名 `第NN章-章节标题.md`（NN 两位数字保证排序）。
   - **md / txt**：按章节标题正则拆分。
   - **epub**：先解包（epub 本质是 zip + html），逐章转 Markdown。
   - **pdf**：先提取文本再拆章；扫描版 pdf 提取质量差时，如实告知人类，不要硬拆。
2. 序言、后记、附录如有实质内容，同样入库（命名 `序言-xxx.md` 等）；目录页、版权页、推荐语不入库。
3. 在实例目录下生成 `_章节清单.md`，列出全部拆出的章节与状态。
4. **停下来，等人类审核章节清单**（是否有拆错、缺章、该排除的部分），确认后再登记 `_pipeline_tasks.md` 并继续。

## 第 4 步：轻度整理（替代阶段 1 重写）

书稿本身可读，**不做口语转写式重写**，只做：

1. 清理转换残留：页眉页脚、页码、断行、乱码。
2. 结构化：恢复标题层级（`##` / `###`）、列表、表格、引用块；开头加 3 行以内摘要。
3. **禁止改写作者的表达、禁止增删内容**；转换缺字处标 `（原文此处不清晰）`。
4. 产出 `01_articles/`（一章一篇），frontmatter 写：

```yaml
---
来源: 00_raw/<章节文件名>
书名: <书名>
作者: <作者>
章节: 第NN章 <章节标题>
处理日期: <YYYY-MM-DD>
阶段: 01_整理完成
---
```

## 第 5 步：拆解 / 大纲 / 成文（与视频流水线相同）

逐章执行，每完成一章更新 `_pipeline_tasks.md`：

- 拆解原子 → 读 `_prompts/03_拆解.md`，原子命名 `<领域>-<子领域>-<概念名>.md`
- 大纲更新 → 读 `_prompts/04_大纲更新.md`
- 成文 → 读 `_prompts/05_成文.md`

## 第 6 步（可选）：上线到网站

仅当人类确认后执行。网站项目在 `site/`，构建脚本为 `site/scripts/build-content.mjs`。

1. **注册实例**：在 `build-content.mjs` 的 `VAULTS` 数组新增条目：
   - `key`：书名拼音或英文短名（URL 用）；`dir`：实例目录名
   - `name` / `subject` / `description` / `tags`：按书的主题写
   - `type: 'book'`（现有 UP 主实例视为 `type: 'up'`，如字段不存在则先加上）
2. **文案按来源类型区分**：站内所有「视频文章 / 逐视频文章 / UP 主」字样（首页卡片、目录页 `CatalogGrid.vue`、阅读层级页、库入口页、侧边栏分组标题），书籍实例应显示「章节精读 / 逐章阅读 / 作者」。实现方式：根据 `type` 字段选择标签，不要写死。
3. **卡片素材**：书封面处理成黑白线稿透明底 PNG（运行 `site/scripts/make-lineart.mjs`，在脚本里加一行文件名），头像位用书封面小图或作者像，均放 `site/assets/images/`。
4. **章节分组**：在 `build-content.mjs` 的 `ARTICLE_GROUP_RULES` 里按书的部/篇结构给该实例配置分组。
5. **兼容确认**：书籍文章没有 `视频链接` frontmatter，页面不应渲染 B 站播放器（现有逻辑天然兼容，验证即可）。
6. **版权声明**：在 `site/docs/about.md` 和该实例库入口页注明：内容摘自《<书名>》（作者），仅供个人学习，请支持正版。
7. **构建验证**：`cd site && npm run sync && npm run build`。有浏览器工具时，桌面（1440px）+ 移动（390×844）各截图检查新实例的首页卡片、目录页、文章页、原子页；没有浏览器工具时至少保证构建零报错。
8. **部署**（需人类确认）：提交后用 `git subtree split --prefix="B站知识库工厂/site"` 推送到 `bili-knowledge` 仓库 main 分支（force push），并推送 `code-lab` 远端。线上地址：https://shendunjunshandiangou.github.io/bili-knowledge/

## 完成标准

- `00_raw/` 章节与原书目录一一对应，无缺章
- `01_articles/` 保持作者原意，无 AI 添油加醋
- 原子、大纲、成文符合 AGENTS.md 第 6、7 节标准
- 如上线：网站构建通过，书籍实例文案无「视频 / UP 主」字样残留，版权声明齐全

## 通用约束（对所有模型强调）

- 遵守 AGENTS.md 全部规则：半自动模式、逐文件处理、及时回写任务状态
- 两个**必须停下来等人类**的节点：① 章节清单审核（第 3 步）；② 上线部署前（第 6 步）
- 不得虚构原书没有的观点、案例、数据
