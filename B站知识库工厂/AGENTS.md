# B站视频知识库工厂 · 全局规则

> 本仓库是一个「知识库工厂」：输入任意一个 B 站 UP 主、或任意一个想学的主题，
> 通过标准化的流水线，产出两种成品：
> ① 结构化的 Obsidian **知识库**（存知识）；② 可选的 **思维体**（用知识——让 AI 用该 UP 主的思维解决实际问题）。
> 本文件是全局唯一规则来源。任何 AI 模型（Cursor / Claude Code / Codex / 其他）接手本仓库时，必须先完整读完本文件。

---

## 1. 方法论总览：五阶段流水线 + 蒸馏

```text
采集 → 重写 → 拆解 → 结构化（大纲） → 成文 →（可选）蒸馏思维体
└──────────────── 知识库 ────────────────┘    └──── 思维体 ────┘
```

| 阶段 | 提示词文件 | 输入 | 输出 |
| --- | --- | --- | --- |
| 0 采集 | `_prompts/01_采集.md` | UP 主 / 搜索主题 | `_采集清单.md` + `00_raw/` 字幕 |
| 0B 导入书籍 | `_prompts/07_导入书籍.md` | 书籍原文（md/txt/epub/pdf） | `_章节清单.md` + `00_raw/` 章节 |
| 1 重写 | `_prompts/02_重写.md` | `00_raw/` | `01_articles/` 可读文章 |
| 2 拆解 | `_prompts/03_拆解.md` | `01_articles/` | `02_atoms/` 原子知识点 |
| 3 大纲 | `_prompts/04_大纲更新.md` | 新增原子 | `03_outline/大纲.md` 增量更新 |
| 4 成文 | `_prompts/05_成文.md` | 大纲 + 原子 | `04_knowledge/` 最终文章 |
| 5 蒸馏 | `_prompts/06_蒸馏思维体.md` | 完整知识库实例 | `思维体/<UP主名>/` 三件套 |

阶段 0B 是阶段 0 的书籍版替代：内容源从视频字幕换成书籍章节，阶段 1 简化为轻度整理（不改写作者表达），阶段 2-4 完全复用；可选的网站上线步骤也在 `_prompts/07_导入书籍.md` 中。

**执行任何阶段前，必须先读对应的提示词文件。**

阶段 5 为可选阶段，仅适用于模式 A（UP 主全集）实例，由人类说"**把 <UP主名> 蒸馏成思维体**"触发；前置条件、五步流程和质量标准全部在 `_prompts/06_蒸馏思维体.md`。

## 2. 三种建库模式

| 模式 | 触发场景 | 采集范围 |
| --- | --- | --- |
| **A · UP 主全集** | "帮我把某 UP 主的视频做成知识库" | 该 UP 主空间下全部（或按规则筛选后的）视频 |
| **B · 主题搜索** | "我想系统学习某个知识点" | B 站搜索该关键词的前 N 页 + 相关搜索词各前 N 页，去重、筛选后的视频 |
| **C · 书籍导入** | "把《某书》导入知识库" | 一本书的全部章节（md/txt/epub/pdf） |

三种模式只在**阶段 0（采集/导入）**有差异，阶段 2-4 完全相同（模式 C 的阶段 1 简化为轻度整理）。模式 A/B 见 `_prompts/01_采集.md`，模式 C 见 `_prompts/07_导入书籍.md`。

## 4. 实例规范

- 每个知识库 = `知识库/` 下的一个独立文件夹，互不干扰。
- 实例的一切个性化信息（UP 主、主题、读者、筛选规则）只写在该实例的 `项目配置.md` 里；**提示词和全局规则永远不写死具体实例的信息**。
- 新建实例：读 `_prompts/00_新建知识库.md`，按流程走。
- 原子笔记命名 `领域-子领域-概念名.md`，领域前缀天然避免跨实例重名。

### 思维体规范

- 每个思维体 = `思维体/` 下的一个独立文件夹（按 UP 主名命名），由三件套构成：`人格卡.md`（思维内核）、`方法论索引.md`（问题类型 → 框架 → 原子双链的检索地图）、`调用提示词.md`（人类使用入口）。
- 思维体与知识库**单向依赖**：蒸馏和调用时只读知识库，永不写入；不同思维体之间互不引用。
- 人格卡中每条观点必须附真实原子双链，口头禅必须是原话摘录；无来源的内容一律不写。
- 新建/迭代思维体：读 `_prompts/06_蒸馏思维体.md`；模块使用说明见 `思维体/README.md`。

## 5. 工作模式：半自动（严禁全自动）

- **人类负责**：决定建哪个库、审核采集清单（哪些视频值得进）、触发流水线、对产出做最终判断。
- **AI 负责**：采集清单生成、字幕获取、执行 `_pipeline_tasks.md` 里的 ⏳ 任务、回写状态和日志。
- AI **不得**处理任务队列之外的文件；发现未登记的新字幕只提醒，不擅自处理。
- AI **不得**搭建 watchdog / 定时任务等全自动方案，除非人类明确要求。
- 采集清单生成后**必须停下来等人类审核勾选**，再批量获取字幕。

标准触发指令：

> 在 `知识库/<实例名>/` 下，读取 `_pipeline_tasks.md`，执行所有 ⏳ 任务。

蒸馏思维体触发指令（人类确认知识库已成形后使用）：

> 把 <UP主名> 蒸馏成思维体。

导入书籍触发指令：

> 把《<书名>》导入知识库。

## 6. 原子笔记三条合格标准

1. **可独立解释**：不依赖原文上下文也能讲清楚。
2. **可独立搜索**：可以被单独引用。
3. **可合并的不强拆**：两个概念总是同时出现且无法分开时，合并为一个原子。

## 7. 大纲增量更新四情况

| 情况 | 处理 |
| --- | --- |
| 大纲已覆盖该知识点 | 跳过，不重复写 |
| 有相关章节但角度不同 | 在该章节追加双链引用 `[[原子文件名]]` |
| 大纲完全没有 | 新增子条目，扩展大纲框架 |
| 与主线不相关 | 跳过，并在日志中注明 |

## 8. 通用约束

- 所有产物为 Markdown，UTF-8 编码，中文可读命名。
- 双链用 Obsidian 语法 `[[文件名]]`（不带路径、不带 .md 后缀）。
- **不得虚构原视频/原书没有的数据、案例、结论**；字幕或原文缺失处标注 `（原文此处不清晰）`，不脑补。
- 长字幕/长章节注意上下文窗口：一次只处理一个文件，逐文件完成。
- 每完成一条任务，立即更新对应实例的 `_pipeline_tasks.md` 状态和日志，再处理下一条。
- 多视频讲同一概念时，原子笔记做合并增补，不覆盖、不重复建文件。
